선형 점화식과 행렬 거듭제곱

N ≤ 1018 인 피보나치도 눈 깜짝할 사이에 — O(N)에서 O(log N)으로

중등 정보올림피아드(KOI) 고난도 대비 | 점화식 심화 강의

문제 상황: N이 1018이라면?

피보나치 수열은 대표적인 선형 점화식입니다.

F(1) = 1,   F(2) = 1,   F(n) = F(n-1) + F(n-2)  (n ≥ 3)

"선형(linear)"이란 앞의 항들에 상수를 곱해 더한 꼴이라는 뜻입니다. (제곱, 곱셈 F(n-1)×F(n-2) 같은 것이 없음)
지난 시그마 강의에서 배운 대로, 점화식은 반복문으로 바로 번역됩니다. 그런데…

반복문 vs 행렬 거듭제곱 — 연산 횟수 대결

N의 크기를 조절해 보세요. 1초에 약 1억(108)번 연산한다고 가정합니다.

O(N) 반복문
O(log N) 행렬

핵심 아이디어 미리보기

① 점화식 한 번 진행 = 행렬 하나를 곱하는 것과 같다.
② 그러면 N번 진행 = 행렬의 N제곱이다.
③ 거듭제곱은 "제곱을 반복"하는 분할 정복으로 log₂N 번 만에 구할 수 있다.
결론: 점화식 문제 → 행렬 거듭제곱 문제로 바꾸면 N이 아무리 커도 60번 남짓 계산으로 끝!

왜 답에 mod(나머지)를 취할까?

F(90)만 되어도 2,880,067,194,370,816,120 — long long 최대치(약 9.2×1018)에 육박합니다. F(1018)은 우주에 다 적을 수도 없는 크기라서, 대회에서는 "1,000,000,007로 나눈 나머지를 출력하라"는 형태로 출제됩니다. 나머지 연산은 곱셈·덧셈 할 때마다 취해도 결과가 같다는 성질을 이용합니다.

(a × b) mod p = ((a mod p) × (b mod p)) mod p

점화식을 행렬 곱으로 바꾸기

행렬을 처음 본다면 "숫자를 직사각형으로 배열하고, 곱하는 규칙을 약속한 것" 정도로 충분합니다. 곱셈 규칙은 단 하나: (앞 행렬의 행) × (뒤 행렬의 열)을 짝지어 곱해 더한다.

1단계 — 피보나치 상태를 세로 벡터로 묶기

F(n)을 계산하려면 직전 두 항이 필요합니다. 그래서 연속한 두 항을 한 묶음(상태)으로 봅니다.

F(n+1)F(n) = 1110 × F(n)F(n-1)

1행: 1×F(n) + 1×F(n-1) = F(n+1)  (점화식 그 자체!)
2행: 1×F(n) + 0×F(n-1) = F(n)  (그냥 한 칸 내려 보관)
즉 행렬 M = [[1,1],[1,0]] 을 한 번 곱하면 수열이 한 걸음 전진합니다. N걸음 전진 = M을 N번 곱하기 = MN.

2단계 — 행렬 곱셈 계산기 (성분별 단계 실행)

M을 거듭 곱하며 Mk를 만들어 봅니다. [한 성분 계산]을 누르면 행×열 짝짓기가 어떻게 이루어지는지 보여 줍니다.

피보나치 대조표 — 행렬 성분이 곧 피보나치!

Mk = F(k+1)F(k)F(k)F(k-1)  →  오른쪽 위 성분만 읽으면 F(k)

분할 정복 거듭제곱 — 제곱을 반복하라

행렬을 N번 일일이 곱하면 도로 O(N)입니다. 진짜 마법은 거듭제곱을 빠르게 하는 데 있습니다. 먼저 숫자로 원리를 익힙시다.

핵심 관찰: 지수를 반으로 접기

a13 = a8 × a4 × a1   (13 = 1101(2) = 8 + 4 + 1)

a → a² → a⁴ → a⁸ → … 처럼 제곱만 반복하면 a의 2k 제곱들이 공짜로 얻어집니다. 그중 지수의 이진법 표현에서 1인 자리에 해당하는 것들만 골라 곱하면 끝! 13번 곱할 것을 단 5번의 곱셈으로 해결합니다. N = 1018이어도 곱셈 약 120번이면 충분합니다.

숫자 거듭제곱 단계 실행기

이진법을 오른쪽(1의 자리)부터 읽습니다. 코드의 n % 2, n /= 2 와 같은 방향입니다.

단계현재 비트행동base (a2ᵗ)result

최종 결과: -

알고리즘 요약 (이대로 코드가 됩니다)

result = 1                // 항등원에서 출발 (행렬이면 단위행렬 I)
while (n > 0) {
    if (n이 홀수)  result = result * base
    base = base * base    // 제곱을 반복: a¹→a²→a⁴→a⁸…
    n = n / 2             // 지수를 반으로 접기
}

반복 횟수 = 이진법 자릿수 = ⌈log₂N⌉. 이 뼈대에서 곱셈만 행렬 곱셈으로 바꾸면 그대로 행렬 거듭제곱이 됩니다.

특성방정식과 일반항 — 수학으로 한 방에

행렬이 "계산"으로 점화식을 정복한다면, 특성방정식은 "수식"으로 점화식을 정복합니다. 닫힌 형태(일반항)를 알면 수렴 속도, 크기 어림, 증명 문제에 강해집니다.

1단계 — 특성방정식 세우기

점화식 F(n) = F(n-1) + F(n-2)의 해가 F(n) = xⁿ 꼴일 것이라 추측하고 대입합니다.

xⁿ = xⁿ⁻¹ + xⁿ⁻²  →  (양변을 xⁿ⁻²로 나누면)   x² = x + 1

이 이차방정식이 특성방정식(characteristic equation)입니다. 점화식의 계수를 그대로 옮겨 적으면 됩니다: F(n) = a·F(n-1) + b·F(n-2)  →  x² = ax + b

2단계 — 근을 구하고 일반항 만들기

x² - x - 1 = 0  →  x = (1 ± √5) / 2   ⇒   φ ≈ 1.618… (황금비),   ψ ≈ -0.618…

서로 다른 두 근이 나오면 일반항은 두 근의 거듭제곱을 섞은 꼴이고, 계수는 초기 조건 F(1)=F(2)=1로 결정합니다. 그 결과가 유명한 비네(Binet) 공식:

F(n) = ( φⁿ - ψⁿ ) / √5

|ψ| < 1 이므로 ψⁿ은 금방 0에 가까워집니다. 그래서 F(n) ≈ φⁿ/√5 를 반올림한 값이라는 놀라운 사실이 나옵니다.

비네 공식 검증기

φⁿ/√5 (실수 계산)반올림실제 F(n)일치?
----

⚠ 코딩에서 비네 공식을 그대로 쓰면 안 됩니다. n이 70을 넘으면 double의 정밀도(약 15~16자리)가 부족해 반올림이 어긋나기 시작합니다. 이론·어림용은 비네, 정확한 계산은 행렬 — 역할이 다릅니다.

3단계 — 누적 합도 점화식 안에 넣기

"F(1)+F(2)+…+F(N)을 구하라"(N ≤ 1018)는 변형도 단골입니다. 두 가지 무기가 있습니다.

무기 1 — 항등식: 시그마 강의의 누적 개념으로 증명되는 유명한 식

S(n) = F(1) + F(2) + … + F(n) = F(n+2) - 1
n1234567
F(n)11235813
S(n)1247122033
F(n+2)-11247122033

→ 합 문제가 F(n+2) 하나 구하기로 바뀌므로 행렬 거듭제곱을 그대로 재사용!

무기 2 — 상태 확장(3×3 행렬): 항등식이 없는 일반 점화식이라면, 합 S를 상태에 직접 포함시킵니다.

F(n+1)F(n)S(n) = 110100011 × F(n)F(n-1)S(n-1)

셋째 행이 S(n) = F(n) + S(n-1) — 바로 시그마의 점화식입니다. 상태에 넣고 싶은 값(합, 짝수 항 합, 개수 등)을 행 하나로 추가하는 이 기법이 행렬 거듭제곱 문제의 진짜 일반화입니다.

2단계 — 곱셈만 행렬로 교체한 최종 코드

    
구조는 1단계와 완전히 동일합니다. 곱셈이 mul() 함수 호출로 바뀌고, 1이 단위행렬 I로 바뀌었을 뿐입니다.

    
#include <stdio.h>
typedef long long ll;
const ll MOD = 1000000007;

/* 2x2 행렬 곱: C = A x B (성분마다 mod) */
void mul(ll A[2][2], ll B[2][2], ll C[2][2]) {
    ll T[2][2] = {{0,0},{0,0}};
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++)
            for (int k = 0; k < 2; k++)
                T[i][j] = (T[i][j] + A[i][k] * B[k][j]) % MOD;
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++) C[i][j] = T[i][j];
}

ll fibo(ll n) {                       /* F(n) mod 1e9+7 */
    ll R[2][2] = {{1,0},{0,1}};       /* result = 단위행렬 I */
    ll M[2][2] = {{1,1},{1,0}};       /* base = 점화식 행렬 */
    while (n > 0) {
        if (n % 2 == 1) mul(R, M, R);   /* 비트가 1이면 답에 곱하기 */
        mul(M, M, M);                   /* base 제곱 */
        n /= 2;
    }
    return R[0][1];                   /* M^n 의 오른쪽 위 = F(n) */
}

int main(void) {
    ll n;
    scanf("%lld", &n);
    printf("%lld\n", fibo(n));
    return 0;
}

    

      
숫자 버전행렬 버전의미

      
result = 1R = I (단위행렬)곱셈의 항등원에서 출발

      
result *= amul(R, M, R)비트가 1인 자리만 답에 반영

      
a *= amul(M, M, M)제곱을 반복해 M2ᵗ 생성

      
n /= 2n /= 2지수의 이진법을 오른쪽부터 소비

❌ 실수 1 — result를 0 또는 M으로 초기화

❌ 실수 2 — mod를 마지막에 한 번만

    
T[i][j] += A[i][k] * B[k][j];        /* mod 없이 누적 → 오버플로 */

    
성분 하나가 최대 약 109이므로 곱 하나가 1018 — 두 개만 더해도 long long을 넘습니다. 곱셈마다 % MOD가 철칙입니다.
