시그마(Σ)란 무엇인가?

수학에서 Σ(시그마)는 그리스 문자 S에 해당하며, Sum(합)의 머리글자입니다. "규칙이 있는 수들을 일일이 쓰지 않고 한 기호로 더하겠다"는 약속이 바로 시그마 표기법입니다.

시그마 기호의 해부도

기호 하나에 4가지 정보가 들어 있습니다. 각 부분에 마우스를 올려 보세요.

읽는 방법

5Σi=1 i  =  1 + 2 + 3 + 4 + 5  =  15

읽기: "i가 1부터 5까지 변할 때, i를 모두 더하라"
즉 i에 1, 2, 3, 4, 5를 차례로 대입하여 나온 값들을 전부 더하는 것입니다.

직접 전개해 보기 (인터랙티브)

일반항과 끝값 n을 바꾸면 시그마가 어떻게 펼쳐지는지(전개) 즉시 확인할 수 있습니다.

시그마 시각화 시뮬레이터

시그마는 결국 "막대를 하나씩 쌓아 올리는 것"과 같습니다. 한 단계씩 진행하며 각 항이 누적 합에 더해지는 과정을 눈으로 확인하세요.

현재 i
-
이번 항 a_i
-
누적 합 S
0

아직 아무것도 더하지 않았습니다. S = 0에서 출발합니다.

이 그림이 말해 주는 것

① 시그마는 순서대로 하나씩 더하는 절차이다. ② 매 순간 필요한 것은 지금까지의 합(누적 변수)과 이번에 더할 항 딱 두 가지다. ③ 이 구조가 그대로 점화식이 되고, C 언어의 반복문 + 누적 변수가 됩니다.

점화식으로 다시 보는 시그마

점화식(recurrence relation)이란 "앞 단계의 결과를 이용해 다음 단계를 정의하는 식"입니다. 시그마를 점화식으로 바꾸는 순간, 코딩으로 가는 문이 열립니다.

합 S(k)를 점화식으로 정의하기

"1부터 k까지의 합"을 S(k)라고 하면, 새 항 하나를 더하는 규칙은 다음과 같습니다.

S(0) = 0   (아직 아무것도 안 더함)
S(k) = S(k-1) + ak   (이전 합에 k번째 항을 더한다)

이 두 줄이 시그마의 전부입니다. 초기값 1줄 + 반복 규칙 1줄.

점화식 단계별 실행기

k점화식 계산S(k)

S(0) = 0 부터 시작합니다. [다음 단계]를 눌러 보세요.

점화식 ↔ C 코드 1:1 번역표

S(0) = 0
→
int sum = 0;
k = 1, 2, …, n
→
for (i = 1; i <= n; i++)
S(k) = S(k-1) + ak
→
sum = sum + a(i);

코드의 sum은 "직전까지의 합 S(k-1)"을 담고 있다가, 대입문이 실행되는 순간 "S(k)"로 갱신됩니다. 변수 하나가 점화식의 S(k-1)과 S(k) 역할을 번갈아 맡는 것이 핵심입니다.

같은 답, 두 가지 길: 반복문 vs 공식

    

      

        
방법 1 — 점화식(반복문): O(n)

        
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;      /* S(i)=S(i-1)+i */
}

      

      

        
방법 2 — 합 공식: O(1)

        
int sum = n * (n + 1) / 2;
/* n이 커도 계산 1번! */

      

    

    

      n이 10억이라면? 방법 1은 10억 번 반복하지만 방법 2는 곱셈 한 번입니다.
      단, n이 크면 int 범위(약 21억)를 넘으므로 long long을 써야 합니다.
    

  







  
자주 하는 실수 포인트


  

    
❌ 실수 1 — sum을 초기화하지 않음

    
int sum;              /* 쓰레기값! */
for (int i = 1; i <= n; i++) sum += i;

    
점화식의 S(0) = 0이 빠진 것과 같습니다. 초기값 없는 점화식은 정의되지 않습니다. 반드시 int sum = 0;

  


  

    
❌ 실수 2 — 경계(시작·끝) 착각: i < n vs i <= n

    
for (int i = 1; i < n; i++)   /* n-1까지만 더해짐! */

    
시그마의 위끝이 n이면 i <= n이어야 합니다. 아래끝이 0인지 1인지도 문제마다 반드시 확인하세요. 항의 개수는 (위끝 - 아래끝 + 1)개입니다.

  


  

    
❌ 실수 3 — 오버플로

    
int n = 100000;
int sum = n * (n + 1) / 2;    /* int * int 에서 이미 터짐 */

    
n = 100000이면 합이 약 50억으로 int 범위를 초과합니다. long long sum = (long long)n * (n + 1) / 2; 처럼 계산 전에 형을 올려야 합니다.

  


  

    
❌ 실수 4 — 상수의 시그마를 상수로 착각

    

      nΣi=13
       =  3n   (3이 아니다!)
    

    
상수 c를 n번 더하므로 결과는 c × n입니다. 반복문으로 생각하면 자명합니다: 매 회전마다 3이 더해집니다.

  


  

    
❌ 실수 5 — 곱까지 분배된다고 착각

    

      Σ(ai + bi) = Σai + Σbi  (✔ 덧셈은 분배됨)

      Σ(ai × bi) ≠ (Σai) × (Σbi)  (✘ 곱은 안 됨)
    

    
예: (1×1 + 2×2) = 5 이지만 (1+2)×(1+2) = 9. 시그마는 덧셈·상수배에만 선형입니다.
