다익스트라 — 가중치가 있을 때의 최단 거리

BFS는 모든 간선의 길이가 1로 같을 때만 최단 거리를 줍니다. 그런데 도로마다 거리가 다르다면? 간선에 가중치(weight)가 붙으면 BFS로는 안 됩니다. 이때 쓰는 것이 다익스트라 알고리즘이에요.

핵심 아이디어는 BFS와 놀랍도록 비슷합니다. 딱 하나만 바꿉니다 — 큐를 우선순위 큐(최소 힙)로. "다음에 볼 정점"을 먼저 온 순서가 아니라 가장 가까운 순서로 꺼내는 거죠.

탐색자료구조다음에 꺼내는 것거리
BFS큐 (FIFO)먼저 들어온 정점가중치 1만
다익스트라우선순위 큐dist가 가장 작은 정점임의 양의 가중치
💡 "그릇만 바꾼다"는 말이 진짜입니다. BFS 코드에서 큐를 최소 힙으로, 방문 표시를 거리 비교(완화)로 바꾸면 곧 다익스트라예요. 오늘 그 변화를 한 줄씩 봅니다.

이 강의의 예제 그래프

정점 6개, 간선 9개인 가중치 무방향 그래프입니다. 간선 위 숫자가 거리(가중치)예요.

INPUT (u v w = 정점 u–v 사이 거리 w)
6 9 1 2 7 1 3 9 1 6 14 2 3 10 2 4 15 3 4 11 3 6 2 4 5 6 5 6 9
⚠️ 주목할 점 — 1번에서 6번으로 직접 가는 간선은 거리 14입니다. 하지만 1→3→6으로 돌아가면 9+2 = 11로 더 짧아요! 다익스트라는 이런 "돌아가는 게 더 빠른" 경우를 정확히 찾아냅니다.

최소 힙 — 항상 가장 작은 것이 위로

우선순위 큐(최소 힙)는 무엇을 넣든 꺼낼 때 항상 가장 작은 값이 나오는 자료구조입니다. 다익스트라에서는 (거리, 정점) 쌍을 넣고, 거리가 가장 작은 쌍을 꺼냅니다.

힙에 든 (거리, 정점) 지금 꺼낼 최소값 낡은 항목 (무시됨)
C에는 표준 우선순위 큐가 없습니다. 직접 이진 힙을 배열로 구현하거나(3번 탭), 정점 수가 적으면 매번 최소를 선형 탐색하는 O(V²) 방식을 씁니다. C++은 priority_queue, 파이썬은 heapq가 있죠.

완화(Relaxation) — 다익스트라의 심장

다익스트라의 핵심 동작은 딱 하나, 완화입니다. "u를 거쳐서 v로 가는 게 지금까지 알던 것보다 짧으면, dist[v]를 줄인다."

완화 — 세 줄이 전부
/* u에서 v로 가는 간선의 가중치 w */
if (dist[u] + w < dist[v]) {
    dist[v] = dist[u] + w;   /* 더 짧으면 갱신 */
    push(v, dist[v]);        /* 힙에 새 후보 */
}
무슨 뜻인가
식의미
dist[u]시작점에서 u까지 확정된 최단 거리
wu에서 v로 가는 간선의 길이
dist[u]+wu를 거쳐서 v로 가는 거리
dist[v]지금까지 알던 v까지의 최단 거리
비교거쳐 가는 게 더 짧으면 갱신!
💡 "완화"라는 이름은 팽팽한 고무줄이 느슨해지듯 거리 추정값이 점점 작아지며 진짜 최단값에 가까워진다는 뜻입니다. 한 번 확정된 정점은 다시 완화되지 않아요.

왜 "가장 가까운 것부터" 꺼내야 하나

다익스트라의 정확성은 탐욕적 선택에 기댑니다. 아직 확정 안 된 정점 중 dist가 가장 작은 것을 꺼내면, 그 값은 이미 최단임이 보장됩니다. (양의 가중치라면 더 짧은 우회로가 있을 수 없으니까요.)

⚠️ 음의 가중치가 있으면 다익스트라는 틀립니다. "지금 가장 가까운 게 최단"이라는 보장이 깨지기 때문이죠(나중에 음수 간선으로 더 줄어들 수 있음). 음수 간선엔 벨만–포드를 씁니다.
낡은 항목(stale) 처리 — 한 정점이 여러 번 완화되면 힙에 옛 거리로 된 중복 항목이 남습니다. 꺼냈을 때 d > dist[u]이면(힙 속 거리가 지금 dist보다 크면) 이미 더 좋은 값으로 처리됐다는 뜻이라 그냥 건너뜁니다. 이 한 줄이 중복을 깔끔히 걸러 줍니다.

손으로 미리 따라가 보기

1번에서 시작. dist[1]=0, 나머지는 ∞(아직 모름). 힙에서 최소를 꺼내 이웃을 완화합니다.

완성 코드 — 이진 힙 직접 구현

C에는 우선순위 큐가 없으니 이진 최소 힙을 배열로 만듭니다. push는 아래에서 위로, pop은 위에서 아래로 정렬합니다.

C — 최소 힙
/* 힙에 (거리 d, 정점 v) 저장 */
int hd[MAXN*4], hv[MAXN*4], hs; /* size */

void push(int d, int v) {
    int i = ++hs;
    hd[i] = d; hv[i] = v;
    while (i > 1 && hd[i/2] > hd[i]) {
        /* 부모와 교환 (위로) */
        int td=hd[i],tv=hv[i];
        hd[i]=hd[i/2];hv[i]=hv[i/2];
        hd[i/2]=td;hv[i/2]=tv;
        i /= 2;
    }
}
void pop(int *od, int *ov) {
    *od = hd[1]; *ov = hv[1];
    hd[1]=hd[hs]; hv[1]=hv[hs]; hs--;
    int i = 1;
    while (1) {              /* 아래로 */
        int s = i, l = i*2, r = i*2+1;
        if (l<=hs && hd[l]<hd[s]) s=l;
        if (r<=hs && hd[r]<hd[s]) s=r;
        if (s==i) break;
        int td=hd[i],tv=hv[i];
        hd[i]=hd[s];hv[i]=hv[s];hd[s]=td;hv[s]=tv;
        i = s;
    }
}
C — 다익스트라 본체
int dist[MAXN];

void dijkstra(int s) {
    for (int i=1;i<=n;i++) dist[i]=INF;
    dist[s] = 0;
    hs = 0;
    push(0, s);
    while (hs > 0) {
        int d, u;
        pop(&d, &u);
        if (d > dist[u]) continue; /* stale */
        for (int k=0;k<deg[u];k++) {
            int v = adjv[u][k];
            int w = adjw[u][k];
            if (dist[u]+w < dist[v]) {
                dist[v] = dist[u]+w;
                push(dist[v], v);
            }
        }
    }
}
복잡도 — 힙 기반 다익스트라는 O((V+E) log V)입니다. 간선마다 힙에 push할 수 있고(log V), 모든 간선을 한 번씩 보니까요. 정점이 아주 적으면 O(V²) 선형 탐색이 오히려 간단하고 충분합니다.

Python — heapq로 간결하게

Python — 다익스트라
import heapq

def dijkstra(s, n, adj):
    # adj[u] = [(v, w), ...]
    dist = [float('inf')] * (n + 1)
    dist[s] = 0
    pq = [(0, s)]         # (거리, 정점)
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]:      # stale
            continue
        for v, w in adj[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(pq, (dist[v], v))
    return dist

BFS → 다익스트라, 무엇이 바뀌었나

💡 한 문장 요약 — 큐를 우선순위 큐로 바꾸고, "안 갔으면"을 "더 짧으면"으로 바꾸면 BFS가 다익스트라가 됩니다. 그래서 가중치가 모두 1이면 다익스트라 = BFS예요(모든 간선이 같으니 거리 순 = 먼저 온 순).
