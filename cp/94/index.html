벨만–포드 — 음수 간선도 다루는 최단 거리

다익스트라는 빠르지만 치명적 약점이 있습니다. 음수 간선이 있으면 틀린 답을 내죠. "지금 가장 가까운 게 최단"이라는 전제가 음수 앞에서 깨지기 때문입니다.

벨만–포드는 더 느리지만 음수 간선을 정확히 처리하고, 심지어 음수 사이클(무한히 줄어드는 순환)까지 탐지합니다. 방법은 놀랍도록 단순해요 — 모든 간선을 V−1번 반복해서 완화합니다.

다익스트라벨만–포드
음수 간선❌ 불가✅ 가능
음수 사이클탐지 못함✅ 탐지
방식우선순위 큐로 가까운 것부터모든 간선을 V−1번 완화
복잡도O((V+E)logV)O(V·E)
💡 다익스트라가 "똑똑하게 골라 한 번씩"이라면, 벨만–포드는 "단순무식하게 모두 여러 번"입니다. 대신 그 단순함이 음수까지 포용하는 힘이 됩니다.

이 강의의 예제 그래프

정점 5개, 간선 10개인 가중치 유방향 그래프입니다. 빨간 음수 간선이 섞여 있어요.

INPUT (u v w = u→v 방향, 가중치 w)
5 10 1 2 6 1 3 7 2 3 8 2 4 5 2 5 -4 3 4 -3 3 5 9 4 2 -2 5 1 2 5 4 7
⚠️ 2→5가 −4, 3→4가 −3, 4→2가 −2로 음수입니다. 다익스트라라면 여기서 틀린 답을 낼 수 있어요. 벨만–포드는 이 음수들을 정확히 반영합니다.

왜 V−1번인가?

최단 경로는 최대 V−1개의 간선으로 이루어집니다(정점 V개를 한 번씩만 지나면 간선은 V−1개). 한 라운드에 모든 간선을 완화하면 경로가 최소 1칸씩 뻗어나가므로, V−1라운드면 가장 긴 최단 경로까지 완성됩니다.

핵심 직관 — 라운드 1이 끝나면 "간선 1개로 갈 수 있는 최단"이 확정되고, 라운드 2면 "간선 2개까지"... 라운드 k면 "간선 k개까지의 최단"이 보장됩니다. 정점 V개짜리 최단 경로는 간선이 최대 V−1개이므로 V−1라운드로 충분합니다.
💡 만약 V−1라운드 이후에도 완화가 일어난다면? 그건 최단 경로가 V개 이상의 간선을 갖는다는 모순 → 음수 사이클이 존재한다는 신호입니다. 이것이 3번 탭의 주제예요.

반복 완화 — 모든 간선을 몇 번이고

다익스트라의 완화와 식은 똑같습니다. 차이는 "어떤 순서로, 몇 번" 하느냐죠. 벨만–포드는 모든 간선을 순서 상관없이 V−1번 반복합니다.

C — 벨만–포드 핵심
/* dist 초기화: 시작점 0, 나머지 INF */
for (int i = 1; i <= n; i++) dist[i] = INF;
dist[s] = 0;

/* V-1 라운드 */
for (int r = 1; r <= n - 1; r++) {
    /* 모든 간선을 완화 */
    for (int e = 0; e < m; e++) {
        int u = eu[e], v = ev[e], w = ew[e];
        if (dist[u] != INF &&
            dist[u] + w < dist[v]) {
            dist[v] = dist[u] + w;
        }
    }
}
다익스트라와 무엇이 다른가
항목차이
자료구조우선순위 큐 없음 — 그냥 간선 배열
순서간선을 아무 순서로나 다 훑음
횟수전체를 V−1번 반복
완화식dist[u]+w < dist[v](동일!)
확정"한 번 꺼내면 확정" 개념 없음
⚠️ dist[u] != INF 검사를 잊지 마세요. u에 아직 도달 못 했는데(INF) 완화하면, INF + w가 오버플로되거나 엉뚱한 값이 됩니다. "아직 못 간 정점에서는 완화하지 않는다"는 안전장치입니다.

라운드가 쌓이며 경로가 뻗어나간다

각 라운드는 최단 경로를 간선 1개씩 더 멀리 확정합니다. 예제에서 라운드별 dist 변화를 미리 봅니다.

라운드dist [1 2 3 4 5]일어난 일
초기0 ∞ ∞ ∞ ∞시작점만 0
10 2 7 4 2여러 간선 완화 (6건)
20 2 7 4 -22→5 재완화 (2가 줄어든 덕에 5도 −2로)
30 2 7 4 -2변화 없음 → 수렴 완료
💡 조기 종료 최적화 — 어떤 라운드에서 단 하나의 완화도 일어나지 않으면, 이미 수렴한 것이라 남은 라운드를 건너뛸 수 있습니다. changed 플래그 하나면 충분해요. 위 예제는 라운드 3에서 변화가 없어 조기 종료됩니다.

음수 사이클 — 끝없이 줄어드는 함정

간선 가중치의 합이 음수인 순환이 있으면, 그 사이클을 돌 때마다 거리가 계속 줄어듭니다. 최단 거리가 −∞로 발산해 정의되지 않죠. 벨만–포드는 이걸 탐지할 수 있습니다.

음수 사이클 예제: 2→3→4→2
2→3: -1 3→4: -1 4→2: -1 사이클 합 = -1 + -1 + -1 = -3 (음수!) 이 사이클을 한 바퀴 돌 때마다 거리가 3씩 줄어듦 → 최단 거리 없음 (−∞)
⚠️ 음수 간선은 괜찮지만 음수 사이클은 최단 거리를 정의 불가하게 만듭니다. 이 둘을 구분하세요. 예제의 메인 그래프는 음수 간선은 있어도 음수 사이클은 없습니다.

탐지 방법 — 한 라운드 더 돌려본다

V−1라운드면 음수 사이클이 없는 한 완전히 수렴합니다. 그러니 V번째 라운드를 한 번 더 돌렸을 때 여전히 완화가 일어난다면, 그건 음수 사이클이 있다는 확실한 증거입니다.

C — 음수 사이클 탐지
/* V-1 라운드 완화 후... */

int has_neg_cycle = 0;
/* V번째 라운드: 한 번 더 */
for (int e = 0; e < m; e++) {
    int u = eu[e], v = ev[e], w = ew[e];
    if (dist[u] != INF &&
        dist[u] + w < dist[v]) {
        has_neg_cycle = 1;  /* 또 줄어듦! */
        break;
    }
}
if (has_neg_cycle)
    printf("음수 사이클 존재\n");
왜 이게 통하나
// V-1 라운드 = 최장 최단경로 완성
// (경로는 최대 V-1개 간선)

// 음수 사이클이 없다면:
//   V번째 라운드엔 변화 없음
//   (이미 수렴 완료)

// 음수 사이클이 있다면:
//   돌 때마다 계속 줄어듦
//   → V번째에도 완화 발생!

// 이 차이가 탐지의 원리

네 알고리즘 한눈에

DFS·BFS·다익스트라·벨만–포드는 모두 그래프 탐색의 형제입니다. 완화라는 관점에서 정리하면 이렇습니다.

BFS다익스트라벨만–포드
그릇큐우선순위 큐간선 배열
완화 순서거리 순가장 가까운 것모든 간선 반복
음수 간선❌❌✅
음수 사이클❌❌✅ 탐지
가중치1만양수만임의(음수 OK)
복잡도O(V+E)O((V+E)logV)O(V·E)
💡 선택 기준 — 가중치 없으면 BFS, 양의 가중치면 다익스트라(빠름), 음수 간선이 있거나 음수 사이클을 탐지해야 하면 벨만–포드. 속도(다익스트라)와 포용력(벨만–포드)의 맞바꿈입니다.

다익스트라가 음수에서 틀리는 이유

작은 예로 봅시다. 1→2가 5, 1→3이 4, 3→2가 −3인 그래프. 진짜 최단은 1→3→2 = 4+(−3) = 1입니다.

다익스트라 (틀림)
// 1을 확정, dist=[0,5,4]
// 가장 가까운 3(거리4) 확정
// 그다음 2(거리5) 확정 ← 여기서!
// 2를 이미 확정해버려서
// 3→2(-3)로 1이 되는 걸 놓침

// 결과: dist[2] = 5  ❌
// (진짜는 1)
벨만–포드 (맞음)
// 확정 개념이 없음
// 라운드1: 1→2로 dist[2]=5
//          1→3으로 dist[3]=4
// 라운드2: 3→2(-3)로
//          dist[2] = 4-3 = 1 ✓

// 결과: dist[2] = 1  ✅
// 반복 완화가 음수를 포착
본질 — 다익스트라는 "한 번 확정하면 끝"이라 나중에 음수로 더 줄어드는 걸 못 봅니다. 벨만–포드는 확정 없이 몇 번이고 다시 완화하므로 음수의 뒤늦은 개선을 놓치지 않습니다.

SPFA — 벨만–포드의 큐 최적화

벨만–포드는 매 라운드 모든 간선을 훑지만, 실제로는 dist가 바뀐 정점에서 나가는 간선만 다시 볼 가치가 있습니다. 바뀐 정점을 큐에 넣어 관리하는 것이 SPFA(Shortest Path Faster Algorithm)입니다.

SPFA 요약 — BFS처럼 큐를 쓰되, "완화에 성공한 정점"만 큐에 넣습니다. 평균적으로 빠르지만 최악은 여전히 O(V·E)이고, 특정 그래프에선 느려질 수 있어 대회에선 주의가 필요합니다. 개념은 "변화가 전파되는 곳만 따라간다"입니다.
💡 다음 단계 — 모든 쌍 최단 거리는 플로이드–워셜 O(V³), 음수 간선이 있는 모든 쌍은 존슨 알고리즘(벨만–포드로 재가중 후 다익스트라)을 씁니다. 벨만–포드는 이 고급 기법들의 부품이 됩니다.

내 그래프로 벨만–포드 돌려보기

유방향 가중치 간선을 u v w 형식으로 입력하세요(음수 허용). 최단 거리와 음수 사이클 여부, 각 정점까지의 경로를 보여줍니다.

그래프 입력

간선 목록 (한 줄에 "u v w" — u→v 방향, w는 음수 가능)
입력을 넣고 [벨만–포드 실행]을 눌러 보세요.
💡 실험: 4 2 -2를 4 2 -20으로 바꿔 보세요. 2→4→2 순환의 합이 음수가 되어(5+(−20)=−15) 음수 사이클이 탐지됩니다!
