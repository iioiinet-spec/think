최소 신장 트리 — 가장 싸게 모두 잇기

신장 트리(Spanning Tree)는 그래프의 모든 정점을 연결하면서 사이클이 없는 부분 그래프입니다. 정점이 V개면 간선은 정확히 V-1개죠. 그중 간선 가중치의 합이 최소인 것이 최소 신장 트리(MST)입니다.

도시들을 최소 비용으로 도로·전선·통신망으로 잇는 문제가 바로 MST예요. 두 대표 알고리즘 프림과 크루스칼은 전혀 다른 방식으로 접근하지만 같은 최소 비용에 이릅니다.

조건의미
모든 정점 연결어느 정점에서든 다른 모든 정점에 도달 가능
사이클 없음간선 V-1개 — 하나라도 더하면 사이클
가중치 합 최소연결 방법 중 가장 저렴
💡 최단 경로(다익스트라 등)가 "한 점에서 다른 점까지 빠른 길"이라면, MST는 "모두를 잇는 가장 싼 뼈대"입니다. 목적이 완전히 달라요.

두 가지 접근 — 정점 확장 vs 간선 정렬

🌱 프림 (Prim)

한 정점에서 시작해 나무를 키웁니다. 매번 나무에 붙일 수 있는 가장 싼 간선으로 새 정점을 흡수. 다익스트라와 형제예요.

🔗 크루스칼 (Kruskal)

모든 간선을 가중치 순으로 정렬하고, 싼 것부터 사이클이 안 생기면 채택. 유니온-파인드로 사이클을 판별해요.

둘 다 탐욕(greedy) 알고리즘 — "지금 가장 싼 선택"을 반복합니다. 놀랍게도 이 단순한 탐욕이 MST에서는 항상 최적을 보장해요. 프림은 정점 중심, 크루스칼은 간선 중심이라는 관점 차이가 전부입니다.

이 강의의 예제 그래프

정점 6개, 간선 9개인 가중치 무방향 그래프입니다.

INPUT (u v w = u-v 사이 가중치 w)
6 9 1 2 4 1 3 3 2 3 1 2 4 2 3 4 4 3 5 5 4 5 7 4 6 6 5 6 8
정답 MST (프림·크루스칼 동일)
채택 간선: 2-3(1) 2-4(2) 1-3(3) 3-5(5) 4-6(6) 총 가중치 = 1+2+3+5+6 = 17

프림 — 나무를 키워 나가기

프림은 정점 하나에서 시작해 나무를 키웁니다. 매 단계마다 "나무에 아직 없는 정점 중, 나무에 가장 싸게 붙일 수 있는 정점"을 흡수해요. 이 "가장 싼 연결 비용"을 key[] 배열에 담습니다.

key[]의 의미
배열뜻
key[v]v를 나무에 붙이는 가장 싼 간선의 비용
inMST[v]v가 이미 나무에 있는가
parent[v]v를 나무에 붙인 상대 정점
다익스트라와 닮은 점
// 다익스트라: dist[v]
//   = 시작점부터 v까지 최단

// 프림: key[v]
//   = 나무부터 v까지 한 간선

// 둘 다 "가장 작은 값" 정점을
// 우선순위 큐로 꺼내 확장!
// 갱신식만 다르다:
//  다익: dist[u]+w < dist[v]
//  프림:        w < key[v]
💡 프림과 다익스트라는 뼈대가 같습니다. 다익스트라가 "시작점부터의 누적 거리"를 최소화한다면, 프림은 "나무까지의 한 간선" 비용만 봅니다. dist[u]+w가 w로 바뀐 것뿐이에요.

한 줄씩 실행 — 코드 · key[] 배열 · 그래프

[다음 ▶]마다 실행 코드 줄이 빛나고, key[] 배열이 갱신되며, 나무에 정점이 흡수됩니다. key가 가장 작은 정점이 노란색으로 뽑혀요.

실행 중인 코드
key[] 배열  (∞=미연결, 노랑=다음에 뽑을 최소, 초록=나무에 포함)
parent[]  (각 정점을 붙인 상대)
뽑은 정점 0 / 6
MST 누적 0
시작
[다음 ▶]을 눌러 프림 알고리즘을 시작하세요.
나무에 포함 (inMST) 지금 뽑는 최소 key 후보 (key 갱신됨)

완성 코드 — 프림

C — 프림 (O(V²))
int key[MAXN], inMST[MAXN], parent[MAXN];
for (int i=1;i<=n;i++){
    key[i]=INF; inMST[i]=0; parent[i]=0;
}
key[1] = 0;          /* 시작 정점 */
int total = 0;

for (int c=0; c<n; c++) {
    /* 나무 밖에서 key 최소 정점 u */
    int u=-1, best=INF;
    for (int i=1;i<=n;i++)
        if (!inMST[i] && key[i]<best)
            { best=key[i]; u=i; }
    inMST[u]=1; total+=key[u];
    /* u의 이웃 key 갱신 */
    for (int k=0;k<deg[u];k++){
        int v=adjv[u][k], w=adjw[u][k];
        if (!inMST[v] && w<key[v])
            { key[v]=w; parent[v]=u; }
    }
}
우선순위 큐 버전 (O(E log V))
// 힙에 (key, 정점) 저장
// 다익스트라와 거의 동일

pq.push({0, 1});
while (!pq.empty()) {
    auto [k,u] = pq.top(); pq.pop();
    if (inMST[u]) continue;
    inMST[u]=1; total+=k;
    for (auto [v,w] : adj[u])
        if (!inMST[v] && w<key[v]) {
            key[v]=w;
            pq.push({w, v});
        }
}
// 정점 적으면 O(V²)가 더 간단
복잡도 — 선형 최소 탐색은 O(V²), 우선순위 큐는 O(E log V)입니다. 조밀한 그래프(간선 많음)면 O(V²)가 오히려 유리하고, 희소한 그래프면 힙 버전이 빠릅니다.

크루스칼 — 싼 간선부터 줍기

크루스칼은 정점이 아니라 간선에 주목합니다. 모든 간선을 가중치 오름차순으로 정렬한 뒤, 싼 것부터 하나씩 보며 사이클을 만들지 않으면 채택합니다. 간선 V-1개를 채택하면 끝이에요.

크루스칼의 두 단계
// 1) 간선을 가중치순 정렬
//    2-3:1, 2-4:2, 1-3:3,
//    1-2:4, 3-4:4, 3-5:5, ...

// 2) 싼 것부터:
//    사이클 안 생기면 채택
//    사이클 생기면 버림

// "사이클?"을 어떻게 아나?
//   → 유니온-파인드!
사이클 판별 = 같은 집합?
// 두 정점이 이미 연결돼 있으면
// (= 같은 집합에 속하면)
// 그 사이 간선은 사이클!

find(u) == find(v)
  → 같은 집합 → 사이클 → 버림

find(u) != find(v)
  → 다른 집합 → 채택 + 합침
       union(u, v)
💡 크루스칼의 핵심은 "이 간선을 넣으면 사이클이 생기나?"를 빠르게 판별하는 것입니다. 그 도구가 유니온-파인드(서로소 집합)예요. 3번 탭에서 자세히 봅니다.

완성 코드 — 크루스칼

C — 크루스칼
int par[MAXN];
int find(int x){
    while (par[x]!=x){
        par[x]=par[par[x]]; /* 경로 압축 */
        x=par[x];
    }
    return x;
}
/* 간선 가중치순 정렬 후 */
qsort(E, m, sizeof(E[0]), ecmp);
for (int i=1;i<=n;i++) par[i]=i;

int total=0, cnt=0;
for (int e=0; e<m && cnt<n-1; e++){
    int a=find(E[e].u), b=find(E[e].v);
    if (a != b) {          /* 다른 집합 */
        par[a] = b;        /* union */
        total += E[e].w;
        cnt++;             /* 채택! */
    }
}
실행 흐름
// 정렬: 2-3:1 2-4:2 1-3:3
//       1-2:4 3-4:4 3-5:5 ...

2-3:1  find(2)≠find(3) → 채택 ✓
2-4:2  find(2)≠find(4) → 채택 ✓
1-3:3  find(1)≠find(3) → 채택 ✓
1-2:4  find(1)=find(2) → 버림 ✗
3-4:4  find(3)=find(4) → 버림 ✗
3-5:5  find(3)≠find(5) → 채택 ✓
4-6:6  find(4)≠find(6) → 채택 ✓
// 간선 5개 채택 → 종료
// 총합 = 1+2+3+5+6 = 17
복잡도 — 정렬이 O(E log E)로 지배적입니다. 유니온-파인드 연산은 거의 상수 시간(역아커만 함수)이라, 전체는 O(E log E)예요. 희소 그래프에서 특히 효율적입니다.

유니온-파인드 — 서로소 집합

유니온-파인드(Union-Find) 또는 서로소 집합(Disjoint Set)은 "이 둘이 같은 그룹인가?"를 아주 빠르게 답하는 자료구조입니다. 크루스칼의 사이클 판별에 딱 맞아요.

연산하는 일
find(x)x가 속한 집합의 대표(루트)를 찾음
union(a,b)a와 b의 두 집합을 하나로 합침
같은 집합?find(a) == find(b)
핵심 아이디어 — 각 원소는 부모를 가리키고, 부모를 계속 따라가면 루트가 나옵니다. 같은 루트를 가지면 같은 집합이에요. par[] 배열 하나로 숲(트리들)을 표현합니다.

두 가지 최적화

경로 압축 (Path Compression)
int find(int x){
    while (par[x]!=x){
        par[x]=par[par[x]]; /* 손자→조부모 */
        x=par[x];
    }
    return x;
}
// find 하는 김에 트리를 납작하게
// 다음 find가 훨씬 빨라짐
랭크(크기) 기준 합치기
void unite(int a, int b){
    a=find(a); b=find(b);
    if (a==b) return;
    if (sz[a]<sz[b]){ int t=a;a=b;b=t; }
    par[b]=a;          /* 작은 걸 큰 쪽에 */
    sz[a]+=sz[b];
}
// 트리가 한쪽으로 길어지는 것 방지
거의 상수 시간 — 경로 압축과 랭크 최적화를 함께 쓰면 각 연산이 평균 O(α(N))(역아커만 함수)입니다. 실질적으로 4 이하의 상수라 사실상 O(1)이에요. 그래서 크루스칼의 병목은 유니온-파인드가 아니라 정렬입니다.

프림 vs 크루스칼 — 한눈에

같은 MST에 이르는 두 길입니다. 관점과 자료구조, 그리고 언제 무엇이 유리한지 정리합니다.

사이클 방지나무 밖 정점만 선택find로 판별

💡 선택 기준 — 간선이 많은(조밀한) 그래프면 프림, 간선이 적은(희소한) 그래프면 크루스칼이 대체로 유리합니다. 하지만 둘 다 탐욕 알고리즘이고 결과 MST의 총 가중치는 항상 같아요(트리 모양은 다를 수 있음).

MST의 활용과 성질

주제내용
통신망 설계최소 비용으로 모든 지점 연결 (전선·광케이블)
클러스터링MST에서 비싼 간선을 끊어 그룹 분할
근사 알고리즘외판원 문제(TSP)의 2배 근사에 MST 사용
유일성모든 간선 가중치가 다르면 MST는 유일
사이클 성질사이클에서 가장 무거운 간선은 MST에 없음
컷 성질어떤 컷을 가로지르는 최소 간선은 MST에 있음
💡 컷 성질(Cut Property)이 프림·크루스칼 둘 다의 정확성을 보장합니다. "정점을 두 그룹으로 나눴을 때, 그 사이를 잇는 가장 싼 간선은 반드시 MST에 포함된다"는 정리예요. 프림은 나무 안/밖으로 컷을 만들고, 크루스칼은 매 간선이 두 조각을 잇는 최소 간선이죠.
