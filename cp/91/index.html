큐 배열의 head·tail 포인터와 dist 배열이 실제로 어떻게 움직이는지 코드와 나란히

C 시각화: 큐 배열 · dist[] · 코드 라인

BFS — 가까운 곳부터 물결처럼 퍼지는 탐색

너비 우선 탐색(Breadth-First Search)은 시작점에서 가까운 순서대로 훑습니다. 연못에 돌을 던지면 물결이 동심원으로 퍼지듯, 거리 1인 정점을 모두 본 뒤 거리 2, 그다음 거리 3으로 나아갑니다.

이 강의는 그 과정에서 큐(queue)라는 배열의 head·tail 포인터가 어떻게 움직이고, dist 배열이 어떻게 층층이 채워지는지를 코드와 나란히 보여줍니다.

추적할 대상역할
q[] + head, tail다음에 방문할 정점을 먼저 온 순서대로 담는 큐
visited[]각 정점을 방문했는지 0/1로 기록
dist[]시작점으로부터의 최단 거리 — BFS의 선물
💡 DFS가 스택(나중에 온 것 먼저)이라면, BFS는 큐(먼저 온 것 먼저)입니다. 이 한 끗 차이가 "깊이 우선"과 "너비 우선"을 가릅니다.

이 강의의 예제 그래프

정점 6개, 간선 7개인 무방향 그래프입니다. (DFS 편과 같은 그래프라 두 탐색의 차이를 바로 비교할 수 있습니다.)

INPUT
6 7 정점 6, 간선 7 1 2 1 3 2 4 2 5 3 5 4 6 5 6

각 정점의 이웃은 번호가 작은 순서로 큐에 넣습니다(인접 리스트 정렬).

손으로 미리 따라가 보기

1번에서 시작합니다. 규칙 — "큐에서 하나 꺼내 그 이웃들을 모두 큐 뒤에 넣는다. 큐가 빌 때까지."

큐 상태꺼낸 정점새로 넣는 이웃
[1]12, 3 (거리 1)
[2, 3]24, 5 (거리 2)
[3, 4, 5]3없음 (1,5 방문됨)
[4, 5]46 (거리 3)
[5, 6]5없음
[6]6없음 → 큐 빔, 종료
최종 방문 순서와 거리
BFS(1) = 1 → 2 → 3 → 4 → 5 → 6 dist = 0 1 1 2 2 3
⚠️ DFS와 비교하세요. 같은 그래프인데 DFS는 1 2 4 6 5 3(한 길로 깊이), BFS는 1 2 3 4 5 6(거리 순)입니다. BFS는 번호가 아니라 거리 순으로 나오는 게 핵심이에요(여기선 우연히 번호와 일치).

BFS 코드 — 큐를 배열로 만들기

C에서는 큐를 배열 하나와 두 정수로 만듭니다. head(꺼낼 위치)와 tail(넣을 위치)이 오른쪽으로만 이동하죠.

C — 배열 큐 BFS
int visited[105], q[105];

void bfs(int s) {
    int head = 0, tail = 0;
    visited[s] = 1;          /* ① 시작점 방문 */
    q[tail++] = s;          /* ② 큐에 넣기 */
    while (head < tail) {      /* 큐가 안 비면 */
        int u = q[head++];     /* ③ 앞에서 꺼내기 */
        printf("%d ", u);
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k];
            if (!visited[v]) {   /* ④ 안 갔으면 */
                visited[v] = 1;   /* ⑤ 넣을 때 표시! */
                q[tail++] = v;   /* ⑥ 큐 뒤에 넣기 */
            }
        }
    }
}
각 줄의 의미
줄하는 일
①②시작점을 방문 표시하고 큐에 넣음 (tail이 한 칸 전진)
whilehead < tail — 아직 꺼낼 게 남았으면 반복
③q[head++] — 큐 앞에서 하나 꺼냄 (head 전진)
④이웃이 아직 방문 안 됐는지 확인
⑤안 됐으면 지금 바로 방문 표시 — 꺼낼 때가 아니라 넣을 때!
⑥q[tail++] — 큐 뒤에 넣음 (tail 전진)
⚠️ ⑤의 시점이 생명 — 방문 표시를 큐에 넣는 순간(pop이 아니라 push할 때)에 해야 합니다. 만약 꺼낼 때 표시하면, 같은 정점이 여러 이웃을 통해 큐에 중복으로 들어가 시간이 낭비되고 dist가 틀립니다. "넣을 때 도장 찍기"가 BFS의 철칙입니다.

head와 tail — 배열 위를 걷는 두 포인터

큐를 실제 원형 버퍼로 만들 수도 있지만, 정점을 한 번씩만 넣는 BFS에서는 tail이 최대 n까지만 가므로 그냥 큰 배열이면 충분합니다. 두 포인터는 오른쪽으로만 이동해요.

큐에 있음 (대기 중) 이미 꺼냄 (head 지나감) 방금 넣음
head < tail이 곧 "큐가 비어있지 않다" — head가 tail을 따라잡으면(같아지면) 꺼낼 게 없다는 뜻이라 반복이 끝납니다. 두 포인터 사이의 칸들이 현재 큐에 든 정점들입니다.

💡 핵심 관찰 — 큐에 정점들이 거리 순으로 줄 서 있습니다. 거리 1인 {2,3}이 먼저 다 처리된 뒤에야 거리 2인 {4,5}가 처리돼요. head와 tail 사이 칸이 "지금 큐에 대기 중인 정점"이고, 그 정점들의 dist는 항상 연속된 두 값(예: 1과 2)뿐입니다. 이것이 BFS가 최단 거리를 보장하는 이유입니다.

BFS의 선물 — 최단 거리가 공짜로

모든 간선의 길이가 같다면(가중치 1), BFS가 정점을 방문하는 순서가 곧 거리 순서입니다. 그래서 dist[v] = dist[u] + 1 한 줄만 추가하면 시작점에서 모든 정점까지의 최단 거리를 얻습니다.

C — 최단 거리 BFS
int dist[MAXN], q[MAXN];

void bfs(int s) {
    for (int i = 1; i <= n; i++)
        dist[i] = -1;      /* -1 = 미방문 */
    dist[s] = 0;          /* 시작점 거리 0 */
    int head = 0, tail = 0;
    q[tail++] = s;
    while (head < tail) {
        int u = q[head++];
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k];
            if (dist[v] == -1) {  /* 미방문이면 */
                dist[v] = dist[u] + 1;
                q[tail++] = v;
            }
        }
    }
}

파이썬에서는 deque를 써라

C는 배열 큐가 자연스럽지만, 파이썬은 list.pop(0)이 O(n)이라 느립니다. 반드시 collections.deque를 쓰세요.

Python — 올바른 방법
from collections import deque

def bfs(s):
    dist = [-1] * (n + 1)
    dist[s] = 0
    q = deque([s])
    while q:
        u = q.popleft()   # O(1)!
        for v in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                q.append(v)
    return dist

BFS와 DFS — 그릇만 바꾸면 된다

놀랍게도 BFS와 DFS는 거의 같은 코드입니다. "다음에 어디를 볼까"를 담는 그릇이 큐냐 스택이냐만 다를 뿐이에요.

BFS — 큐 (먼저 온 것 먼저)
int head = 0, tail = 0;
q[tail++] = s;
while (head < tail) {
    int u = q[head++];  /* 앞에서 (FIFO) */
    for (...) {
        if (!visited[v]) {
            visited[v] = 1;
            q[tail++] = v;
        }
    }
}
/* → 너비 우선, 최단 거리 */
DFS — 스택 (나중 온 것 먼저)
int top = 0;
st[top++] = s;
while (top > 0) {
    int u = st[--top];  /* 뒤에서 (LIFO) */
    if (visited[u]) continue;
    visited[u] = 1;
    for (...) {
        if (!visited[v]) st[top++] = v;
    }
}
/* → 깊이 우선, 백트래킹 */
💡 큐의 q[head++](앞에서 꺼냄)를 스택의 st[--top](뒤에서 꺼냄)로 바꾸는 것, 이 한 곳이 전부입니다. 그릇을 우선순위 큐로 바꾸면? 그게 다익스트라예요!
