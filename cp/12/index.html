전부 연결하고 싶다, 최대한 싸게 — MST

8개의 마을에 전기를 공급하려 합니다. 전선을 깔 수 있는 후보 구간과 공사비가 주어질 때, 모든 마을에 전기가 통하면서 총 공사비가 최소가 되게 하려면 어떤 구간을 골라야 할까요?

이것이 최소 신장 트리(MST, Minimum Spanning Tree) 문제입니다. 이름을 분해하면:

단어뜻
신장(Spanning)모든 정점을 빠짐없이 포함한다
트리(Tree)사이클이 없다 — 그래서 간선은 정확히 V−1개 (8개 정점이면 7개)
최소(Minimum)가능한 신장 트리 중 가중치 합이 가장 작은 것
💡 왜 사이클이 없어야 할까? 사이클이 있다면 그중 간선 하나를 끊어도 여전히 전부 연결되어 있습니다. 즉 사이클의 간선 하나는 돈 낭비 — 최소 비용 연결망은 반드시 트리 모양이 됩니다.

다익스트라와 무엇이 다를까?

⑫편 다익스트라와 헷갈리기 쉽지만, 묻는 질문 자체가 다릅니다.

현실 예내비게이션, 지하철 환승 안내전력망, 통신 케이블, 수도관 설계

⚠️ 결정적 증거 — 같은 그래프, 다른 트리! 아래 실험 그래프에서 ⑫편 다익스트라(1번 출발)가 만든 트리는 4번 정점을 2—4(비용 5)로 연결했고 총합이 22였습니다. 하지만 MST는 4번을 4—5(비용 3)로 연결해 총합 20을 만듭니다. "1번에서 4번까지 빨리 가는 것"과 "4번을 싸게 연결하는 것"은 다른 문제이기 때문이죠!

두 명의 주인공 — 프림과 크루스칼

MST를 구하는 유명한 방법이 둘 있습니다. 놀랍게도 둘 다 그리디(greedy, 탐욕) 전략 — "매 순간 가장 싼 것을 고른다"만으로 항상 정답이 나옵니다.

비교프림 (Prim, 1탭)크루스칼 (Kruskal, 2탭)
한 줄 요약정점 하나에서 시작해 트리를 키운다싼 간선부터 골라 담는다
매 순간의 선택트리 안↔밖을 잇는 최소 간선남은 간선 중 최소 — 단 사이클 금지
핵심 도구우선순위 큐 (다익스트라의 친척!)정렬 + Union-Find (이산수학 편의 재등장!)
닮은꼴물감이 번지듯 한 덩어리가 커짐여러 조각이 생겼다가 합쳐짐

실험용 가중 그래프 — ⑫편과 동일!

프림 — 트리 한 그루를 키우기

아무 정점 하나에서 시작해, 나무가 가지를 뻗듯 정점을 하나씩 영입합니다. 규칙은 세 줄:

순서규칙
①트리 안과 밖을 잇는 간선들을 후보로 모은다
②후보 중 가중치가 가장 작은 간선을 채택한다
③그 간선의 바깥쪽 정점이 트리에 합류 — 정점이 V개가 될 때까지 반복
🔗 ⑫편 연결 — 어디서 본 것 같지 않나요? 다익스트라와 구조가 똑같습니다. 딱 하나, 비교 기준만 다릅니다: 다익스트라는 dist[u] + w(출발점부터의 누적 거리)로, 프림은 그냥 w(간선 자체의 값)로 최소를 고릅니다. 그래서 코드도 한 글자 차이!
💡 왜 이 그리디가 항상 정답일까? 트리 안/밖을 가르는 경계를 생각하면, 경계를 건너는 간선 중 최소인 것은 어떤 MST에든 넣어도 손해가 없음이 증명되어 있습니다 (컷 속성, cut property). "지금 제일 싼 다리를 건너는 게 늘 옳다"는 드문 행운의 문제죠.

크루스칼 — 싼 간선부터 골라 담기

프림이 한 그루 나무를 키운다면, 크루스칼은 간선 시장에서 쇼핑을 합니다:

순서규칙
①모든 간선을 가중치 오름차순으로 정렬한다
②싼 것부터 하나씩 검사 — 사이클이 생기면 버리고, 아니면 담는다
③담은 간선이 V−1개가 되면 끝!

문제는 ②입니다. "이 간선을 담으면 사이클이 생기나?"를 매번 빠르게 판정해야 하는데… 그 답이 바로 Union-Find입니다.

🔗 이산수학 편 연결 — Union-Find(서로소 집합)를 배웠던 것 기억하나요? "두 원소가 같은 그룹인가?"를 거의 상수 시간에 답하는 그 자료구조가 여기서 주인공으로 재등장합니다!

Union-Find 3분 복습

핵심 아이디어: 각 그룹을 뿌리(대표) 정점으로 식별합니다. find(x)는 x가 속한 그룹의 뿌리를 찾고, union(a, b)는 두 그룹을 합칩니다.

C — find (경로 압축) + union
int par[101];   /* 처음엔 par[i] = i (자기가 뿌리) */

int find(int x) {
    if (par[x] == x) return x;
    return par[x] = find(par[x]);  /* 경로 압축! */
}

/* 사이클 판정 + 합치기 */
int a = find(u), b = find(v);
if (a == b) { /* 같은 그룹 → 사이클! 버림 */ }
else        { par[a] = b; /* 합치기 */ }
Python — find (경로 압축) + union
par = list(range(n + 1))  # 처음엔 par[i] = i

def find(x):
    while par[x] != x:
        par[x] = par[par[x]]   # 경로 압축!
        x = par[x]
    return x

# 사이클 판정 + 합치기
a, b = find(u), find(v)
if a == b:
    pass       # 같은 그룹 → 사이클! 버림
else:
    par[a] = b # 합치기
💡 사이클 판정의 원리 — 간선 u—v를 담기 전에 u와 v가 이미 같은 그룹이라면? 이미 다른 길로 이어져 있다는 뜻이므로, 이 간선까지 담으면 사이클이 생깁니다. 그래서 조건은 단 한 줄: find(u) != find(v)일 때만 담는다 — 헤더의 수식이 바로 이것!

프림 구현 — 다익스트라에서 한 글자만 바꾸기

⑫편의 기본형 다익스트라와 나란히 비교해 보세요. dist가 key로, dist[u] + w가 w로 바뀐 것이 사실상 전부입니다.

C — 프림 O(V²)
#include <stdio.h>
#define INF 1000000000

int n, m;
int adjv[101][101], adjw[101][101], deg[101];
int key[101], inT[101];   /* key[v]: 트리와 v를 잇는 최솟값 */

int main(void) {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d %d %d", &u, &v, &w);
        adjv[u][deg[u]] = v; adjw[u][deg[u]] = w; deg[u]++;
        adjv[v][deg[v]] = u; adjw[v][deg[v]] = w; deg[v]++;
    }
    for (int i = 1; i <= n; i++) key[i] = INF;
    key[1] = 0;                     /* 1번에서 시작 */
    long long total = 0;

    for (int t = 0; t < n; t++) {
        int u = 0;                  /* ① 트리 밖 최소 key */
        for (int i = 1; i <= n; i++)
            if (!inT[i] && (u == 0 || key[i] < key[u])) u = i;
        if (u == 0 || key[u] == INF) break;
        inT[u] = 1;                 /* ② 트리에 합류 */
        total += key[u];
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k], w = adjw[u][k];
            if (!inT[v] && w < key[v])   /* ③ w 하나로 비교! */
                key[v] = w;
        }
    }
    printf("MST 가중치 합 = %lld\n", total);
    return 0;
}
Python — 프림 O(V²)
import sys
input = sys.stdin.readline
INF = float('inf')

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v, w = map(int, input().split())
    adj[u].append((v, w))
    adj[v].append((u, w))

key = [INF] * (n + 1)   # 트리와 v를 잇는 최솟값
inT = [False] * (n + 1)
key[1] = 0               # 1번에서 시작
total = 0

for _ in range(n):
    u, best = 0, INF     # ① 트리 밖 최소 key
    for i in range(1, n + 1):
        if not inT[i] and key[i] < best:
            u, best = i, key[i]
    if u == 0:
        break
    inT[u] = True        # ② 트리에 합류
    total += key[u]
    for v, w in adj[u]:
        if not inT[v] and w < key[v]:  # ③ w 하나로!
            key[v] = w

print('MST 가중치 합 =', total)
💡 힙 버전이 필요하다면? ⑫편의 heapq 다익스트라에서 nd = d + w를 nd = w로 바꾸고 방문 체크만 유지하면 그대로 프림 O(E log V)가 됩니다. 직접 고쳐 보는 것을 이번 주 숙제로!

크루스칼 구현 — 정렬 + Union-Find

C — 크루스칼 O(E log E)
#include <stdio.h>
#include <stdlib.h>

int n, m;
int eu[10001], ev[10001], ew[10001], idx[10001];
int par[101];

int find(int x) {
    if (par[x] == x) return x;
    return par[x] = find(par[x]);
}
int cmp(const void *a, const void *b) {
    return ew[*(int*)a] - ew[*(int*)b];  /* 가중치 기준 */
}

int main(void) {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        scanf("%d %d %d", &eu[i], &ev[i], &ew[i]);
        idx[i] = i;
    }
    qsort(idx, m, sizeof(int), cmp);   /* ① 정렬 */
    for (int i = 1; i <= n; i++) par[i] = i;

    long long total = 0;
    int cnt = 0;
    for (int i = 0; i < m; i++) {      /* ② 싼 것부터 */
        int e = idx[i];
        int a = find(eu[e]), b = find(ev[e]);
        if (a == b) continue;          /* 사이클! 버림 */
        par[a] = b;                    /* 그룹 합치기 */
        total += ew[e];
        cnt++;
        if (cnt == n - 1) break;       /* ③ V-1개면 끝 */
    }
    printf("MST 가중치 합 = %lld\n", total);
    return 0;
}
Python — 크루스칼 O(E log E)
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
edges = []
for _ in range(m):
    u, v, w = map(int, input().split())
    edges.append((w, u, v))   # 가중치를 맨 앞에!
edges.sort()                  # ① 정렬

par = list(range(n + 1))
def find(x):
    while par[x] != x:
        par[x] = par[par[x]]
        x = par[x]
    return x

total = 0
cnt = 0
for w, u, v in edges:         # ② 싼 것부터
    a, b = find(u), find(v)
    if a == b:                # 사이클! 버림
        continue
    par[a] = b                # 그룹 합치기
    total += w
    cnt += 1
    if cnt == n - 1:          # ③ V-1개면 끝
        break

print('MST 가중치 합 =', total)
OUTPUT — 실험 그래프 (프림·크루스칼 모두 동일)
MST 가중치 합 = 20
⚠️ 정렬 기준 실수 주의! Python에서 튜플을 (u, v, w) 순서로 넣고 sort하면 정점 번호순으로 정렬됩니다. 반드시 (w, u, v) — 가중치를 맨 앞에! C의 qsort도 비교 함수가 ew를 보고 있는지 확인하세요. 크루스칼 오답의 절반은 정렬 기준 실수입니다.
비교프림 O(V²)프림 O(E log V)크루스칼 O(E log E)
유리한 그래프간선이 빽빽한 밀집 그래프대부분의 실전 문제간선이 듬성한 희소 그래프
준비물배열만우선순위 큐정렬 + Union-Find
구현 체감가장 단순다익스트라 경험 있으면 쉬움Union-Find만 알면 짧고 깔끔
💡 음수 가중치도 OK! 다익스트라와 달리 MST 알고리즘은 음수 간선이 있어도 정확합니다. "제일 싼 것부터"라는 논리에 누적 거리가 등장하지 않기 때문이죠. 이런 차이를 묻는 서술형 문제가 종종 나옵니다.
