KMP 문자열 검색

실패 함수로 되돌아가지 않는다 — 텍스트를 한 번만 훑어 O(N+M)

C 시각화: 접두사=접미사 · pi 배열 · 패턴 슬라이딩 · 점프

단순 검색의 낭비 — 왜 처음부터 다시?

텍스트 T에서 패턴 P를 찾는다고 합시다. 가장 단순한 방법은 모든 위치에서 하나씩 대보는 것입니다.

단순 검색 (naive)
for (int i=0; i<=n-m; i++){
    int k=0;
    while (k<m && T[i+k]==P[k]) k++;
    if (k==m) printf("발견 %d\n", i);
    /* 실패하면 i++ 하고 k=0부터 다시! */
}
/* 최악 O(N·M) */
무엇이 낭비인가
// T = A A A A A A B
// P = A A A B

// i=0: AAA까지 맞고 B에서 실패
// i=1: 또 AAA 맞고 실패
// i=2: 또 AAA 맞고 실패...

// 이미 "AAA를 봤다"는 정보를
// 그냥 버리고 처음부터 다시!
KMP의 통찰 — 실패했을 때 이미 맞춘 부분이 정보를 담고 있습니다. "앞에서 ABABA를 맞췄다"면, 그 안에서 접두사와 접미사가 겹치는 부분을 활용해 텍스트를 되돌아가지 않고 패턴만 적당히 밀어낼 수 있어요.
💡 핵심 한 줄 — 텍스트 인덱스 i는 절대 뒤로 가지 않습니다. 앞으로만 N번 전진하고 끝나요. 되돌리는 건 패턴 인덱스 j뿐이고, 그 되돌릴 위치를 미리 계산해 둔 게 실패 함수 pi입니다.

💡 패턴 ABABAC를 고른 이유가 있습니다. ABABA까지는 접두사와 접미사가 3글자나 겹쳐요(ABA). 그래서 실패해도 3글자를 공짜로 건너뜁니다. 이게 KMP의 마법이에요.

실패 함수 — 접두사이면서 접미사인 것

KMP의 심장은 실패 함수 pi입니다. 정의는 딱 한 줄이에요.

pi[i] = P[0..i]의 진접두사이면서 동시에 접미사인 것 중 가장 긴 것의 길이
"진(proper)"접두사 — 자기 자신은 빼고 셉니다. ABA의 진접두사는 A, AB뿐(ABA 제외)이에요. 안 그러면 항상 자기 자신이 답이 되어 버리니까요.

우리 패턴 ABABAC로 하나씩 손으로 구해 봅시다.

iP[0..i]접두사 == 접미사?pi[i]
0A진접두사가 없음0
1ABA vs B ✗0
2ABAA == A ✓1
3ABABAB == AB ✓2
4ABABAABA == ABA ✓3

💡 ABABA를 보세요. 앞의 ABA와 뒤의 ABA가 겹칩니다(가운데 A를 공유!). 그래서 pi[4]=3이에요. "앞 3글자와 뒤 3글자가 같다"는 이 사실이 공짜 점프를 만들어 줍니다.

왜 이게 점프를 만드나

텍스트에서 ABABA까지 맞추고 6번째 글자에서 실패했다고 합시다. 단순 검색이면 한 칸 밀고 처음부터 다시 보겠죠. 하지만 KMP는 압니다 —

실패 순간
// T: ... A B A B A B ...
//            ↑ 여기까지 맞춤(5글자)
// P:     A B A B A C
//                  ↑ 여기서 실패

// 방금 맞춘 건 "ABABA"
// pi[4] = 3 → 뒤의 "ABA"는
//   패턴의 앞 "ABA"와 같다!
그래서 이렇게
// T: ... A B A B A B ...
// P:         A B A B A C
//            ↑ 2칸 밀어도
//              "ABA"는 이미 맞음!

// j = 5 → pi[4] = 3
// i 는 그대로! 다시 안 봄
// P[3]='B'와 T의 'B' 비교부터
공짜로 얻은 3글자 — 패턴을 2칸 밀었는데, 새 위치에서 앞 3글자(ABA)는 이미 맞는 게 보장됩니다. 왜냐? 접두사 ABA == 접미사 ABA이고, 그 접미사는 방금 텍스트와 맞춘 부분이니까요. 그래서 비교를 건너뛰고 4번째 글자부터 이어갑니다.

pi 만들기 — 패턴을 자기 자신과 매칭

재미있는 사실: pi를 만드는 것도 KMP 매칭입니다. 패턴을 자기 자신과 맞춰 보는 거예요. i는 접미사 끝, j는 접두사 끝을 가리킵니다.

C — 실패 함수 만들기
void buildPi(const char* P){
    int m=strlen(P);
    pi[0]=0;
    int j=0;          /* 접두사 끝 */
    for (int i=1; i<m; i++){   /* 접미사 끝 */
        while (j>0 && P[i]!=P[j])
            j = pi[j-1];   /* 되돌림! */
        if (P[i]==P[j]) j++;
        pi[i] = j;
    }
}
두 포인터의 의미
// i : 지금 보는 접미사의 끝
// j : 지금까지 맞춘 접두사 길이

// P[i] == P[j] 면?
//   접두사가 한 글자 더 자람 → j++

// 다르면?
//   j = pi[j-1] 로 줄여서 재시도
//   (더 짧은 접두사=접미사로)

// j가 0이 되면 포기, pi[i]=0
💡 j = pi[j-1]의 재귀적 아름다움 — 길이 j짜리 접두사가 실패하면, 그 접두사 안에서 다시 접두사=접미사인 더 짧은 것으로 줄입니다. 이미 계산해 둔 pi를 자기 자신이 다시 쓰는 거예요. 우리 예제의 i=5에서 이 연쇄가 나옵니다(3 → 1 → 0).

i=5의 되돌림 연쇄

패턴의 마지막 글자 C에서 무슨 일이 벌어지는지 봅시다. 이게 j = pi[j-1]의 진짜 모습이에요.

시도비교결과다음 j

2차P[5]=C vs P[1]=B불일치j = pi[0] = 0
3차P[5]=C vs P[0]=A불일치j=0 → 종료
결론pi[5] = 0
왜 3 → 1 → 0인가 — j=3은 "ABA가 접두사=접미사"라는 뜻이었죠. 그게 실패하자 그 ABA 안에서 더 짧은 접두사=접미사를 찾습니다. pi[2]=1 → A. 그것도 실패하니 pi[0]=0 → 포기. 점점 짧은 후보로 미끄러지는 거예요.

매칭 — i는 앞으로만, j만 되돌린다

pi를 만들었으니 이제 텍스트를 훑습니다. 구조가 pi 만들기와 똑같아요 — 자기 자신 대신 텍스트와 맞출 뿐입니다.

C — KMP 매칭
void kmpSearch(const char* T,
                const char* P){
    int n=strlen(T), m=strlen(P);
    buildPi(P);
    int j=0;
    for (int i=0; i<n; i++){   /* i는 전진만! */
        while (j>0 && T[i]!=P[j])
            j = pi[j-1];       /* j만 되돌림 */
        if (T[i]==P[j]) j++;
        if (j==m){
            printf("발견 %d\n", i-m+1);
            j = pi[j-1];   /* 겹친 매칭도 찾기 */
        }
    }
}

⚠️ 매칭 후에도 j = pi[j-1] — 패턴을 다 찾았다고 j=0으로 리셋하면 겹치는 매칭을 놓칩니다. 예를 들어 T=AAAA, P=AA면 0, 1, 2 세 곳에서 발견돼야 해요. pi를 써서 되돌리면 겹친 것도 정확히 찾습니다.

왜 O(N+M)인가 — j의 상환 분석

while 루프가 안에 있는데 어떻게 O(N)일까요? 언뜻 O(N·M)처럼 보이지만, 상환 분석(amortized)으로 보면 선형입니다.

j는 한 번에 1씩만 증가(최대 N번) → 총 감소량도 N을 못 넘음
돈에 비유하면 — 바깥 for가 한 바퀴 돌 때 j++로 1원씩 저금합니다(총 N원). 안쪽 while의 j = pi[j-1]은 저금한 돈을 빼 쓰는 것이고요. j는 0 아래로 못 가니, 넣은 것보다 많이 뺄 수 없어요. 그래서 while의 총 실행 횟수도 N번 이하입니다.

실제로 문자 비교를 몇 번 하는지 세어 봅시다.

버튼을 눌러 두 방식의 비교 횟수를 확인하세요.
⚠️ naive의 진짜 약점 — T = AAAA...AB, P = AAAAAB 같은 입력에서 단순 검색은 매 위치마다 거의 끝까지 맞추다 마지막에 실패합니다. 그걸 N번 반복해 O(N·M)이 돼요. KMP는 pi 덕분에 이런 반복 구조를 한 번만 훑습니다.

최소 주기의 마법 — 문자열 ABABAB(길이 6)의 pi[5]는 4입니다. 그러면 최소 주기는 6 − 4 = 2 → AB가 3번 반복! 접두사=접미사 정보 하나로 주기까지 나옵니다. pi는 문자열의 반복 구조를 통째로 담은 표예요.
💡 다음 여정 — 아호–코라식 — 트라이 편에서 만든 접두사 트리에, 이번에 배운 실패 링크를 달면 아호–코라식이 됩니다. 패턴 하나가 아니라 수천 개를 동시에 한 번의 스캔으로 찾아요. 트라이 + KMP = 아호–코라식입니다.
