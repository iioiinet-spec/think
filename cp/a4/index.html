트리 분할 HLD

트리를 무거운 사슬로 쪼개, 경로 질의를 세그먼트 트리의 구간 질의로 바꾸기

C 시각화: heavy/light · 체인 분해 · pos 번호 · 경로→구간

트리 경로 질의 — 배열이 아닌 트리에서

세그먼트 트리는 배열의 구간을 O(log n)에 다뤘습니다. 그런데 문제가 트리에서 나오면요?

이런 질문들 — "노드 u에서 v로 가는 경로 위 값들의 합은?", "그 경로의 최댓값은?", "경로 위 모든 노드에 v를 더하라". 트리의 경로는 배열의 구간이 아니라서 세그먼트 트리를 바로 쓸 수 없어요.

순진하게 경로를 한 노드씩 걸어가면 최악 O(n)입니다. 한 줄로 늘어진 트리(사슬)라면 경로가 n개 노드를 지나거든요. 질의가 많으면 감당이 안 됩니다.

방법경로 질의값 갱신
경로 직접 순회O(n)O(1)
HLD + 세그먼트 트리O(log²n)O(log n)
💡 HLD(Heavy-Light Decomposition)의 아이디어 한 줄 — "트리를 몇 개의 사슬(체인)로 쪼개고, 각 체인을 배열에 연속으로 눕히자." 그러면 트리의 경로가 몇 개의 배열 구간이 되고, 세그먼트 트리를 그대로 쓸 수 있습니다.

핵심 전략 — 경로를 구간으로 환원

⛓️ 체인으로 쪼개기

각 노드에서 가장 무거운 자식(서브트리가 큰)으로 가는 간선을 heavy로 정해, heavy 간선을 이어 사슬을 만듭니다.

📏 배열에 눕히기

각 체인의 노드에 연속된 번호(pos)를 매겨 배열로 만듭니다. 한 체인 = 배열의 연속 구간!

환원의 완성 — 경로 u→v를 따라가면 체인을 몇 번 갈아타며 올라갑니다. 각 체인 조각은 배열의 연속 구간이니 세그먼트 트리 질의 한 번이면 돼요. 갈아타는 횟수가 O(log n)이라, 전체가 O(log n) × O(log n) = O(log²n)입니다.
💡 앞선 편들의 총집합입니다 — 세그먼트 트리(구간 질의)와 LCA(경로 이해)를 배웠고, 이제 HLD가 트리 경로를 배열 구간으로 번역해 둘을 잇습니다. "문제 환원"의 정점이에요.

heavy 간선 — 가장 무거운 자식으로

HLD의 출발점은 서브트리 크기입니다. 각 노드에서 서브트리가 가장 큰 자식 하나를 골라, 그리로 가는 간선을 heavy(무거운) 간선이라 부릅니다. 나머지는 모두 light(가벼운) 간선이에요.

C — 서브트리 크기 + heavy 고르기
void dfs_size(int u,int p){
    par[u]=p; sz[u]=1; heavy[u]=0;
    int maxsz=0;
    for (int v : adj[u]){
        if (v==p) continue;
        dep[v]=dep[u]+1;
        dfs_size(v,u);
        sz[u]+=sz[v];
        if (sz[v] > maxsz){
            maxsz=sz[v]; heavy[u]=v;
        }
    }
}
용어 정리
// sz[u]  : u의 서브트리 노드 수
// heavy[u]: 가장 큰 자식 (하나)

// heavy 간선: u → heavy[u]
// light 간선: 그 외 모든 자식

// 리프는 heavy[u] = 0 (없음)
// 자식이 하나면 그게 heavy
💡 왜 "가장 큰 자식"일까요? light 간선을 타면 서브트리 크기가 절반 이하로 준다는 성질 때문입니다. 그래서 어떤 경로든 light 간선을 O(log n)번만 지나요. 이게 HLD 복잡도의 뿌리입니다(4번 탭에서 증명).

체인 분해 — heavy를 따라 사슬 만들기

heavy 간선들을 이으면 사슬(체인)이 됩니다. 각 노드는 정확히 하나의 체인에 속해요. 그리고 체인의 노드들에 연속된 번호 pos를 매깁니다.

C — 체인 분해
void decompose(int u,int h){
    head[u]=h;              /* 체인 머리 */
    pos[u]=cur_pos++;       /* 연속 번호! */
    base[pos[u]]=val[u];
    if (heavy[u])
        decompose(heavy[u], h);  /* 같은 체인 */
    for (int v : adj[u]){
        if (v!=par[u] && v!=heavy[u])
            decompose(v, v);       /* 새 체인! */
    }
}
/* 호출: decompose(root, root) */
두 배열의 뜻
// head[u]: u가 속한 체인의 머리
//   같은 체인 = head가 같음

// pos[u] : u의 배열 위치
//   한 체인은 pos가 연속!

// 핵심: heavy를 먼저 재귀
//   → 체인이 배열에 통째로
//     연속으로 눕혀짐
💡 heavy를 먼저 방문하는 것이 전부입니다. heavy 자식을 먼저 재귀하면 그 체인의 노드들이 pos를 연속으로 받아요. light 자식은 그 뒤에 새 체인으로 시작합니다. 이 한 줄의 순서가 "체인 = 배열 구간"을 만듭니다.

경로 질의 — 체인을 갈아타며 올라가기

이제 본론입니다. 경로 u→v의 합을 구하려면, 체인 머리가 더 깊은 쪽을 골라 그 체인 구간을 통째로 질의하고, 체인 머리의 부모로 점프합니다. 두 노드가 같은 체인에 올 때까지 반복해요.

C — 경로 질의
ll path_query(int u,int v){
    ll res=0;
    while (head[u] != head[v]){  /* 다른 체인 */
        if (dep[head[u]] < dep[head[v]])
            swap(u,v);          /* u의 머리가 더 깊게 */
        res += query(pos[head[u]], pos[u]);
        u = par[head[u]];       /* 머리 위로 점프! */
    }
    /* 같은 체인 도착 */
    if (dep[u] > dep[v]) swap(u,v);
    res += query(pos[u], pos[v]);  /* 마지막 구간 */
    return res;
}
왜 "머리가 깊은 쪽"인가
// 두 노드가 다른 체인에 있으면
// 아직 LCA에 도달 못 함

// 체인 머리가 더 깊은 쪽이
// LCA에서 더 멀다
//   → 그쪽을 먼저 올린다

// 그래야 LCA를 지나쳐
// 너무 올라가지 않음

// 같은 체인 = LCA 만남!
마지막 구간이 LCA를 포함 — 반복이 끝나 head[u]==head[v]가 되면 두 노드는 같은 체인에 있고, 그 사이 구간이 LCA를 포함한 마지막 조각입니다. 얕은 쪽이 곧 LCA예요. 즉 HLD는 LCA도 공짜로 구해줍니다.

light 간선의 마법 — 절반씩 줄어든다

HLD 복잡도의 뿌리는 딱 한 가지 성질입니다.

u → v가 light 간선이면    sz[v] < sz[u] / 2
왜 그런가 — v가 light라는 건 v보다 큰 형제(heavy)가 있다는 뜻입니다. 즉 sz[heavy] ≥ sz[v]. 그런데 sz[u] ≥ 1 + sz[v] + sz[heavy] ≥ 1 + 2·sz[v]이므로 sz[v] < sz[u]/2. 광탈이죠!

따라서 루트에서 어떤 노드로 내려가며 light 간선을 탈 때마다 서브트리가 절반 이하로 줄어듭니다. n에서 시작해 절반씩 줄면 최대 log₂n번밖에 못 타요.

💡 정리 — 어떤 경로든 light 간선을 O(log n)번만 지납니다. 그리고 경로에서 체인을 갈아타는 건 항상 light 간선을 탈 때예요(같은 체인 안은 heavy로 쭉 이어짐). 그래서 체인 조각의 개수 = O(log n)입니다.

💡 서브트리 질의는 공짜 — DFS 순서로 pos를 매겼으니, u의 서브트리는 pos[u]부터 sz[u]개가 연속입니다. 즉 서브트리 = 배열 구간 하나! 경로 질의(O(log²n))와 서브트리 질의(O(log n))를 한 자료구조로 동시에 처리할 수 있어요.
지연 전파와의 결합 — 앞 편에서 배운 lazy propagation을 이 세그먼트 트리에 얹으면 "경로 위 모든 노드에 v를 더하라" 같은 구간 갱신도 O(log²n)에 됩니다. 세그먼트 트리 → 지연 전파 → HLD로 쌓아 올린 탑이 여기서 완성돼요.
