트리 — 사이클 없는 연결 그래프

가계도, 폴더 구조, 조직도, 토너먼트 대진표… 모두 트리입니다. 트리는 그래프의 특별한 한 종류로, 정의가 아주 간결합니다.

트리의 조건설명
연결되어 있다어느 정점에서든 다른 모든 정점에 갈 수 있다
사이클이 없다빙 돌아 제자리로 오는 길이 없다
⟹ 간선이 정확히 V−1개정점 9개면 간선은 반드시 8개
⟹ 두 정점 사이 경로는 유일돌아가는 다른 길이 없다
💡 위 네 가지는 서로 맞물려 있습니다. "연결 + 사이클 없음"이면 자동으로 "간선 V−1개"가 되고, "연결 + 간선 V−1개"여도 자동으로 사이클이 없습니다. 문제에서 "트리가 주어진다"고 하면 이 성질들을 마음껏 써도 됩니다.

루트를 정하면 위아래가 생긴다

트리 자체에는 원래 방향이 없습니다. 하지만 정점 하나를 루트(root)로 정해 매달면, 갑자기 부모·자식·조상·자손이라는 관계가 생깁니다. 이것을 루트 있는 트리(rooted tree)라고 합니다.

용어뜻위 그림에서
루트 (root)맨 위, 부모가 없는 정점1번
부모 (parent)바로 위 정점 — 루트 말고는 모두 하나씩5의 부모 = 2
자식 (child)바로 아래 정점들1의 자식 = 2, 3, 4
리프 (leaf)자식이 없는 정점3, 5, 7, 8, 9
깊이 (depth)루트에서 그 정점까지 간선 수depth(8) = 3
높이 (height)가장 깊은 정점의 깊이이 트리는 3
서브트리 (subtree)어떤 정점과 그 모든 자손2의 서브트리 = {2,5,6,8,9}
⚠️ 깊이는 0부터? 1부터? 문제마다 다릅니다! "루트의 깊이는 0"이 흔하지만 1로 세는 문제도 많아요. 지문을 반드시 확인하세요. 이 강의에서는 루트 깊이 = 0으로 통일합니다.

이진트리 — 자식이 최대 둘

자식을 왼쪽(left)·오른쪽(right) 최대 2개까지만 갖는 트리를 이진트리라 합니다. 자식의 순서가 의미를 갖는다는 점이 일반 트리와 다릅니다 — 왼쪽 자식만 있는 것과 오른쪽 자식만 있는 것은 서로 다른 트리입니다!

이번 강의의 이진트리 예제입니다. 3번은 왼쪽 자식이 없고(빈 자리는 점선), 오른쪽 자식만 6번을 가집니다. 이런 비대칭이 순회 결과를 흥미롭게 만듭니다.

비교일반 트리이진트리
자식 수제한 없음최대 2개
자식 순서보통 의미 없음왼쪽/오른쪽 구분이 중요
저장인접 리스트left[]/right[] 배열 또는 노드 포인터
대표 순회DFS(전위), BFS(레벨)전위·중위·후위·레벨

트리는 어떻게 입력되는가

KOI 문제에서 트리는 대개 두 가지 형태로 주어집니다. 어느 쪽인지 파악하는 것이 첫 단추입니다.

형태 ① 간선 목록 (일반 트리)
9 ← 정점 수 n 1 2 ← 간선 (부모·자식 구분 없음!) 1 3 1 4 2 5 2 6 4 7 6 8 6 9 ← 총 n-1 = 8줄
형태 ② 자식 지정 (이진트리)
8 ← 정점 수 1 2 3 ← 노드 왼쪽 오른쪽 2 4 5 3 0 6 ← 0 = 자식 없음 4 0 0 5 7 8 6 0 0 7 0 0 8 0 0
⚠️ 형태 ①의 함정 — "1 2"는 1이 부모라는 뜻이 아닙니다! 그냥 두 정점이 이어져 있다는 것뿐. 부모·자식 관계는 루트를 정하고 DFS를 돌려야 비로소 결정됩니다. 그래서 양방향으로 저장해야 해요.

저장 ① 일반 트리 — 인접 리스트 (양방향)

트리도 그래프이므로 인접 리스트에 넣습니다. 간선을 양쪽 모두 저장하는 것이 핵심입니다.

C — 인접 리스트
int adjv[100005][20], deg[100005];
/* 간선이 많으면 정적 배열 대신
   벡터 형태(연결 리스트)를 쓰기도 함 */

scanf("%d", &n);
for (int i = 0; i < n - 1; i++) {  /* n-1개! */
    int u, v;
    scanf("%d %d", &u, &v);
    adjv[u][deg[u]++] = v;
    adjv[v][deg[v]++] = u;   /* 양방향! */
}
Python — 인접 리스트
import sys
input = sys.stdin.readline

n = int(input())
adj = [[] for _ in range(n + 1)]
for _ in range(n - 1):        # n-1개!
    u, v = map(int, input().split())
    adj[u].append(v)
    adj[v].append(u)       # 양방향!
💡 양방향인데 어떻게 부모로 안 돌아가지? DFS를 돌 때 부모 번호를 인자로 들고 다니며 "이웃 v가 부모면 건너뛴다"로 처리합니다. 이것이 헤더에 적힌 dfs(u, parent)의 의미입니다. 트리는 사이클이 없으므로 visited 배열조차 필요 없습니다!

저장 ② 이진트리 — left / right 배열

C — 자식 배열
int L[105], R[105];  /* 0 = 자식 없음 */

scanf("%d", &n);
for (int i = 0; i < n; i++) {
    int u, l, r;
    scanf("%d %d %d", &u, &l, &r);
    L[u] = l; R[u] = r;
}
/* 0번 노드를 "없음"으로 쓰므로
   정점 번호는 1부터 시작해야 함 */
Python — 딕셔너리 또는 리스트
n = int(input())
L = [0] * (n + 1)
R = [0] * (n + 1)
for _ in range(n):
    u, l, r = map(int, input().split())
    L[u], R[u] = l, r

# 클래스로 노드를 만드는 방법도 있다
class Node:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None
C 관점 — 배열 방식은 메모리가 연속이라 빠르고 구현이 짧습니다. 노드를 malloc으로 만들고 포인터로 잇는 방식도 있지만, 대회에서는 배열이 압도적으로 유리합니다.
Python 관점 — 배열(리스트) 방식이 가장 간단합니다. None을 자식 없음으로 쓰는 클래스 방식은 개념 학습이나 실무에 좋지만, 대회에선 리스트가 빠릅니다.

완전 이진트리의 마법 — 인덱스 계산

빈 자리 없이 위에서 아래로, 왼쪽에서 오른쪽으로 꽉 찬 완전 이진트리라면, 자식 포인터조차 필요 없습니다. 배열 인덱스만으로 부모와 자식을 찾을 수 있어요.

왼쪽 자식 = 2i  ·  오른쪽 자식 = 2i + 1  ·  부모 = i / 2

순회 — 모든 노드를 빠짐없이 한 번씩

트리의 모든 노드를 방문하는 것을 순회(traversal)라 합니다. 이진트리에는 세 가지 재귀 순회가 있는데, 차이는 딱 하나 — "나 자신"을 언제 출력하느냐입니다.

순회순서기억법
전위 (preorder)나 → 왼쪽 → 오른쪽내가 먼저(pre)
중위 (inorder)왼쪽 → 나 → 오른쪽내가 가운데(in)
후위 (postorder)왼쪽 → 오른쪽 → 나내가 나중(post)
C — 세 순회는 한 줄 위치만 다르다
void preorder(int u) {
    if (!u) return;       /* 0 = 빈 노드 */
    printf("%d ", u);   /* ← 나 */
    preorder(L[u]);
    preorder(R[u]);
}
void inorder(int u) {
    if (!u) return;
    inorder(L[u]);
    printf("%d ", u);   /* ← 나 */
    inorder(R[u]);
}
void postorder(int u) {
    if (!u) return;
    postorder(L[u]);
    postorder(R[u]);
    printf("%d ", u);   /* ← 나 */
}
Python — 똑같이 한 줄 위치만
def preorder(u):
    if u == 0:            # 빈 노드
        return
    print(u, end=' ')    # ← 나
    preorder(L[u])
    preorder(R[u])

def inorder(u):
    if u == 0:
        return
    inorder(L[u])
    print(u, end=' ')    # ← 나
    inorder(R[u])

def postorder(u):
    if u == 0:
        return
    postorder(L[u])
    postorder(R[u])
    print(u, end=' ')    # ← 나
💡 놀랍도록 단순합니다 — 세 함수는 print 한 줄의 위치만 다릅니다. 재귀 호출 순서는 항상 왼쪽 먼저, 오른쪽 나중이에요. "나를 언제 찍을까"만 정하면 끝!

레벨 순회 — 위층부터 한 층씩

앞의 세 순회는 모두 재귀(DFS)였습니다. 네 번째 순회인 레벨 순회(level order)는 큐(BFS)를 씁니다. 루트부터 시작해 같은 깊이의 노드를 왼쪽에서 오른쪽으로 훑습니다.

C — 배열 큐로 레벨 순회
int q[105];

void levelorder(int root) {
    int head = 0, tail = 0;
    q[tail++] = root;
    while (head < tail) {
        int u = q[head++];
        printf("%d ", u);
        if (L[u]) q[tail++] = L[u];
        if (R[u]) q[tail++] = R[u];
    }
}
Python — deque로 레벨 순회
from collections import deque

def levelorder(root):
    q = deque([root])
    while q:
        u = q.popleft()
        print(u, end=' ')
        if L[u]: q.append(L[u])
        if R[u]: q.append(R[u])
🔗 ⑪편 BFS 연결 — 구조가 BFS와 완전히 같습니다. 트리는 사이클이 없으므로 visited 배열이 필요 없다는 점만 다릅니다. 부모로 되돌아갈 일이 없으니까요!

일반 트리 DFS — 부모를 들고 다니기

일반 트리(인접 리스트, 양방향)에서는 부모 번호를 인자로 넘겨 되돌아가지 않게 합니다. 이 한 번의 DFS로 부모·깊이·서브트리 크기를 한꺼번에 구할 수 있습니다.

C — 한 번의 DFS로 세 가지
int par[N], dep[N], sz[N];

void dfs(int u, int p) {
    par[u] = p;
    sz[u] = 1;              /* 자기 자신 */
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (v == p) continue;  /* 부모면 스킵 */
        dep[v] = dep[u] + 1;
        dfs(v, u);
        sz[u] += sz[v];       /* 후위: 자식 끝난 뒤 */
    }
}

dep[1] = 0;
dfs(1, 0);   /* 루트 1, 부모는 0(없음) */
Python — 같은 구조
import sys
sys.setrecursionlimit(300000)

par = [0] * (n + 1)
dep = [0] * (n + 1)
sz  = [1] * (n + 1)

def dfs(u, p):
    par[u] = p
    for v in adj[u]:
        if v == p:
            continue       # 부모면 스킵
        dep[v] = dep[u] + 1
        dfs(v, u)
        sz[u] += sz[v]    # 후위: 자식 끝난 뒤

dfs(1, 0)             # 루트 1
💡 서브트리 크기가 왜 후위에서 계산되나? "내 크기 = 1 + 모든 자식 서브트리 크기의 합"이므로, 자식들이 먼저 끝나야 내 값을 확정할 수 있습니다. 그래서 sz[u] += sz[v]가 재귀 호출 뒤에 있는 것이죠. 이것이 트리 DP의 출발점입니다.

C — 전체 프로그램
#include <stdio.h>
#define MAXN 105

int L[MAXN], R[MAXN];
int q[MAXN];

void preorder(int u) {
    if (!u) return;
    printf("%d ", u);
    preorder(L[u]); preorder(R[u]);
}
void inorder(int u) {
    if (!u) return;
    inorder(L[u]);
    printf("%d ", u);
    inorder(R[u]);
}
void postorder(int u) {
    if (!u) return;
    postorder(L[u]); postorder(R[u]);
    printf("%d ", u);
}
void levelorder(int root) {
    int h = 0, t = 0;
    q[t++] = root;
    while (h < t) {
        int u = q[h++];
        printf("%d ", u);
        if (L[u]) q[t++] = L[u];
        if (R[u]) q[t++] = R[u];
    }
}
int height(int u) {
    if (!u) return -1;   /* 빈 트리 = -1 */
    int a = height(L[u]), b = height(R[u]);
    return 1 + (a > b ? a : b);
}

int main(void) {
    int n, root;
    scanf("%d %d", &n, &root);
    for (int i = 0; i < n; i++) {
        int u, l, r;
        scanf("%d %d %d", &u, &l, &r);
        L[u] = l; R[u] = r;
    }
    printf("전위: "); preorder(root);  printf("\n");
    printf("중위: "); inorder(root);   printf("\n");
    printf("후위: "); postorder(root); printf("\n");
    printf("레벨: "); levelorder(root);printf("\n");
    printf("높이: %d\n", height(root));
    return 0;
}
Python — 전체 프로그램
import sys
from collections import deque
input = sys.stdin.readline
sys.setrecursionlimit(300000)

n, root = map(int, input().split())
L = [0] * (n + 1)
R = [0] * (n + 1)
for _ in range(n):
    u, l, r = map(int, input().split())
    L[u], R[u] = l, r

def preorder(u, acc):
    if u == 0: return
    acc.append(u)
    preorder(L[u], acc); preorder(R[u], acc)

def inorder(u, acc):
    if u == 0: return
    inorder(L[u], acc)
    acc.append(u)
    inorder(R[u], acc)

def postorder(u, acc):
    if u == 0: return
    postorder(L[u], acc); postorder(R[u], acc)
    acc.append(u)

def levelorder(root):
    acc, q = [], deque([root])
    while q:
        u = q.popleft()
        acc.append(u)
        if L[u]: q.append(L[u])
        if R[u]: q.append(R[u])
    return acc

def height(u):
    if u == 0: return -1   # 빈 트리 = -1
    return 1 + max(height(L[u]), height(R[u]))

a = []; preorder(root, a);  print('전위:', *a)
a = []; inorder(root, a);   print('중위:', *a)
a = []; postorder(root, a); print('후위:', *a)
print('레벨:', *levelorder(root))
print('높이:', height(root))

재귀 없이 순회하기 — 스택 사용

재귀가 부담스러울 때(파이썬 재귀 한계, C 스택 오버플로) 명시적 스택으로 바꿀 수 있습니다. 전위 순회가 가장 쉽습니다.

C — 반복 전위 순회
void preorder_iter(int root) {
    int st[MAXN], top = 0;
    st[top++] = root;
    while (top > 0) {
        int u = st[--top];
        printf("%d ", u);
        /* 오른쪽 먼저 넣어야
           왼쪽이 먼저 나온다! */
        if (R[u]) st[top++] = R[u];
        if (L[u]) st[top++] = L[u];
    }
}
Python — 반복 전위 순회
def preorder_iter(root):
    acc, st = [], [root]
    while st:
        u = st.pop()
        acc.append(u)
        # 오른쪽 먼저 넣어야
        # 왼쪽이 먼저 나온다!
        if R[u]: st.append(R[u])
        if L[u]: st.append(L[u])
    return acc
⚠️ 스택은 LIFO(나중에 넣은 게 먼저 나옴)이므로, 왼쪽을 먼저 방문하려면 오른쪽을 먼저 push해야 합니다. 반대로 하면 오른쪽부터 방문하게 돼요. 큐(레벨 순회)는 FIFO라 왼쪽부터 넣는 것과 대조됩니다!
💡 중위·후위의 반복 버전은 훨씬 까다롭습니다. 실전에서는 재귀를 쓰되 파이썬은 재귀 한계를 올리고, C는 트리 깊이가 10만을 넘지 않는지 확인하는 편이 낫습니다.
