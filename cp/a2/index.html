두 개의 트리 구조 — 구간과 조상

이번엔 그래프에서 트리 자료구조로 넘어옵니다. 두 강력한 도구를 배워요.

📊 세그먼트 트리

배열의 구간 질의(합·최소·최대)와 갱신을 각각 O(log n)에 처리. 단순 반복(O(n))보다 훨씬 빠릅니다.

🌳 LCA (최소 공통 조상)

트리에서 두 노드의 가장 가까운 공통 조상을 찾기. 이진 리프팅으로 전처리 후 각 질의를 O(log n)에 답합니다.

둘의 연결 — 놀랍게도 LCA는 세그먼트 트리로 풀 수 있습니다. 트리를 오일러 투어로 펼치면 LCA 질의가 구간 최소 질의(RMQ)가 되고, 그건 세그먼트 트리의 특기예요. 4번 탭에서 이 다리를 건넙니다.

왜 세그먼트 트리인가 — 구간 질의의 딜레마

배열에서 "구간 [l, r]의 합"을 자주 물어보고, 중간에 값도 바꾼다고 합시다.

방법구간 질의값 갱신
매번 더하기O(n)O(1)
누적 합 배열O(1)O(n)
세그먼트 트리O(log n)O(log n)
💡 누적 합은 질의는 빠르지만 갱신이 느립니다(값 하나 바뀌면 뒤를 다 고쳐야 함). 세그먼트 트리는 질의와 갱신 둘 다 O(log n)으로 균형을 잡아요. 질의와 갱신이 섞여 나오는 문제에서 필수입니다.

세그먼트 트리 — 구간을 반씩 쪼개기

세그먼트 트리의 아이디어는 단순합니다. 배열의 구간을 계속 반으로 쪼개 트리로 만들어요. 루트는 전체 구간, 각 노드는 자기 구간의 요약값(합·최소 등)을 저장합니다.

트리 구조
// 루트 [0,7]: 전체 합
//   왼쪽 [0,3]  오른쪽 [4,7]
//     [0,1][2,3]  [4,5][6,7]
//       ...리프까지 계속

// 각 노드 = 자기 구간의 합
// 부모 = 왼쪽자식 + 오른쪽자식
// 리프 = 배열 원소 하나
C — 재귀 build
void build(int node,int s,int e){
    if (s==e){         /* 리프 */
        tree[node]=arr[s];
        return;
    }
    int mid=(s+e)/2;
    build(2*node, s, mid);
    build(2*node+1, mid+1, e);
    tree[node]=tree[2*node]
             +tree[2*node+1];
}
💡 1-인덱스 트릭 — 노드 번호를 1부터 시작하면, 노드 i의 왼쪽 자식은 2i, 오른쪽 자식은 2i+1입니다. 배열 하나로 트리를 표현할 수 있어 간단해요. 크기는 넉넉히 4n으로 잡습니다.

구축 복잡도

노드가 O(n)개(리프 n개 + 내부 노드 ~n개)이고, 각 노드를 한 번씩 방문하므로 구축은 O(n)입니다.

합만이 아니다 — 세그먼트 트리는 결합 법칙이 성립하는 연산이면 뭐든 담을 수 있습니다. 합, 최소, 최대, 최대공약수(GCD) 등. 부모를 만들 때 tree[node] = 자식1 ⊕ 자식2에서 ⊕만 바꾸면 돼요. 최소 세그먼트 트리는 min(왼쪽, 오른쪽)이 됩니다.

구간 질의 — 세 가지 경우

구간 [l, r]의 합을 물으면, 루트부터 내려가며 각 노드를 세 경우로 나눕니다.

경우처리
완전히 벗어남이 노드는 무시 (0 반환)
완전히 포함됨이 노드 값을 그대로 사용 (더 안 내려감)
일부만 겹침양쪽 자식으로 재귀
C — 구간 질의
int query(int node,int s,int e,
          int l,int r){
    if (r<s || e<l)           /* 벗어남 */
        return 0;
    if (l<=s && e<=r)         /* 포함 */
        return tree[node];
    int mid=(s+e)/2;      /* 겹침 */
    return query(2*node,s,mid,l,r)
         + query(2*node+1,mid+1,e,l,r);
}
왜 O(log n)인가
// 질의 구간은 각 레벨에서
// 최대 4개 노드만 방문

// "완전 포함"을 만나면 멈춤
// → 아래로 안 내려감

// 트리 높이 = log n
// → 전체 O(log n)개 노드
💡 핵심은 "완전히 포함"에서 멈추는 것입니다. 큰 구간이 통째로 포함되면 그 노드의 요약값을 바로 쓰고 더 내려가지 않아요. 그래서 O(n)개 리프를 다 보지 않고 O(log n)개 노드만 봅니다.

점 갱신 — 리프에서 루트로

값 하나를 바꾸면, 그 리프부터 루트까지의 경로에 있는 노드들만 다시 계산합니다. 경로 길이는 트리 높이 O(log n)이에요.

C — 점 갱신
void update(int node,int s,int e,
            int idx,int val){
    if (s==e){ tree[node]=val; return; }
    int mid=(s+e)/2;
    if (idx<=mid)
        update(2*node,s,mid,idx,val);
    else
        update(2*node+1,mid+1,e,idx,val);
    tree[node]=tree[2*node]
             +tree[2*node+1];
}
지연 전파 (Lazy) 맛보기
// 점 갱신: O(log n)

// 구간 전체 갱신도 필요하면?
//   "지연 전파(lazy propagation)"
//   갱신을 미뤄뒀다 필요할 때만
//   자식에게 전달

// → 구간 갱신도 O(log n)!
//   (별도 주제, 심화 과정)
왜 루트까지 올라가나 — 리프 값이 바뀌면 그 조상 노드들의 요약값도 바뀝니다. 리프에서 루트까지 경로의 노드들을 재귀에서 돌아오며 다시 계산하면, 트리가 항상 올바른 상태를 유지해요. 이 경로가 O(log n)이라 갱신도 빠릅니다.

LCA — 최소 공통 조상

트리에서 두 노드 u, v의 최소 공통 조상(LCA)은 둘 다의 조상이면서 가장 깊은(가까운) 노드입니다. 가계도에서 두 사람의 "가장 가까운 공통 조상"을 찾는 것과 같아요.

단순하게 하면 한 노드씩 위로 올라가며 만나는 지점을 찾지만, 최악에 O(n)입니다. 이진 리프팅(binary lifting)은 2의 거듭제곱만큼 점프해 각 질의를 O(log n)에 답해요.

이진 리프팅 테이블
// par[k][v] = v의 2^k번째 조상

// par[0][v] = 직접 부모
// par[1][v] = 2칸 위 조상
// par[2][v] = 4칸 위 조상
// par[k][v] = 2^k칸 위

// 점화식:
// par[k][v] = par[k-1][par[k-1][v]]
//   (2^(k-1) 두 번 = 2^k)
C — 테이블 구축
/* par[0][v] = 부모 (BFS/DFS로) */
for (int k=1; k<LOG; k++)
    for (int v=1; v<=n; v++)
        par[k][v] =
          par[k-1][ par[k-1][v] ];

/* 전처리 O(n log n) */
/* 각 LCA 질의 O(log n) */
💡 2의 거듭제곱 점프가 핵심입니다. 어떤 수든 2의 거듭제곱의 합으로 나타낼 수 있듯(예: 13=8+4+1), 어떤 높이든 2^k 점프의 조합으로 도달할 수 있어요. 그래서 한 칸씩 O(n)이 아니라 log n번 점프로 끝납니다.

LCA 알고리즘 두 단계

int lca(int u,int v){
    if (depth[u]<depth[v]) swap(u,v);  /* u가 더 깊게 */
    /* 1단계: 깊이 맞추기 */
    int diff=depth[u]-depth[v];
    for (int k=0; k<LOG; k++)
        if ((diff>>k)&1) u=par[k][u];   /* 깊은 쪽 끌어올림 */
    if (u==v) return u;             /* 한쪽이 조상이었음 */
    /* 2단계: 함께 점프하며 갈라지는 곳 찾기 */
    for (int k=LOG-1; k>=0; k--)
        if (par[k][u]!=par[k][v]){    /* 아직 안 만남 → 점프 */
            u=par[k][u]; v=par[k][v];
        }
    return par[0][u];              /* 부모가 LCA */
}
2단계의 묘미 — 큰 점프부터 시도하며 "아직 공통 조상에 도달하지 않은 만큼만" 올라갑니다. par[k][u]와 par[k][v]가 다르면 아직 안 만난 것이니 점프. 같으면 너무 올라간 것이니 건너뜀. 끝나면 u, v는 LCA 바로 아래에 있고, 그 부모가 답입니다.

LCA와 세그먼트 트리의 다리 — 오일러 투어

두 주제가 어떻게 만날까요? 오일러 투어(Euler tour)로 트리를 배열로 펼치면, LCA 질의가 구간 최소 질의(RMQ)가 됩니다. 그리고 RMQ는 세그먼트 트리의 특기예요.

오일러 투어란 — 트리를 DFS로 순회하며 방문하는 노드를 순서대로 기록합니다(자식에 들어갈 때, 나올 때마다). 각 노드는 여러 번 등장하고, 그때의 깊이도 함께 기록해요. 이 배열이 트리를 1차원으로 "펼친" 것입니다.

왜 LCA가 구간 최소가 되나

오일러 투어 배열에서 노드 u와 v가 처음 등장하는 위치 사이를 봅시다. 그 구간에서 깊이가 가장 작은(얕은) 노드가 바로 LCA입니다!

💡 문제 환원의 아름다움 — 트리 문제(LCA)를 배열 문제(RMQ)로 바꾸고, 그걸 세그먼트 트리로 풉니다. 겉보기엔 전혀 다른 두 개념(구간 질의와 조상 찾기)이 오일러 투어라는 다리로 연결돼요. 이런 "환원"이 알고리즘 설계의 핵심 사고입니다.
