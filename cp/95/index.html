경유 정점 k를 늘려가며 2차원 거리 행렬이 어떻게 갱신되는지 3중 for문과 나란히

C 시각화: 거리 행렬 d[i][j] · 경유 정점 k · 3중 for문

플로이드–워셜 — 모든 쌍의 최단 거리를 한 번에

다익스트라·벨만–포드는 한 시작점에서 모든 곳까지의 거리를 구합니다. 그런데 모든 정점에서 모든 정점까지의 거리가 필요하다면? 다익스트라를 V번 돌릴 수도 있지만, 플로이드–워셜은 이걸 딱 3중 for문으로 우아하게 풉니다.

핵심 아이디어는 경유 정점입니다. "i에서 j로 갈 때, 1번 정점을 거쳐도 되는지" 따져보고, 그다음 "1,2번을 거쳐도 되는지"... 이렇게 거쳐도 되는 정점을 하나씩 늘려가며 모든 최단 거리를 완성합니다.

알고리즘구하는 것복잡도
BFS한 점 → 모두 (가중치 1)O(V+E)
다익스트라한 점 → 모두 (양수)O((V+E)logV)
벨만–포드한 점 → 모두 (음수 OK)O(V·E)
플로이드–워셜모든 쌍 → 모든 쌍O(V³)
💡 코드가 믿기지 않을 만큼 짧습니다. 단 세 줄의 for문과 한 줄의 갱신이 전부예요. 그 간결함 속에 "경유 정점을 늘려간다"는 깊은 아이디어가 숨어 있습니다.

이 강의의 예제 그래프

정점 4개, 간선 7개인 가중치 유방향 그래프입니다. 음수 간선도 있지만 음수 사이클은 없어요.

INPUT (u v w = u→v 방향, 가중치 w)
4 7 1 2 3 1 3 8 1 4 -4 2 3 4 2 4 1 3 2 -2 4 3 2

플로이드–워셜은 이 그래프의 16개 쌍(4×4) 모두에 대한 최단 거리를 한 번에 구합니다.

거리 행렬 — 모든 쌍을 담는 표

플로이드–워셜은 1차원 dist[]가 아니라 2차원 행렬 d[i][j]를 씁니다. i행 j열은 "i에서 j까지의 최단 거리"예요.

대각선 d[i][i]=0 (자기 자신) 간선으로 직접 연결 INF아직 경로 모름
초기 행렬 — 처음엔 간선 정보만 채웁니다. 대각선은 0(자기 자신까지 0), 직접 간선이 있으면 그 가중치, 나머지는 INF(아직 가는 법 모름). 여기서 시작해 경유 정점을 늘려갑니다.

경유 정점 — "k를 거쳐 가면 더 짧은가?"

플로이드–워셜의 모든 것은 이 한 질문에 담겨 있습니다. "i에서 j로 곧장 가는 것보다, 정점 k를 거쳐 가면 더 짧은가?"

d[i][j] = min( d[i][j],  d[i][k] + d[k][j] )

왼쪽 d[i][j]는 "지금까지 알던 i→j 최단". 오른쪽 d[i][k]+d[k][j]는 "i→k 최단 + k→j 최단", 즉 k를 경유하는 거리. 둘 중 작은 걸 택합니다.

갱신 한 줄
/* k를 경유하면 더 짧은가? */
if (d[i][k] + d[k][j] < d[i][j])
    d[i][j] = d[i][k] + d[k][j];
그림으로
    d[i][k]      d[k][j]
  i ────────► k ────────► j
   \                     /
    \___________________/
         d[i][j]
    (직접 경로)

  경유가 더 짧으면 d[i][j] 갱신
💡 벨만–포드의 완화가 "간선 하나로 줄이기"였다면, 플로이드–워셜은 "중간 정점 하나로 줄이기"입니다. 관점이 간선에서 경유지로 바뀐 것이죠.

바깥 for가 k인 이유 — 순서가 생명

3중 for문에서 k가 가장 바깥이어야 합니다. 이것이 플로이드–워셜의 정확성을 보장하는 가장 중요한 규칙이에요.

✅ 올바른 순서
for (k = 1; k <= n; k++)   // 경유!
  for (i = 1; i <= n; i++)
    for (j = 1; j <= n; j++)
      if (d[i][k]+d[k][j] < d[i][j])
        d[i][j] = d[i][k]+d[k][j];
❌ k가 안쪽이면 틀림
for (i = 1; i <= n; i++)
  for (j = 1; j <= n; j++)
    for (k = 1; k <= n; k++)  // ✗
      ...
// k가 안쪽이면 "1~k 경유"의
// 단계적 보장이 깨져 틀린 답
왜 k가 바깥인가 — k 루프가 끝날 때마다 "1번부터 k번까지만 경유해서 갈 수 있는 모든 최단 거리"가 완성됩니다. k=1이면 1만 경유, k=2면 1·2 경유... k=n이면 모든 정점을 경유 가능 = 진짜 최단. 이 단계적 확장이 k를 바깥에 둬야만 성립합니다.
⚠️ 이것은 사실 동적 계획법(DP)입니다. "k번까지 경유한 최단 거리"가 "k−1번까지 경유한 최단 거리"로부터 만들어지는 점화식이에요. k가 바깥이라는 건 DP를 올바른 순서로 채운다는 뜻입니다.

💡 핵심 관찰 — k=4 라운드를 보세요. d[1][2]가 3에서 −4로 줄어듭니다. 왜? 1→4가 −4이고 4→...→2가 0이라, 4를 경유하면 1→2가 −4가 되죠. 직접 간선(3)보다 경유가 짧은 전형적 사례입니다. "다음 k로 ⏭" 버튼으로 각 경유 단계 결과만 빠르게 볼 수도 있어요.

경유 단계별 행렬 스냅샷

k가 0(초기)부터 4까지 갈 때 행렬이 어떻게 진화하는지 단계별로 봅니다. 각 단계에서 바뀐 칸은 노랗게 표시됩니다.

k = 0 (초기)
간선 정보만 담긴 초기 행렬입니다. [다음 경유 단계 ▶]를 눌러보세요.
💡 단계별 의미 — k=1 행렬은 "1번만 경유 가능한 최단", k=2는 "1·2번 경유 가능", ... k=4는 "모든 정점 경유 가능 = 진짜 최단 거리"입니다. 각 단계가 이전 단계 위에 쌓입니다.

음수 사이클 탐지

플로이드–워셜도 음수 사이클을 탐지할 수 있습니다. 알고리즘이 끝난 뒤 대각선을 보세요. 어떤 d[i][i]가 음수가 되면, i를 지나 다시 i로 돌아오는데 거리가 줄었다는 뜻 → 음수 사이클입니다.

⚠️ 음수 사이클이 있으면 그 사이클에 닿는 모든 쌍의 최단 거리가 −∞가 되어 의미가 없습니다. 그래서 먼저 대각선으로 탐지하는 것이 중요해요.

경로 복원 — next 행렬

거리뿐 아니라 실제 경로도 복원할 수 있습니다. next[i][j]에 "i에서 j로 갈 때 첫 걸음"을 기록하고, 갱신 때 함께 바꿉니다.

C — 경로 기록
/* 초기화: 간선 i→j 있으면 */
next[i][j] = j;

/* 갱신 시 함께 */
if (d[i][k]+d[k][j] < d[i][j]) {
    d[i][j] = d[i][k]+d[k][j];
    next[i][j] = next[i][k];  /* k로 가는 첫 걸음 */
}
C — 경로 출력
void path(int i, int j) {
    if (next[i][j] == 0) return;
    printf("%d ", i);
    while (i != j) {
        i = next[i][j];
        printf("%d ", i);
    }
}
💡 5번 탭 플레이그라운드에서 실제로 각 쌍의 최단 경로가 복원되어 나옵니다. 거리 행렬과 next 행렬, 두 표가 함께 움직입니다.

최단 경로 5형제 총정리

이 시리즈에서 다룬 다섯 알고리즘을 한 표로 정리합니다. 무엇을 언제 쓸지 한눈에 보세요.

알고리즘범위가중치복잡도
BFS한 점 출발없음(1)O(V+E)
다익스트라한 점 출발양수만O((V+E)logV)
벨만–포드한 점 출발음수 OKO(V·E)
플로이드–워셜모든 쌍음수 OKO(V³)
존슨모든 쌍음수 OKO(V²logV+VE)
💡 모든 쌍이 필요할 때 — 정점이 적으면(V ≤ 400 정도) 플로이드–워셜이 구현이 압도적으로 간단해 최고의 선택입니다. 정점이 많고 간선이 적은(희소) 그래프라면 다익스트라를 V번 돌리거나 존슨 알고리즘이 유리해요.

다익스트라 V번 vs 플로이드–워셜

모든 쌍 최단 거리를 구하는 두 가지 길을 비교합니다.

다익스트라 × V플로이드–워셜
복잡도O(V(V+E)logV)O(V³)
음수 간선❌ 불가✅ 가능
구현 난이도힙 필요, 복잡3중 for, 매우 간단
희소 그래프유리 (E 작을 때)항상 V³
조밀 그래프E≈V²라 불리유리
실전 감각 — 대회에서 "모든 쌍 최단 거리"가 필요하고 V가 수백 이하라면, 고민 없이 플로이드–워셜을 씁니다. 세 줄 for문이면 5분 안에 짜고 버그도 적으니까요. V가 크거나 음수가 없으면 다른 선택을 고려합니다.

플로이드–워셜의 다른 얼굴들

같은 "경유 정점 늘리기" 뼈대로 다른 문제도 풉니다. min/+를 다른 연산으로 바꾸는 거죠.

문제갱신식 변형
최단 거리d[i][j] = min(d[i][j], d[i][k]+d[k][j])
도달 가능성(이행 폐포)r[i][j] |= (r[i][k] && r[k][j])
최대 용량 경로max(d[i][j], min(d[i][k], d[k][j]))
경로 개수 세기cnt[i][j] += cnt[i][k]*cnt[k][j]
💡 워셜 알고리즘(이행 폐포)은 사실 플로이드–워셜의 형제입니다. "거리"를 "연결됨(참/거짓)"으로 바꾸면 "i에서 j로 갈 수 있는가"를 모든 쌍에 대해 구할 수 있어요. 같은 3중 for문 뼈대예요.

내 그래프로 플로이드–워셜 돌려보기

💡 시리즈의 완성 — DFS·BFS·다익스트라·벨만–포드가 "한 점에서 출발"이었다면, 플로이드–워셜은 "모든 점에서 모든 점으로". 관점이 간선·경유지로 바뀌며 단 세 줄 for문에 모든 쌍의 최단 거리가 담깁니다. 이 우아함이 플로이드–워셜을 사랑받게 하는 이유예요.
