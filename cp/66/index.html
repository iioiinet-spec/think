함수 — 이름 붙인 작업 상자

함수는 입력을 받아 무언가를 하고 결과를 돌려주는 작업 상자입니다. 같은 코드를 반복해 쓰지 않게 해 주고, 긴 프로그램을 이해할 수 있는 크기로 잘라 줍니다.

함수를 쓰는 이유설명
재사용한 번 만들면 어디서든 호출 — 복사·붙여넣기 지옥 탈출
이름 짓기gcd(a, b)라는 이름만 봐도 무슨 일인지 안다
분할 정복큰 문제를 작은 함수들로 쪼개 각각 검증
재귀자기 자신을 불러 문제를 축소 (DFS·팩토리얼 등)
💡 KOI 문제를 풀다 보면 main이 100줄을 넘어가는 순간 실수가 폭발합니다. "이 부분에 이름을 붙일 수 있나?" 싶으면 그게 함수로 뺄 타이밍입니다.

함수의 해부 — C

C 함수는 네 부분으로 이루어집니다. 특히 반환형을 반드시 앞에 적어야 한다는 점이 파이썬과 가장 큰 차이입니다.

int gcd(int a, int b) { ... return ...; }
반환형 — 무엇을 돌려줄지 미리 선언 함수 이름 매개변수 — 타입을 반드시 명시 몸통 — 실제 작업과 return

함수의 해부 — Python

파이썬은 def 키워드로 시작하고, 타입은 선택입니다(타입 힌트). 돌려줄 게 없으면 return을 생략해도 됩니다.

def gcd(a, b): ... return ...
def 키워드 (반환형 선언 없음) 함수 이름 매개변수 — 타입 생략 가능 몸통 — 들여쓰기로 구분

첫 비교 — 최대공약수 gcd

C — 반환형·타입 필수
#include <stdio.h>

int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}

int main(void) {
    printf("%d\n", gcd(48, 18));  // 6
    return 0;
}
Python — def, 타입 생략 가능
def gcd(a, b):
    if b == 0:
        return a
    return gcd(b, a % b)


print(gcd(48, 18))   # 6

호출하면 무슨 일이 일어나는가

함수를 호출하면 프로그램의 흐름이 잠시 그 함수로 옮겨 갔다가, 일을 마치면 부른 자리로 정확히 돌아옵니다. 그 과정은 네 단계입니다.

단계일어나는 일
① 인자 전달호출할 때 준 값이 매개변수에 복사된다
② 스택 프레임 생성그 함수만의 지역 변수 공간이 새로 만들어진다
③ 몸통 실행코드를 실행하다 return을 만나면 즉시 종료
④ 반환 & 정리값을 돌려주고 스택 프레임은 사라진다 (지역 변수도 함께!)
💡 return은 "값 돌려주기 + 함수 즉시 종료" 두 가지를 동시에 합니다. return 뒤의 코드는 절대 실행되지 않아요. 그래서 조건 분기에서 일찍 return하는 패턴(early return)이 코드를 깔끔하게 만듭니다.

가장 헷갈리는 지점 — 함수가 원본을 바꿀 수 있나?

이 탭은 이 강의 전체에서 가장 중요한 내용입니다. "함수 안에서 값을 바꿨는데 왜 바깥은 그대로지?" — 두 언어 모두에서 학생들이 가장 많이 넘어지는 곳입니다.

⚠️ 핵심 문장 — C는 언제나 값을 복사합니다(원본을 바꾸려면 포인터를 넘겨야 함). 파이썬은 언제나 객체 참조를 전달합니다(바꿀 수 있는 객체면 원본이 바뀌고, 바꿀 수 없는 객체면 안 바뀜).

C — 값 복사와 포인터

❌ 값 전달 — 원본이 안 바뀐다
void swap_wrong(int a, int b) {
    int t = a; a = b; b = t;
    /* a, b는 복사본일 뿐! */
}

int x = 10, y = 20;
swap_wrong(x, y);
printf("%d %d\n", x, y);  // 10 20 (그대로!)
✅ 포인터 전달 — 원본이 바뀐다
void swap_right(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
    /* 주소를 통해 원본에 직접 접근 */
}

int x = 10, y = 20;
swap_right(&x, &y);
printf("%d %d\n", x, y);  // 20 10 (바뀜!)
C의 규칙은 단순 — 무조건 값 복사. 원본을 고치고 싶으면 주소(&x)를 복사해서 넘기고 함수 안에서 *a로 그 주소를 따라갑니다. 배열은 예외처럼 보이지만, 사실 배열 이름이 첫 원소의 주소라서 자동으로 주소가 넘어가는 것입니다.

Python — mutable이면 바뀌고, immutable이면 안 바뀐다

❌ 정수(immutable) — 안 바뀐다
def swap_wrong(a, b):
    a, b = b, a      # 지역 이름만 재연결

x, y = 10, 20
swap_wrong(x, y)
print(x, y)          # 10 20 (그대로!)

# 파이썬식 교환:
x, y = y, x          # 20 10
✅ 리스트(mutable) — 바뀐다!
def add_item(lst, v):
    lst.append(v)    # 객체 자체를 수정

def rebind(lst):
    lst = [0, 0]   # 이름만 새 객체에 재연결

arr = [1, 2, 3]
add_item(arr, 99)
print(arr)           # [1, 2, 3, 99] 바뀜!
rebind(arr)
print(arr)           # [1, 2, 3, 99] 안 바뀜!
결정적 차이 — lst.append(v)는 객체를 수정(mutate)하므로 바깥에도 보입니다. 반면 lst = [0,0]은 지역 이름 lst를 새 객체에 다시 붙이는 것뿐이라 바깥의 arr은 그대로입니다. "수정이냐, 재대입이냐"가 갈림길입니다.
⚠️ KOI 실전 함정 — 파이썬으로 함수에 리스트를 넘겨 정렬하거나 원소를 바꾸면 호출한 쪽의 리스트도 바뀝니다. 원본을 보존하려면 lst[:] 또는 list(lst)로 복사본을 만들어 넘기세요.

C의 함수는 값을 딱 하나만 돌려준다

최솟값과 최댓값을 동시에 알고 싶다면? C에서는 return이 하나뿐이라 우회해야 합니다. 파이썬은 그냥 튜플로 돌려주면 끝입니다.

C 방법 1 — 포인터로 받아 오기
/* 결과를 담을 자리의 주소를 받는다 */
void min_max(const int a[], int n,
             int *mn, int *mx) {
    *mn = *mx = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] < *mn) *mn = a[i];
        if (a[i] > *mx) *mx = a[i];
    }
}

int a[] = {5, 3, 9, 1, 7};
int mn, mx;
min_max(a, 5, &mn, &mx);
printf("%d %d\n", mn, mx);  // 1 9
C 방법 2 — 구조체로 묶어 반환
typedef struct { int mn, mx; } MinMax;

MinMax min_max2(const int a[], int n) {
    MinMax r = { a[0], a[0] };
    for (int i = 1; i < n; i++) {
        if (a[i] < r.mn) r.mn = a[i];
        if (a[i] > r.mx) r.mx = a[i];
    }
    return r;      /* 구조체 통째로 반환 */
}

MinMax r = min_max2(a, 5);
printf("%d %d\n", r.mn, r.mx);  // 1 9
Python — 튜플로 간단히
def min_max(a):
    return min(a), max(a)   # 튜플 반환


a = [5, 3, 9, 1, 7]
mn, mx = min_max(a)          # 언패킹
print(mn, mx)                # 1 9

# 튜플 그대로 받을 수도 있다
r = min_max(a)
print(r)                     # (1, 9)
print(r[0], r[1])           # 1 9

def evens(n):

지역 변수는 함수와 함께 태어나고 사라진다

함수 안에서 선언한 변수는 그 함수 안에서만 살아 있습니다. 함수가 끝나면 스택 프레임과 함께 사라져요. 그런데 "값을 유지하고 싶다"면?

C — 지역 / static / 전역
int g_count = 0;        /* 전역 변수 */

void tick(void) {
    int local = 0;      /* 매번 새로 태어남 */
    static int keep = 0; /* 호출 사이 값 유지 */
    local++; keep++; g_count++;
    printf("%d %d %d\n", local, keep, g_count);
}

tick();   // 1 1 1
tick();   // 1 2 2
tick();   // 1 3 3
Python — global / 클로저
g_count = 0              # 전역 변수

def tick():
    local = 0
    global g_count      # 전역을 쓰겠다 선언
    local += 1; g_count += 1
    print(local, g_count)

# C의 static 대응 = 클로저
def counter():
    keep = 0
    def inner():
        nonlocal keep
        keep += 1
        return keep
    return inner

c = counter()
print(c(), c(), c())   # 1 2 3
파이썬의 global / nonlocal — 함수 안에서 바깥 변수에 값을 대입하려면 선언이 필요합니다. 읽기만 할 때는 필요 없어요. global은 모듈 전역, nonlocal은 감싸는 함수의 지역 변수를 가리킵니다.
⚠️ 전역 변수는 편해 보이지만 어디서 바뀌었는지 추적이 어려워 버그의 온상입니다. KOI 코드에서 큰 배열은 전역으로 두는 게 관례지만(스택 오버플로 방지), 상태를 나타내는 값은 되도록 인자와 반환값으로 주고받으세요.

재귀 — 자기 자신을 부르는 함수

재귀 함수에는 반드시 두 가지가 있어야 합니다: 종료 조건(base case)과 더 작은 문제로 줄이는 재귀 호출. 둘 중 하나라도 빠지면 무한 재귀로 스택이 터집니다.

C — 팩토리얼
long long fact(int n) {
    if (n <= 1) return 1;  /* 종료 조건 */
    return n * fact(n - 1);  /* 축소 */
}

printf("%lld\n", fact(10));  // 3628800
printf("%lld\n", fact(20));  // 2432902008176640000
printf("%lld\n", fact(21));  // 오버플로! 음수가 나옴
Python — 팩토리얼
import sys
sys.setrecursionlimit(300000)  # 깊은 재귀 보험

def fact(n):
    if n <= 1:
        return 1            # 종료 조건
    return n * fact(n - 1)  # 축소

print(fact(10))   # 3628800
print(fact(21))   # 51090942171709440000 (정확!)
print(fact(30))   # 26525285981219105863630848...
⚠️ C의 정수 오버플로 — int는 fact(13)부터, long long도 fact(21)부터 넘칩니다. 값이 조용히 음수로 뒤집혀 틀린 답을 냅니다! 파이썬은 정수가 무한 정밀도라 이 걱정이 없지만, 대신 느립니다. KOI에서 큰 수를 다룰 때 C는 반드시 자료형 범위를 확인하세요.
💡 파이썬 재귀 깊이 — 기본 한계는 약 1000입니다. DFS로 깊이 10만까지 들어가면 RecursionError가 나므로 sys.setrecursionlimit()을 올려 두는 것이 KOI 관례입니다 (⑪편 DFS에서 다룬 그 팁!).

파이썬만의 편의 — 기본 인자와 키워드 인자

C에는 없고 파이썬에는 있는 기능들입니다. 이것 때문에 파이썬 함수 호출이 훨씬 유연합니다.

Python — 기본 인자 & 키워드 인자
def greet(name, greeting="안녕", times=1):
    for _ in range(times):
        print(f"{greeting}, {name}!")

greet("민수")                  # 안녕, 민수!
greet("영희", "반가워", 2)      # 2번 출력
greet(times=1, name="철수")     # 순서 무관!
C — 기본값 없음, 전부 명시
void greet(const char *name,
           const char *greeting, int times) {
    for (int i = 0; i < times; i++)
        printf("%s, %s!\n", greeting, name);
}

greet("민수", "안녕", 1);  /* 전부 적어야 함 */
greet("영희", "반가워", 2);
/* 순서를 바꾸거나 생략할 수 없다 */
⚠️ 파이썬 기본 인자의 유명한 함정 — 기본값으로 리스트 같은 mutable 객체를 쓰면 안 됩니다! def f(lst=[])의 그 리스트는 함수 정의 시 딱 한 번 만들어져 호출마다 공유됩니다. 대신 def f(lst=None)로 두고 안에서 if lst is None: lst = []로 처리하세요.

가변 인자 — 개수가 정해지지 않은 인자

Python — *args, **kwargs
def total(*nums):          # 위치 인자 묶음
    return sum(nums)

def info(**kw):            # 키워드 인자 묶음
    return ", ".join(f"{k}={v}"
                     for k, v in kw.items())

print(total(1, 2, 3, 4, 5))   # 15
print(info(age=15, grade=3))  # age=15, grade=3
C — stdarg.h (개수를 알려줘야 함)
#include <stdarg.h>

int total(int count, ...) {
    va_list ap;
    va_start(ap, count);
    int s = 0;
    for (int i = 0; i < count; i++)
        s += va_arg(ap, int);
    va_end(ap);
    return s;
}

printf("%d\n", total(5, 1,2,3,4,5));  // 15
/* 첫 인자로 개수 5를 직접 알려줘야! */
💡 C의 printf가 바로 가변 인자 함수입니다. 첫 인자인 형식 문자열("%d %s")이 "뒤에 몇 개가 어떤 타입으로 오는지" 알려 주는 역할이죠. 형식 문자열과 실제 인자가 안 맞으면 프로그램이 이상하게 동작하는 이유입니다.
