튜플(tuple)은 순서가 있지만 한 번 만들면 바꿀 수 없어요. 좌표(x, y)처럼 세트로 절대 안 바뀌는 값에 딱이에요!

집합(set)은 수학의 집합과 같아요. 같은 값은 하나만! 중복을 넣어도 자동으로 하나로 합쳐져요.

// 튜플 → 고정 배열 (바꾸지 않기로 약속) int point[2] = {3, 5}; // 집합 → 배열 + 중복 검사 int set[100], size = 0; // 추가 전에 "이미 있나?" 확인!

튜플 기본

만들기·접근·불변성·언패킹

1만들기와 접근( )
동작C (고정 배열)파이썬 튜플
만들기int t[3] = {1, 2, 3};t = (1, 2, 3)
접근t[0] // 1t[0] # 1
개수고정 (3)len(t)
수정t[0] = 9; // 됨(배열이라)t[0] = 9 # 오류!
⚠️ 튜플은 수정 불가! 파이썬 t[0] = 9는 TypeError가 나요.
C 배열은 기술적으론 바꿀 수 있지만, "튜플처럼 쓰려면 바꾸지 않기로 약속"하는 거예요. 파이썬은 아예 막아줘요!
2언패킹 — 한 번에 여러 변수로a, b = t

튜플의 강력한 기능! 여러 값을 한 번에 변수로 풀기(언패킹). 값 교환(swap)도 한 줄에 돼요!

🔵 C — 임시 변수 필요
// 두 값 교환 int a = 3, b = 5; int tmp = a; // 임시 변수 a = b; b = tmp; // 3줄 필요!
🟢 Python — 튜플 언패킹
# 두 값 교환 a, b = 3, 5 a, b = b, a # 한 줄! # 튜플로 묶었다 풀기
💡 파이썬 a, b = b, a는 오른쪽이 튜플 (b, a)로 묶인 뒤 왼쪽 변수로 풀려요. 그래서 임시 변수 없이 교환이 돼요! C에서 가장 부러워하는 기능 중 하나예요.
🖥 좌표 두 개를 받아 값 교환(swap)
#include <stdio.h>

int main() {
    int a, b;
    scanf("%d %d", &a, &b);
    printf("교환 전: (%d, %d)\n", a, b);
    int tmp = a;   // 임시 변수
    a = b;
    b = tmp;
    printf("교환 후: (%d, %d)\n", a, b);
    return 0;
}
# 두 값 교환 — 튜플 언패킹
a, b = map(int, input().split())
print(f"교환 전: ({a}, {b})")
a, b = b, a       # 한 줄로 교환!
print(f"교환 후: ({a}, {b})")
C: tmp 사용임시 변수로 값을 잠시 보관해야 교환됨
Py: a, b = b, a튜플 언패킹으로 임시 변수 없이 한 줄!
두 수 입력. 3 5 → 교환 후 (5, 3)

집합 만들기

중복 자동 제거 — C는 추가 전 중복 검사

1집합의 핵심 — 중복 검사이미 있나?

집합에 값을 넣을 때, C는 "이미 있는지 먼저 확인"하고 없을 때만 추가해요. 파이썬은 add가 알아서!

🔵 C — contains 후 추가
// 이미 있는지 검사 int contains(int x) { for (int i = 0; i < size; i++) if (set[i] == x) return 1; // 있음 return 0; } // 없을 때만 추가 if (!contains(x)) set[size++] = x;
🟢 Python — add 하나
s = set() s.add(3) s.add(3) # 중복! 무시됨 # 또는 리스트에서 한 번에 s = set([1, 2, 2, 3]) # {1,2,3}
💡 C에서 집합의 원리는 딕셔너리의 findKey와 똑같아요 — 선형 탐색으로 중복을 찾죠! 이게 집합 = "중복 없는 배열"의 비밀이에요.
2집합 기본 동작add·remove·in
동작파이썬C (직접 구현)
추가s.add(x)중복검사 후 set[size++]=x
삭제s.remove(x)찾아서 당기기
포함 여부x in scontains(x)
크기len(s)size
중복 제거set(리스트)하나씩 중복검사하며 복사
⚠️ 집합은 순서가 없어요! 파이썬 집합을 출력하면 넣은 순서와 다를 수 있어요.
여기 시뮬레이터는 이해를 돕기 위해 처음 등장한 순서로 보여줘요(실제 파이썬 set은 순서 보장 안 함).
🖥 숫자들에서 중복 제거하기
#include <stdio.h>

int set[100], size = 0;

int contains(int x) {
    for (int i = 0; i < size; i++)
        if (set[i] == x) return 1;
    return 0;
}
int main() {
    int n; scanf("%d", &n);
    for (int i = 0; i < n; i++) {
        int x; scanf("%d", &x);
        if (!contains(x))     // 중복 아니면
            set[size++] = x;   // 추가
    }
    for (int i = 0; i < size; i++)
        printf("%d ", set[i]);
    printf("\n개수: %d\n", size);
    return 0;
}
# 중복 제거 — set 한 방!
n = int(input())
nums = list(map(int, input().split()))

s = set(nums)    # 중복 자동 제거!

print(*sorted(s))
print(f"개수: {len(s)}")
C: contains(x)선형 탐색으로 이미 있는지 확인
C: if(!contains) 추가없을 때만 배열에 넣어 중복 방지
Py: set(nums)리스트를 집합으로! 중복이 알아서 사라짐
1줄=개수, 2줄=숫자들. 중복 3,1이 있어요!

집합 연산

합집합 · 교집합 · 차집합 — C는 이중 반복문

1세 가지 집합 연산∪ ∩ −

수학에서 배운 집합 연산이 그대로예요! 두 집합 A, B가 있을 때:

연산의미파이썬
합집합 (∪)A 또는 B에 있는 모든 원소A | B
교집합 (∩)A와 B 둘 다에 있는 원소A & B
차집합 (−)A에는 있고 B에는 없는 원소A - B
💡 파이썬은 기호 하나로! |(합), &(교), -(차). 메소드로도 A.union(B), A.intersection(B), A.difference(B)를 써요.
2C로 교집합 만들기이중 반복

C는 집합 연산이 없어서 이중 반복문으로 직접! 교집합은 "A의 각 원소가 B에도 있나?" 확인해요.

🔵 C — 교집합 직접 구현
// A의 각 원소가 B에 있으면 결과에 for (int i = 0; i < sizeA; i++) { for (int j = 0; j < sizeB; j++) { if (A[i] == B[j]) { result[rsize++] = A[i]; break; // 찾았으니 그만 } } }
🟢 Python — 기호 하나
A = {1, 2, 3, 4} B = {3, 4, 5, 6} print(A & B) # {3, 4} # 교집합 끝!
💡 합집합은 "A를 다 넣고, B에서 중복 아닌 것만 추가", 차집합은 "A 중 B에 없는 것만". 모두 선형 탐색 + 반복문의 조합이에요!
🖥 두 집합의 교집합 구하기
#include <stdio.h>

int main() {
    int A[100], B[100], na, nb;
    scanf("%d", &na);
    for (int i=0;i<na;i++) scanf("%d",&A[i]);
    scanf("%d", &nb);
    for (int i=0;i<nb;i++) scanf("%d",&B[i]);
    // 교집합: A의 원소가 B에도 있으면
    for (int i = 0; i < na; i++) {
        for (int j = 0; j < nb; j++) {
            if (A[i] == B[j]) {
                printf("%d ", A[i]);
                break;
            }
        }
    }
    printf("\n");
    return 0;
}
# 교집합 — 기호 하나!
na = int(input())
A = set(map(int, input().split()))
nb = int(input())
B = set(map(int, input().split()))

print(*sorted(A & B))   # 교집합!

# 합집합: A | B
# 차집합: A - B
C: 이중 forA[i]가 B에 있는지 하나씩 대조
A[i]==B[j] → 출력양쪽에 다 있으면 교집합 원소!
Py: A & B이 모든 걸 기호 하나로 압축
A={1,2,3,4}, B={3,4,5,6} → 교집합 3 4

활용

중복 제거·멤버십 검사·튜플 좌표 — 실전에서 빛나는 순간

1집합이 빛나는 순간빠른 검사

집합의 진짜 강점은 "들어있나?" 확인이 매우 빠르다는 거예요. 리스트보다 훨씬!

상황리스트 사용집합 사용
중복 제거직접 반복 검사set(리스트)
포함 확인처음부터 다 훑음(느림)바로 확인(빠름)
공통 원소이중 반복A & B
💡 파이썬 집합은 내부에 해시(hash)를 써서 x in s가 아주 빨라요. C 배열 흉내는 선형 탐색이라 느리지만, 원리를 이해하는 데는 딱이에요!
🖥 두 튜플(좌표) 사이 거리 계산
#include <stdio.h>

int main() {
    // 두 점 (x1,y1), (x2,y2)
    int x1, y1, x2, y2;
    scanf("%d %d %d %d",
          &x1, &y1, &x2, &y2);
    int dx = x1 - x2;
    int dy = y1 - y2;
    // 거리의 제곱 (루트 없이)
    int d2 = dx*dx + dy*dy;
    printf("거리제곱: %d\n", d2);
    return 0;
}
# 튜플로 좌표 다루기
x1, y1, x2, y2 = map(int, input().split())
p1 = (x1, y1)     # 튜플 좌표
p2 = (x2, y2)

dx = p1[0] - p2[0]
dy = p1[1] - p2[1]
d2 = dx*dx + dy*dy
print(f"거리제곱: {d2}")
Py: p1 = (x1, y1)좌표를 튜플로 묶어 의미를 명확히
p1[0], p1[1]튜플에서 x, y를 인덱스로 꺼냄
dx*dx + dy*dy거리의 제곱 (루트는 정수 아니라 생략)
x1 y1 x2 y2. 0 0 3 4 → 거리제곱 25 (=5²)
