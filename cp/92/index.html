DFS 한 줄씩 따라가기

재귀가 돌 때 visited 배열과 콜스택이 실제로 어떻게 변하는지 코드와 나란히

C 시각화: visited[] · 콜스택 · 코드 라인

DFS — 한 길로 끝까지 파고드는 탐색

깊이 우선 탐색(Depth-First Search)은 갈 수 있는 데까지 깊이 들어갔다가, 막다른 길을 만나면 되돌아와(백트래킹) 다른 길을 봅니다. 미로에서 한 손을 벽에 대고 걷는 것과 같아요.

이 강의는 그 과정에서 두 개의 배열이 실제로 어떻게 변하는지를 코드 한 줄 한 줄과 나란히 보여줍니다.

추적할 대상역할
visited[]각 정점을 방문했는지 0/1로 기록 — 무한 루프를 막는 핵심
콜스택지금 재귀가 얼마나 깊이 들어와 있는지 — 되돌아갈 경로
방문 순서정점이 처음 방문된 차례 — DFS의 결과
💡 이 강의의 목표는 "DFS가 판을 어떻게 훑는가"를 넘어 "visited[v]가 언제 0에서 1로 바뀌고, 스택이 언제 쌓이고 언제 풀리는가"를 눈에 각인시키는 것입니다.

이 강의의 예제 그래프

정점 6개, 간선 7개인 무방향 그래프입니다. 사이클이 있어서 visited의 역할이 뚜렷하게 드러납니다.

INPUT
6 7 정점 6, 간선 7 1 2 1 3 2 4 2 5 3 5 4 6 5 6

각 정점의 이웃은 번호가 작은 순서로 방문합니다(인접 리스트를 정렬). 그래서 1번에서 시작하면 2번을 3번보다 먼저 봅니다.

손으로 미리 따라가 보기

1번에서 시작합니다. 규칙은 딱 하나 — "방문 안 한 가장 작은 이웃으로 계속 들어가고, 없으면 되돌아온다."

순서동작이유
11 방문 → 2로1의 이웃 {2,3} 중 작은 2
22 방문 → 4로2의 이웃 {1,4,5}, 1은 방문됨 → 4
34 방문 → 6으로4의 이웃 {2,6}, 2 방문됨 → 6
46 방문 → 5로6의 이웃 {4,5}, 4 방문됨 → 5
55 방문 → 3으로5의 이웃 {2,3,6}, 2·6 방문됨 → 3
63 방문 → 막힘, 되돌아감3의 이웃 {1,5} 모두 방문됨
최종 방문 순서
DFS(1) = 1 → 2 → 4 → 6 → 5 → 3
⚠️ 6번에서 5로 갈 때 주목하세요. 6의 이웃은 4와 5인데 4는 이미 방문했으니 5로 내려갑니다. 그리고 3까지 간 뒤 더 갈 곳이 없어 스택을 타고 1번까지 쭉 되돌아옵니다. 다음 탭부터 이 과정을 배열과 함께 봅니다.

DFS 코드 — 딱 다섯 줄의 핵심

재귀 DFS는 놀랍도록 짧습니다. 각 줄이 무슨 일을 하는지 색으로 짚어 봅니다.

C — 재귀 DFS
int visited[105];  /* 전역 = 자동 0 */

void dfs(int u) {
    visited[u] = 1;       /* ① 방문 표시 */
    printf("%d ", u);       /* ② 처리(출력) */
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];  /* ③ 이웃 하나 */
        if (!visited[v])     /* ④ 안 갔으면 */
            dfs(v);          /* ⑤ 깊이 들어감 */
    }
}   /* 함수 끝 = 되돌아감(백트래킹) */
각 줄의 의미
줄하는 일
①visited[u]=1 — u를 "방문함"으로. 진입 즉시 표시가 철칙
②정점을 처리 — 여기선 출력. 방문 순서가 여기서 기록됨
③u의 이웃 목록을 하나씩 꺼냄 (작은 번호부터)
④그 이웃이 아직 방문 안 됐는지 확인
⑤안 됐으면 그리로 재귀 진입 — 스택이 한 칸 깊어짐
끝이웃을 다 보면 함수 종료 → 부른 곳으로 복귀
⚠️ ①의 위치가 생명 — 방문 표시를 함수 맨 앞에 둬야 합니다. 만약 ⑤ 뒤에 두면, 같은 정점이 여러 번 스택에 들어가 무한 재귀에 빠집니다. "들어오자마자 도장 찍기"를 기억하세요.

visited 배열이 없으면?

사이클이 있는 그래프에서 visited를 빼면 어떻게 될까요? 1→2→4→6→5→2→4→6→... 영원히 맴돕니다. 트리와 달리 그래프는 되돌아오는 길이 있기 때문입니다.

0 = 미방문 1 = 방문 완료 지금 막 바뀐 칸
C에서 visited 초기화 — 전역 배열은 자동으로 0이라 그냥 쓸 수 있습니다. 하지만 여러 번 DFS를 돌린다면(연결 요소 세기 등) 매번 for나 memset으로 0으로 밀어야 합니다. 지역 배열이라면 반드시 초기화하세요.

재귀 대신 스택으로 — 같은 일, 다른 모습

재귀 DFS는 사실 컴퓨터가 콜스택이라는 숨은 스택을 자동으로 관리해 주는 것입니다. 그 스택을 내 손으로 만들면 재귀 없이 DFS를 할 수 있어요. 파이썬 재귀 한계나 C 스택 오버플로를 피할 때 씁니다.

재귀 — 콜스택은 자동
void dfs(int u) {
    visited[u] = 1;
    printf("%d ", u);
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (!visited[v]) dfs(v);
    }
}
스택 — 내 손으로 관리
void dfs_iter(int s) {
    int st[MAXN], top = 0;
    st[top++] = s;              /* 시작점 push */
    while (top > 0) {
        int u = st[--top];      /* pop */
        if (visited[u]) continue;  /* 중복 방지 */
        visited[u] = 1;
        printf("%d ", u);
        /* 작은 번호를 먼저 꺼내려면 역순 push */
        for (int k = deg[u]-1; k >= 0; k--) {
            int v = adjv[u][k];
            if (!visited[v]) st[top++] = v;
        }
    }
}
⚠️ 스택 DFS의 두 가지 함정 — ① 방문 표시는 pop할 때 해야 재귀와 순서가 같습니다(push 때 하면 순서가 뒤틀림). ② 그래서 같은 정점이 스택에 여러 번 들어갈 수 있어 if(visited[u]) continue;가 필수입니다. ③ 작은 번호부터 방문하려면 이웃을 역순으로 push합니다(스택은 LIFO라 나중에 넣은 게 먼저 나옴).

DFS로 연결 요소 세기

DFS 한 번은 한 덩어리를 전부 방문합니다. 그래서 모든 정점을 훑으며 미방문 정점이 나오면 DFS를 새로 시작하면, 그 시작 횟수가 곧 연결 요소의 개수입니다.

C — 연결 요소 개수
int cnt = 0;
for (int s = 1; s <= n; s++) {
    if (!visited[s]) {
        cnt++;             /* 새 덩어리 발견 */
        dfs(s);            /* 통째로 방문 */
    }
}
printf("연결 요소: %d\n", cnt);
C — 각 덩어리 크기
int sz;
void dfs_size(int u) {
    visited[u] = 1;
    sz++;              /* 이 덩어리에 +1 */
    for (int k=0;k<deg[u];k++){
        int v=adjv[u][k];
        if(!visited[v]) dfs_size(v);
    }
}
/* sz=0; dfs_size(s); 로 크기 얻음 */

DFS 활용 지도

DFS는 단순한 순회를 넘어 수많은 문제의 뼈대입니다.

활용아이디어
연결 요소미방문 정점마다 DFS 시작 → 시작 횟수 = 요소 수
사이클 판별방문 중인 정점을 다시 만나면 사이클 존재
경로 존재 여부u에서 DFS 돌려 v가 방문되면 경로 있음
미로 탈출격자를 그래프로 보고 출구까지 DFS
위상 정렬DFS 종료(exit) 순서의 역순 (⑮편)
트리 순회전위·중위·후위가 모두 DFS의 변형
백트래킹N-Queen, 순열 생성 — DFS + 되돌리기
💡 BFS와의 선택 — "가중치 1 최단 거리"는 BFS, "모든 경우 탐색·백트래킹·위상 정렬"은 DFS가 자연스럽습니다. 판을 훑는다는 점은 같지만 깊이 우선이냐 너비 우선이냐가 문제 성격을 가릅니다.

격자에서의 DFS — 상하좌우 4방향

2차원 격자(미로, 섬 세기)에서는 정점이 칸 (r, c)이고, 이웃은 상하좌우 네 칸입니다. 방향 배열로 간결하게 씁니다.

C — 격자 DFS (섬 세기)
int dr[] = {-1, 1, 0, 0};  /* 상하 */
int dc[] = {0, 0, -1, 1};  /* 좌우 */

void dfs(int r, int c) {
    visited[r][c] = 1;
    for (int d = 0; d < 4; d++) {
        int nr = r + dr[d], nc = c + dc[d];
        if (nr < 0 || nr >= H) continue;
        if (nc < 0 || nc >= W) continue;
        if (visited[nr][nc]) continue;
        if (grid[nr][nc] == 0) continue; /* 바다 */
        dfs(nr, nc);
    }
}
