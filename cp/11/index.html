간선에 "무게"가 생겼다 — 가중 그래프

지금까지의 그래프는 간선이 있다 / 없다만 중요했습니다. 하지만 실제 세상의 연결에는 대부분 비용이 붙어 있죠. 도로의 길이, 버스 요금, 걸리는 시간… 이렇게 간선마다 숫자가 붙은 그래프를 가중 그래프(weighted graph), 그 숫자를 가중치(weight)라고 부릅니다.

비교BFS (⑪편)다익스트라 (이번 편)
최단의 기준지나는 간선 개수지나는 간선들의 가중치 합
쓰는 도구큐 (deque)우선순위 큐 (최소 힙)
가중치 조건모두 1 (또는 동일)0 이상 아무 값 (단, 음수 금지)
시간복잡도O(V+E)O(E log V)
🔗 이전 편 연결 — ⑪편 BFS에서 dist[v] = dist[u] + 1로 거리를 구했죠? 다익스트라는 그 +1이 +w(간선 가중치)로 바뀐 것뿐입니다. 다만 "가까운 순서"가 뒤죽박죽되기 때문에 큐 대신 우선순위 큐가 필요해집니다.

BFS로는 왜 안 될까?

아래 미니 그래프에서 1번 → 2번 최단 거리를 생각해 봅시다.

BFS의 답 — "직행 간선 1개가 제일 빨라!" → 거리 3

진짜 최단 — 3번을 거쳐 돌아가면 1 + 1 = 2

간선 개수는 많아도 가중치 합이 더 작을 수 있습니다. BFS는 이런 "싸게 돌아가는 길"을 놓칩니다. 그래서 새 알고리즘이 필요합니다 — 1956년 네덜란드의 컴퓨터과학자 에츠허르 다익스트라(Edsger Dijkstra)가 커피숍에서 20분 만에 떠올렸다는 바로 그 방법입니다.

다익스트라의 핵심 아이디어 — "확실한 것부터 확정"

규칙은 단 세 줄입니다:

순서규칙
①아직 확정되지 않은 정점 중 dist가 가장 작은 정점 u를 고른다
②u를 확정한다 — 이 값은 앞으로 절대 바뀌지 않는다!
③u의 이웃 v마다 dist[u] + w < dist[v]이면 dist[v]를 갱신한다 — 이것이 완화(relaxation)
💡 왜 최솟값은 확정해도 될까? 모든 가중치가 0 이상이라면, 지금 dist가 가장 작은 정점 u로 "다른 길로 돌아서" 더 싸게 도착하는 건 불가능합니다. 돌아가는 길은 u보다 dist가 큰 정점을 거쳐야 하는데, 거기서 출발해 봤자 더 커지기만 하니까요.
⚠️ 음수 가중치 금지! 위 논리는 "가중치 ≥ 0"일 때만 성립합니다. 음수 간선이 하나라도 있으면 다익스트라의 답은 틀릴 수 있습니다 (그때는 벨만-포드 알고리즘을 씁니다 — 고급 편 예고).

/* adjv[u][k] : u의 k번째 이웃
   adjw[u][k] : 그 간선의 가중치 */
int adjv[101][101], adjw[101][101], deg[101];

scanf("%d %d", &n, &m);
for (int i = 0; i < m; i++) {
    int u, v, w;
    scanf("%d %d %d", &u, &v, &w);
    adjv[u][deg[u]] = v; adjw[u][deg[u]] = w; deg[u]++;
    adjv[v][deg[v]] = u; adjw[v][deg[v]] = w; deg[v]++;
}

다익스트라 한 걸음씩 — 확정과 완화

출발점의 dist는 0, 나머지는 전부 ∞ (무한대)로 시작합니다. 그리고 세 규칙을 반복합니다: ① 미확정 중 최소 dist 정점 선택 → ② 확정 → ③ 이웃 완화. 모든 정점이 확정되면 끝!

말로는 어렵죠? 아래 시뮬레이터에서 dist 표가 ∞에서 하나씩 줄어드는 과정을 직접 보세요. 특히 2번 정점의 dist가 3 → 2로 갱신되는 순간(우회로 발견!)과 8번이 10 → 7로 갱신되는 순간을 놓치지 마세요.

1단계 — 기본형 O(V²) : 원리 그대로 옮기기

시뮬레이터에서 본 것을 그대로 코드로 씁니다. "미확정 중 최솟값 찾기"를 for문으로 전부 훑어서 처리하는, 가장 정직한 버전입니다. 정점이 수천 개 이하라면 이걸로 충분합니다.

C — 기본형 다익스트라 O(V²)
#include <stdio.h>
#define INF 1000000000

int n, m;
int adjv[101][101], adjw[101][101], deg[101];
int dist[101], done[101], pre[101];

int main(void) {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d %d %d", &u, &v, &w);
        adjv[u][deg[u]] = v; adjw[u][deg[u]] = w; deg[u]++;
        adjv[v][deg[v]] = u; adjw[v][deg[v]] = w; deg[v]++;
    }
    int s = 1;                        /* 출발 정점 */
    for (int i = 1; i <= n; i++) dist[i] = INF;
    dist[s] = 0;

    for (int t = 0; t < n; t++) {
        int u = 0;                    /* ① 미확정 중 최소 */
        for (int i = 1; i <= n; i++)
            if (!done[i] && (u == 0 || dist[i] < dist[u])) u = i;
        if (u == 0 || dist[u] == INF) break;
        done[u] = 1;                  /* ② u 확정! */
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k], w = adjw[u][k];
            if (dist[u] + w < dist[v]) {  /* ③ 완화 */
                dist[v] = dist[u] + w;
                pre[v] = u;           /* 경로 복원용 */
            }
        }
    }
    for (int i = 1; i <= n; i++)
        printf("dist[%d] = %d\n", i, dist[i]);
    return 0;
}
Python — 기본형 다익스트라 O(V²)
import sys
input = sys.stdin.readline
INF = float('inf')

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v, w = map(int, input().split())
    adj[u].append((v, w))
    adj[v].append((u, w))

s = 1                          # 출발 정점
dist = [INF] * (n + 1)
done = [False] * (n + 1)
pre  = [0] * (n + 1)
dist[s] = 0

for _ in range(n):
    u, best = 0, INF           # ① 미확정 중 최소
    for i in range(1, n + 1):
        if not done[i] and dist[i] < best:
            u, best = i, dist[i]
    if u == 0:                 # 남은 정점에 못 감
        break
    done[u] = True             # ② 확정
    for v, w in adj[u]:        # ③ 완화
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            pre[v] = u             # 경로 복원용

for i in range(1, n + 1):
    print(f'dist[{i}] = {dist[i]}')

2단계 — 실전형 O(E log V) : 우선순위 큐

정점이 10만 개라면 "전부 훑어 최소 찾기"는 너무 느립니다. 최소 힙(우선순위 큐)에게 "제일 작은 것 꺼내기"를 맡기면 매번 log 시간에 끝납니다. Python은 heapq 한 줄, C는 힙을 직접 만듭니다 (힙 코드는 한 번 이해해 두면 평생 재사용!).

C — 최소 힙 다익스트라 O(E log V)
#include <stdio.h>
#define INF 1000000000
#define MAXH 400005

int n, m;
int adjv[101][101], adjw[101][101], deg[101];
int dist[101];

/* ---- 최소 힙: (거리 hd, 정점 hv) 쌍 ---- */
int hd[MAXH], hv[MAXH], hs;

void swap2(int a, int b) {
    int t;
    t = hd[a]; hd[a] = hd[b]; hd[b] = t;
    t = hv[a]; hv[a] = hv[b]; hv[b] = t;
}
void push(int d, int v) {
    int i = ++hs;
    hd[i] = d; hv[i] = v;
    while (i > 1 && hd[i] < hd[i / 2]) {  /* 부모보다 작으면 위로 */
        swap2(i, i / 2);
        i /= 2;
    }
}
void pop(void) {
    hd[1] = hd[hs]; hv[1] = hv[hs]; hs--;
    int i = 1;
    while (1) {                          /* 작은 자식과 교환하며 아래로 */
        int c = i * 2;
        if (c > hs) break;
        if (c + 1 <= hs && hd[c + 1] < hd[c]) c++;
        if (hd[i] <= hd[c]) break;
        swap2(i, c);
        i = c;
    }
}

int main(void) {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v, w;
        scanf("%d %d %d", &u, &v, &w);
        adjv[u][deg[u]] = v; adjw[u][deg[u]] = w; deg[u]++;
        adjv[v][deg[v]] = u; adjw[v][deg[v]] = w; deg[v]++;
    }
    int s = 1;
    for (int i = 1; i <= n; i++) dist[i] = INF;
    dist[s] = 0;
    push(0, s);

    while (hs > 0) {
        int d = hd[1], u = hv[1];
        pop();
        if (d > dist[u]) continue;   /* 낡은 기록 스킵 */
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k], w = adjw[u][k];
            if (d + w < dist[v]) {
                dist[v] = d + w;
                push(dist[v], v);
            }
        }
    }
    for (int i = 1; i <= n; i++)
        printf("dist[%d] = %d\n", i, dist[i]);
    return 0;
}
Python — heapq 다익스트라 O(E log V)
import heapq, sys
input = sys.stdin.readline
INF = float('inf')

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v, w = map(int, input().split())
    adj[u].append((v, w))
    adj[v].append((u, w))

s = 1
dist = [INF] * (n + 1)
dist[s] = 0
pq = [(0, s)]        # (거리, 정점) 순서가 중요!

while pq:
    d, u = heapq.heappop(pq)   # 최소 거리부터 나옴
    if d > dist[u]:            # 낡은 기록 스킵
        continue
    for v, w in adj[u]:
        nd = d + w
        if nd < dist[v]:           # 완화
            dist[v] = nd
            heapq.heappush(pq, (nd, v))

for i in range(1, n + 1):
    print(f'dist[{i}] = {dist[i]}')
💡 (거리, 정점) 순서인 이유 — heapq는 튜플의 첫 번째 원소부터 비교합니다. 거리를 앞에 둬야 "거리가 가장 작은 것"이 먼저 나옵니다. (정점, 거리) 순서로 넣는 실수가 정말 많아요!
⚠️ if d > dist[u]: continue를 빼먹으면? 같은 정점이 힙에 여러 번 들어갈 수 있는데(더 짧은 길을 찾을 때마다 push하니까), 이미 확정된 정점의 낡은(더 큰) 기록이 나중에 튀어나옵니다. 이 한 줄이 없으면 답은 맞아도 시간 초과가 나는 문제가 많습니다. 이 줄이 사실상 O(V²) 버전의 done[] 역할입니다.
버전최소 찾기시간복잡도추천 상황
기본형for문 전체 훑기O(V²)V ≤ 5,000 · 원리 학습
실전형최소 힙 popO(E log V)V가 큰 실전 문제 전부

거리 말고 "길"도 알고 싶다 — pre 배열

KOI 문제는 종종 "최단 거리"가 아니라 "최단 경로를 출력하라"고 요구합니다. 방법은 간단합니다 — 완화에 성공할 때마다 "v에 오기 직전 정점은 u였다"를 pre[v]에 적어 두면, 도착점에서 pre를 따라 거꾸로 거슬러 올라가 출발점까지 갈 수 있습니다.

C — 재귀로 뒤집어 출력
/* 완화 성공 시: pre[v] = u; (3번 탭 코드 참고) */

void print_path(int v) {
    if (v == 0) return;   /* pre[출발점]=0에서 멈춤 */
    print_path(pre[v]);   /* 먼저 앞부분을 출력 */
    printf("%d ", v);
}

/* main 마지막에: */
printf("1 -> 8 최단 경로: ");
print_path(8);        /* 1 3 6 8 */
Python — while로 모아서 뒤집기
# 완화 성공 시: pre[v] = u  (3번 탭 코드 참고)

def path(v):
    p = []
    while v:              # pre[출발점]=0에서 멈춤
        p.append(v)
        v = pre[v]
    return p[::-1]        # 뒤집어서 출발→도착 순서로

print('1 -> 8 최단 경로:', *path(8))  # 1 3 6 8
💡 재귀 DFS에서 "콜스택이 자동으로 되돌아온다"를 배웠죠? C의 print_path가 딱 그 원리입니다 — 재귀가 풀리면서 자동으로 순서가 뒤집힙니다.
