그래프란 무엇일까?

그래프(Graph)는 정점(vertex, 노드)과 정점 사이를 잇는 간선(edge)으로 이루어진 자료구조입니다. 친구 관계, 지하철 노선도, 미로 — 무언가가 연결되어 있는 모든 상황이 그래프입니다.

용어뜻
정점 V동그라미 하나 (위 그림엔 8개)
간선 E정점을 잇는 선 (위 그림엔 8개)
인접간선으로 직접 연결된 사이 (1과 2는 인접)
탐색한 정점에서 출발해 연결된 모든 정점을 빠짐없이, 딱 한 번씩 방문하는 것
📎 이전 편 연결 — 2차원 배열 편에서 배운 dx/dy 이동은 사실 "격자 그래프 탐색"이었습니다. 4단원에서 다시 만납니다!

DFS vs BFS — 미로에서 길 찾는 두 가지 성격

탐색 방법은 딱 두 가지 성격으로 나뉩니다. 같은 그래프라도 방문 순서가 달라집니다.

DFS (깊이 우선 탐색)BFS (너비 우선 탐색)
성격🕳️ 한 길을 끝까지 파고들고, 막히면 되돌아온다🌊 가까운 곳부터 물결처럼 한 겹씩 퍼진다
사용 도구재귀 (스택)큐 (queue)
잘하는 일연결 영역 세기, 사이클 찾기, 백트래킹최단 거리 (간선 개수 기준), 레벨 탐색
시간 복잡도둘 다 O(V + E) — 정점·간선을 한 번씩만 보기 때문
💡 딱 하나만 외운다면 — "DFS는 스택(재귀), BFS는 큐". 어떤 자료구조를 쓰느냐가 두 탐색의 유일한 차이입니다.

탐색의 생명줄: visited 배열

그래프에는 사이클(빙글빙글 도는 길)이 있을 수 있어서, "이미 가본 곳"을 기억하지 않으면 같은 곳을 무한히 맴돌게 됩니다.

// 방문할 때:
visited[u] = 1;

// 가기 전에 확인:
if (!visited[v]) { /* 방문 */ }
Python

# 방문할 때:

# 가기 전에 확인:

⚠️ visited를 빼먹으면? 1 → 2 → 1 → 2 → … 무한 반복! DFS는 스택 오버플로, BFS는 큐가 터집니다. 그래프 탐색 오답의 1등 원인입니다.

이번 편의 실험용 그래프

모든 단원에서 아래 그래프 하나로 실험합니다. 정점 8개, 간선 8개 — 사이클(5–8–6–3–1–2–5)도 하나 숨어 있습니다.

INPUT — 첫 줄: 정점 수 N, 간선 수 M / 다음 M줄: 간선 (u v)
8 8 1 2 1 3 2 4 2 5 3 6 3 7 5 8 6 8

같은 정점에서 출발해도 DFS는 1 2 4 5 8 6 3 7, BFS는 1 2 3 4 5 6 7 8 순으로 방문합니다. 왜 그런지 2·3단원 시뮬레이터에서 한 걸음씩 확인해 보세요.

컴퓨터에 그래프를 담는 두 가지 방법

인접행렬 (2차원 배열)인접리스트
저장 방식adj[u][v] = 1 이면 연결adj[u] = u의 이웃 목록
메모리O(V²) — 정점 많으면 부담O(V + E) — 간선만큼만
"u와 v가 연결?"O(1) 즉시목록을 훑어야 함
"u의 이웃 전부"V칸을 다 확인이웃 수만큼만 — 탐색에 유리
추천정점 ≤ 1,000 정도의 소형 문제대부분의 KOI 탐색 문제

인접리스트 만들기 — 간선 입력받기

C에는 파이썬의 리스트 같은 가변 배열이 없으므로, 2차원 배열 + 개수 배열(deg)로 인접리스트를 흉내 냅니다. 2차원 배열 편에서 배운 기술이 그대로 쓰입니다.

C — adj[][] + deg[]
#include <stdio.h>
int adj[101][101]; // adj[u][k] = u의 k번째 이웃
int deg[101];      // deg[u]   = u의 이웃 수
int n, m;

int main() {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v;
        scanf("%d %d", &u, &v);
        adj[u][deg[u]++] = v;  // u 목록 끝에 v 추가
        adj[v][deg[v]++] = u;  // 양방향이므로 반대도!
    }
    // 1번 정점의 이웃 출력
    for (int k = 0; k < deg[1]; k++)
        printf("%d ", adj[1][k]);
    return 0;
}
Python — 리스트의 리스트

    graph[v].append(u)  # 양방향이므로 반대도!

# 1번 정점의 이웃 출력
print(*graph[1])

⚠️ 양방향 간선 실수 — "1 2"라는 간선은 1→2와 2→1 둘 다 넣어야 합니다. 한 줄만 넣으면 탐색이 절반만 됩니다. (방향 그래프 문제라면 한 줄만!)
💡 번호 순서로 방문하고 싶다면 — 파이썬은 입력 후 graph[u].sort(), C는 이웃을 넣은 뒤 각 행을 정렬하면 됩니다. 정올 문제는 "작은 번호부터 방문"을 자주 요구합니다.

DFS — 한 길로 끝까지, 막히면 되돌아오기

DFS(Depth-First Search)는 "갈 수 있으면 더 깊이, 막히면 한 칸 후퇴"를 반복합니다. 이 되돌아오기를 공짜로 해 주는 것이 바로 재귀 호출(콜스택)입니다.

규칙은 단 세 줄입니다:

순서규칙
①정점 u에 도착하면 방문 도장을 찍는다 (visited[u] = 1)
②u의 이웃 v를 차례로 보며, 아직 안 가본 곳이면 dfs(v) — 더 깊이!
③모든 이웃이 끝나면 함수가 끝난다 — 자동으로 이전 정점으로 복귀

재귀 DFS — 두 언어 나란히

C — 재귀 DFS
#include <stdio.h>
int adj[101][101], deg[101];
int visited[101];
int n, m;

void dfs(int u) {
    visited[u] = 1;            // ① 방문 도장
    printf("%d ", u);
    for (int k = 0; k < deg[u]; k++) {
        int v = adj[u][k];
        if (!visited[v])
            dfs(v);           // ② 더 깊이!
    }
}                              // ③ 자동 복귀

int main() {
    scanf("%d %d", &n, &m);
    for (int i = 0; i < m; i++) {
        int u, v;
        scanf("%d %d", &u, &v);
        adj[u][deg[u]++] = v;
        adj[v][deg[v]++] = u;
    }
    dfs(1);
    return 0;
}
Python — 재귀 DFS

sys.setrecursionlimit(10**6)  # 필수 보험!

    print(u, end=' ')
    for v in graph[u]:

BFS — 가까운 곳부터 물결처럼

BFS(Breadth-First Search)는 시작점에서 거리 1인 곳을 전부, 그다음 거리 2인 곳을 전부 … 물결처럼 퍼져 나갑니다. "먼저 발견한 곳을 먼저 처리"해야 하므로 큐(선입선출)가 필요합니다.

순서규칙
①시작 정점을 큐에 넣고 방문 도장을 찍는다
②큐에서 하나 꺼낸다(u) → u의 이웃 중 안 가본 v를 전부 큐에 넣으며 도장
③큐가 빌 때까지 ②를 반복
⚠️ 도장은 "꺼낼 때"가 아니라 "넣을 때"! — 넣을 때 체크하지 않으면 같은 정점이 큐에 여러 번 들어가 시간·메모리가 폭발합니다. BFS 최다 실수 1위.

BFS 구현 — C는 배열 큐, 파이썬은 deque

C에는 큐가 없으므로 배열 + front/rear 두 개의 손가락으로 직접 만듭니다. 파이썬은 collections.deque가 O(1) 큐를 제공합니다.

C — 배열로 만든 큐
#include <stdio.h>
int adj[101][101], deg[101];
int visited[101], dist[101];
int queue[101], front, rear;  // 배열 큐
int n, m;

void bfs(int start) {
    queue[rear++] = start;     // ① 넣기
    visited[start] = 1;
    while (front < rear) {     // ③ 빌 때까지
        int u = queue[front++]; // ② 꺼내기
        printf("%d ", u);
        for (int k = 0; k < deg[u]; k++) {
            int v = adj[u][k];
            if (!visited[v]) {
                visited[v] = 1;        // 넣을 때 도장!
                dist[v] = dist[u] + 1; // 최단거리
                queue[rear++] = v;
            }
        }
    }
}
Python — collections.deque
from collections import deque

n, m = map(int, input().split())
graph = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v = map(int, input().split())
    graph[u].append(v)
    graph[v].append(u)

visited = [False] * (n + 1)
dist = [0] * (n + 1)

def bfs(start):
    q = deque([start])         # ① 넣기
    visited[start] = True
    while q:                   # ③ 빌 때까지
        u = q.popleft()        # ② 꺼내기
        print(u, end=' ')
        for v in graph[u]:
            if not visited[v]:
                visited[v] = True       # 넣을 때 도장!
                dist[v] = dist[u] + 1   # 최단거리
                q.append(v)

bfs(1)
OUTPUT — 실험용 그래프, 1번에서 출발

💡 BFS = 공짜 최단거리 — 모든 간선의 길이가 같다면(=1), BFS가 처음 도착한 순간이 곧 최단거리입니다. dist[v] = dist[u] + 1 한 줄이면 끝. 정올 최단거리 문제의 90%는 BFS입니다.

격자도 그래프다 — dx/dy의 귀환

📎 2차원 배열 편 복습 — 각 칸을 정점, 상하좌우 이웃을 간선이라고 생각하면 격자는 그냥 그래프입니다. 그때 배운 dx/dy가 "이웃 목록" 역할을 합니다.
C — 방향 배열
//            상  하  좌  우
int dx[4] = {-1, 1, 0, 0};
int dy[4] = { 0, 0,-1, 1};

for (int d = 0; d < 4; d++) {
    int nx = x + dx[d];
    int ny = y + dy[d];
    // 격자 밖이면 통과!
    if (nx < 0 || nx >= n ||
        ny < 0 || ny >= n) continue;
    // 여기서 방문 처리
}
Python — 방향 배열
#       상  하  좌  우

⚠️ 경계 검사를 먼저! — 배열 밖 접근은 C에선 알 수 없는 값(런타임 에러), 파이썬에선 음수 인덱스가 반대쪽 끝을 가리키는 조용한 버그가 됩니다. "밖이면 continue"가 언제나 첫 줄!

대표 패턴 ① — Flood Fill: 연결 영역 개수 세기 (DFS)

"색칠된 덩어리가 몇 개인가?" — 아직 방문 안 한 색칠 칸을 만날 때마다 DFS로 덩어리 전체를 도장 찍고 개수를 +1 합니다. (정올 단골: 그림, 단지번호붙이기 유형)

C — flood fill
int grid[101][101], visited[101][101];
int dx[4]={-1,1,0,0}, dy[4]={0,0,-1,1};
int n;

void dfs(int x, int y) {
    visited[x][y] = 1;
    for (int d = 0; d < 4; d++) {
        int nx = x+dx[d], ny = y+dy[d];
        if (nx<0||nx>=n||ny<0||ny>=n) continue;
        if (grid[nx][ny]==1 && !visited[nx][ny])
            dfs(nx, ny);
    }
}

// main에서: 덩어리 개수 세기
int count = 0;
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        if (grid[i][j]==1 && !visited[i][j]) {
            dfs(i, j);   // 덩어리 전체 도장
            count++;     // 새 덩어리 발견!
        }
Python — flood fill
import sys

            count += 1  # 새 덩어리 발견!

대표 패턴 ② — 미로 최단거리 (BFS)

"(0,0)에서 (n-1,m-1)까지 몇 칸 만에?" — 벽(1)을 피해 BFS 물결을 퍼뜨리면, 도착 칸의 dist가 곧 정답입니다. (정올 단골: 미로 탐색 유형)

Python — 미로 BFS (C도 3단원 배열 큐에 튜플 대신 x·y 두 큐를 쓰면 동일)
from collections import deque

def bfs(sx, sy):
    q = deque([(sx, sy)])
    dist[sx][sy] = 0
    while q:
        x, y = q.popleft()
        for d in range(4):
            nx, ny = x + dx[d], y + dy[d]
            if not (0 <= nx < n and 0 <= ny < m): continue   # 격자 밖
            if grid[nx][ny] == 1: continue                     # 벽
            if dist[nx][ny] != -1: continue                    # 이미 방문
            dist[nx][ny] = dist[x][y] + 1                      # 최단거리 확정!
            q.append((nx, ny))
💡 DFS로 최단거리를 구하면 안 되나요? — 안 됩니다! DFS는 "처음 도착한 길"이 최단이라는 보장이 없습니다. 최단거리는 무조건 BFS. 이게 이번 편에서 가장 중요한 한 문장입니다.
