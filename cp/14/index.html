순서가 있는 일들 — 위상 정렬

대학 수강신청을 떠올려 보세요. "자료구조"를 들으려면 "프로그래밍기초"를 먼저 들어야 하고, "알고리즘"을 들으려면 "자료구조"가 선행되어야 합니다. 이렇게 선후 관계가 있는 일들을 규칙을 어기지 않는 하나의 순서로 줄 세우는 것이 위상 정렬입니다.

화살표 u → v는 "u를 먼저 해야 v를 할 수 있다"는 뜻입니다. 위상 정렬은 모든 화살표가 왼쪽→오른쪽을 향하도록 정점을 일렬로 세우는 것과 같습니다.

💡 어디에 쓰나? 과목 선수관계, 프로젝트 작업 순서(간트 차트), 소스 파일 컴파일 순서(Makefile), 스프레드시트 수식 계산 순서, 게임 기술 트리… "먼저/나중"이 있는 곳엔 어디든 위상 정렬이 숨어 있습니다.

두 가지 전제 — 방향과 무사이클 (DAG)

위상 정렬이 가능하려면 그래프가 반드시 DAG여야 합니다.

DAG의 조건뜻
Directed (방향)화살표에 방향이 있다 — "누가 먼저"가 정해짐
Acyclic (무사이클)순환 고리가 없다
+ Graph그래프
⚠️ 사이클이 있으면 위상 정렬 불가! A→B→C→A라면 "A가 먼저? C가 먼저?" 답이 없습니다. 뱀이 자기 꼬리를 무는 셈이죠. 뒤에서 배울 두 알고리즘은 이 사이클을 자동으로 감지합니다.
🔗 ⑪편 연결 — DFS와 큐(BFS의 도구)가 다시 등장합니다. 위상 정렬은 사실상 "순서를 기억하는 특별한 탐색"이에요.

정답은 하나가 아니다

중요한 성질: 유효한 위상 순서는 여러 개일 수 있습니다. 선후 관계만 지키면 되니까요. 예를 들어 "양말과 셔츠는 아무 순서나 먼저 신어도 되지만, 둘 다 신발보다는 먼저"인 것과 같습니다.

그래서 두 대표 알고리즘이 서로 다른(그러나 둘 다 올바른) 순서를 내놓기도 합니다. 이 편에서 직접 확인하게 됩니다.

실험용 DAG — 선수과목 그래프

이번 편의 예제입니다. 정점 8개, 방향 간선 10개. 화살표 방향(먼저→나중)에 주목하세요.

진입차수(들어오는 화살표 수)를 눈으로 세어 보세요. 1번과 2번은 들어오는 화살표가 0개 — 아무 선수과목 없이 바로 들을 수 있는 과목입니다. 여기서부터 순서가 시작됩니다.

INPUT — 방향 간선 u→v (u 먼저, v 나중)
8 10 1 3 2 3 3 4 3 5 2 5 4 6 5 6 5 7 6 8 7 8

진입차수 방식 (Kahn 알고리즘) — "선수과목 다 들은 것부터"

가장 직관적인 방법입니다. 진입차수(in-degree)는 "그 정점으로 들어오는 화살표의 개수" = "아직 안 들은 선수과목의 수"입니다. 규칙은 세 줄:

순서규칙
①진입차수가 0인 정점(선수과목 없음)을 모두 큐에 넣는다
②큐에서 하나 꺼내 순서에 추가 — 그 정점이 가리키는 이웃들의 진입차수를 1씩 감소
③감소하다 0이 된 정점을 큐에 넣는다 — 큐가 빌 때까지 반복
💡 "이번 과목을 다 들었으니, 이걸 선수과목으로 하던 과목들의 남은 선수과목 수가 하나 줄어든다." 그러다 남은 선수과목이 0이 되면 그 과목도 이제 들을 수 있게 되는 것 — 딱 그 논리입니다.
🔗 ⑪편 BFS 연결 — 큐를 쓰는 구조가 BFS와 똑같습니다. 다만 "방문했나"가 아니라 "진입차수가 0이 됐나"로 큐에 넣는 조건이 바뀐 것뿐이에요.

Kahn 스텝 시뮬레이터

DFS 방식 — "가장 나중에 끝나는 것이 맨 앞"

두 번째 방법은 DFS를 이용합니다. 핵심 통찰: DFS로 깊이 들어갔다가 함수가 끝나는(빠져나오는) 순서를 기록하면, 그 역순이 위상 정렬입니다.

순서규칙
①모든 정점에 대해 아직 방문 안 했으면 DFS 시작
②이웃을 모두 재귀 방문한 뒤에, 자신을 리스트에 추가 (후위 기록)
③완성된 리스트를 뒤집으면 위상 순서!
💡 왜 역순일까? "가장 나중에 끝난 정점"은 자신에게 딸린 후속 작업이 하나도 없다는 뜻 = 가장 먼저 해도 되는 일. 그래서 후위 순서를 뒤집으면 자연스럽게 "먼저 할 것 → 나중 할 것" 순서가 됩니다.
🔗 ⑪편 DFS 연결 — 재귀 DFS에서 "이웃을 다 본 뒤 함수가 끝나며 되돌아온다"를 배웠죠? 바로 그 "끝나는 순간"을 기록하는 것뿐입니다.

DFS 위상 정렬 시뮬레이터

Kahn 방식 — 큐 + 진입차수 배열

C — Kahn 위상 정렬
#include <stdio.h>

int n, m;
int adjv[101][101], deg[101];  /* 인접 리스트 */
int indeg[101];                /* 진입차수 */
int queue[101], head, tail;
int order[101], oc;

int main(void) {
    scanf("%d %d", &n, &m);
    for (int e = 0; e < m; e++) {
        int u, v;
        scanf("%d %d", &u, &v);   /* u -> v */
        adjv[u][deg[u]++] = v;
        indeg[v]++;                /* v로 들어오는 화살표 +1 */
    }
    /* ① 진입차수 0을 큐에 (작은 번호부터) */
    for (int i = 1; i <= n; i++)
        if (indeg[i] == 0) queue[tail++] = i;

    while (head < tail) {
        int u = queue[head++];      /* ② 꺼내서 순서에 추가 */
        order[oc++] = u;
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k];
            if (--indeg[v] == 0)    /* ③ 0이 되면 큐에 */
                queue[tail++] = v;
        }
    }
    if (oc < n) {                  /* 다 못 넣음 = 사이클! */
        printf("CYCLE (위상 정렬 불가)\n");
    } else {
        for (int i = 0; i < oc; i++)
            printf("%d ", order[i]);
        printf("\n");
    }
    return 0;
}
Python — Kahn 위상 정렬
import sys
from collections import deque
input = sys.stdin.readline

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
indeg = [0] * (n + 1)
for _ in range(m):
    u, v = map(int, input().split())  # u -> v
    adj[u].append(v)
    indeg[v] += 1

# ① 진입차수 0을 큐에
q = deque(i for i in range(1, n + 1) if indeg[i] == 0)
order = []
while q:
    u = q.popleft()          # ② 꺼내서 순서에 추가
    order.append(u)
    for v in adj[u]:
        indeg[v] -= 1
        if indeg[v] == 0:      # ③ 0이 되면 큐에
            q.append(v)

if len(order) < n:           # 사이클!
    print('CYCLE (위상 정렬 불가)')
else:
    print(*order)
💡 사전순 위상 정렬이 필요하면 deque 대신 heapq를 쓰고 heappush/heappop으로 바꾸면 됩니다. 큐에 여러 후보가 있을 때 항상 가장 작은 번호를 꺼내게 되죠. (C는 최소 힙을 직접 구현 — ⑫편 다익스트라의 힙 코드 재활용!)
OUTPUT — 실험 DAG (Kahn, 사전순)
1 2 3 4 5 6 7 8

DFS 방식 — 후위 순서를 뒤집기

C — DFS 위상 정렬
#include <stdio.h>

int n, m;
int adjv[101][101], deg[101];
int visited[101];
int post[101], pc;   /* 후위 기록 */

void dfs(int u) {
    visited[u] = 1;
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (!visited[v]) dfs(v);
    }
    post[pc++] = u;   /* 이웃 다 본 뒤 자신 기록 */
}

int main(void) {
    scanf("%d %d", &n, &m);
    for (int e = 0; e < m; e++) {
        int u, v;
        scanf("%d %d", &u, &v);
        adjv[u][deg[u]++] = v;
    }
    for (int i = 1; i <= n; i++)
        if (!visited[i]) dfs(i);

    /* 후위 기록을 뒤집어 출력 */
    for (int i = pc - 1; i >= 0; i--)
        printf("%d ", post[i]);
    printf("\n");
    return 0;
}
Python — DFS 위상 정렬
import sys
input = sys.stdin.readline
sys.setrecursionlimit(300000)  # 깊은 재귀 보험

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v = map(int, input().split())
    adj[u].append(v)

visited = [False] * (n + 1)
post = []

def dfs(u):
    visited[u] = True
    for v in adj[u]:
        if not visited[v]:
            dfs(v)
    post.append(u)   # 이웃 다 본 뒤 자신 기록

for i in range(1, n + 1):
    if not visited[i]:
        dfs(i)

print(*post[::-1])   # 뒤집으면 위상 순서
OUTPUT — 실험 DAG (DFS 역후위)
2 1 3 5 7 4 6 8
비교Kahn (진입차수)DFS (후위 역순)
도구큐 + 진입차수 배열재귀 + 방문 배열
사이클 감지순서 개수 < n이면 사이클DFS 중 회색 정점 재방문 (아래 탭)
사전순 순서우선순위 큐로 쉽게 가능까다로움
재귀 깊이 부담없음 (반복문)깊으면 스택 오버플로 주의
💡 어느 걸 쓸까? 대부분 Kahn(진입차수)가 무난합니다 — 사이클 감지가 쉽고, 사전순 요구도 우선순위 큐로 대응됩니다. 정점 수가 매우 많아 재귀가 부담될 때 특히 Kahn을 추천합니다.

사이클 감지 — 위상 정렬의 덤

위상 정렬 알고리즘은 사이클 여부를 공짜로 알려줍니다. Kahn 방식이 특히 명쾌합니다:

Kahn의 사이클 판정
큐가 다 비었는데 순서에 담긴 정점 수 < 전체 정점 수(n) → 사이클 존재! (사이클 안의 정점들은 서로가 서로의 선수과목이라 진입차수가 영영 0이 되지 못해 큐에 못 들어감)

DFS 방식의 사이클 감지 — 3색 칠하기

DFS로 사이클을 잡으려면 정점을 3가지 상태로 관리합니다:

색상태
흰색 (0)아직 방문 안 함
회색 (1)현재 재귀 경로 위에 있음 (방문 중, 아직 안 끝남)
검은색 (2)완전히 끝남 (후위 기록 완료)
⚠️ DFS 도중 회색 정점으로 가는 간선을 만나면 → 그 회색 정점은 지금 내가 밟고 온 길 위에 있으므로, 되돌아가는 사이클(back edge)입니다! 검은색으로 가는 건 사이클이 아니니 헷갈리지 마세요.

도전 퀴즈 — 위상 정렬 8문제
