순서와 순환 — 유향 그래프의 두 얼굴

유향 그래프에는 두 가지 근본 질문이 있습니다. 하나는 "일에 순서를 매길 수 있는가?"(위상 정렬), 다른 하나는 "서로 오갈 수 있는 정점 무리는?"(강결합요소)입니다. 둘 다 DFS를 부품으로 씁니다.

📋 위상 정렬

방향이 있고 사이클이 없는 그래프(DAG)의 정점을 줄 세우기. "A→B면 A가 B보다 먼저." 선수과목, 빌드 순서 등에 쓰여요.

🔗 강결합요소 (SCC)

서로 오갈 수 있는 정점끼리 묶기. u→v와 v→u 경로가 모두 있으면 같은 SCC. 코사라주·타잔으로 찾습니다.

둘의 연결 — SCC를 하나의 점으로 압축하면 유향 그래프는 항상 DAG가 됩니다(SCC 응축 그래프). 그 위에서 위상 정렬이 가능해요. 즉 "순환이 있는 그래프"도 SCC로 묶으면 "순서를 매길 수 있는 그래프"가 됩니다. 두 개념이 이렇게 만나요.

위상 정렬 — Kahn 알고리즘 (진입차수)

위상 정렬의 가장 직관적인 방법은 진입차수(in-degree)를 쓰는 Kahn 알고리즘입니다. 진입차수가 0인 정점(아무도 가리키지 않는 = 먼저 할 수 있는 일)부터 하나씩 빼면서 순서를 만듭니다.

핵심 아이디어
// 진입차수 = 나를 가리키는 간선 수
// = 나보다 먼저 해야 할 일의 수

// 1) 진입차수 0인 정점을 큐에
// 2) 큐에서 꺼내 순서에 추가
// 3) 그 정점이 가리키던 곳들의
//    진입차수를 1씩 감소
// 4) 0이 되면 큐에 추가
// 5) 큐가 빌 때까지 반복
C — Kahn
/* 진입차수 계산 후 */
for (int i=1;i<=n;i++)
    if (indeg[i]==0) q[t++]=i;
while (h < t) {
    int u = q[h++];
    order[oc++] = u;
    for (int k=0;k<deg[u];k++){
        int v = adj[u][k];
        if (--indeg[v]==0)
            q[t++] = v;
    }
}
/* oc < n 이면 사이클 존재 */
💡 사이클 탐지 덤 — 위상 정렬을 끝냈는데 순서에 담긴 정점 수가 n보다 적으면? 진입차수가 끝까지 0이 안 된 정점이 남았다는 뜻 → 사이클이 존재합니다. Kahn 알고리즘은 정렬과 사이클 탐지를 동시에 해요.

DFS 기반 위상 정렬

Kahn 말고 DFS로도 위상 정렬을 할 수 있습니다. DFS의 종료 순서(finish)의 역순이 곧 위상 정렬이에요. 이건 코사라주에서 다시 쓰이는 핵심 아이디어입니다.

C — DFS 위상정렬
void dfs(int u){
    visited[u]=1;
    for (int k=0;k<deg[u];k++){
        int v=adj[u][k];
        if (!visited[v]) dfs(v);
    }
    stack[top++] = u; /* 종료 시 push */
}
/* stack을 역순으로 = 위상정렬 */
왜 종료 역순인가
// u→v 간선이 있으면
// v가 u보다 먼저 종료됨
// (v를 다 처리해야 u가 끝나니까)

// 종료 순서: ... v ... u ...
// 역순:      ... u ... v ...
//            → u가 v보다 앞!

// 이게 바로 위상 정렬 조건
두 방법 비교 — Kahn은 진입차수와 큐를 쓰는 BFS 스타일, DFS 버전은 재귀와 스택을 쓰는 DFS 스타일입니다. 결과는 (여러 유효한 순서 중 하나로) 같아요. 코사라주가 이 DFS 종료 순서를 결정적으로 활용합니다.

코사라주 — 두 번의 DFS

코사라주는 놀랍도록 단순합니다. DFS를 두 번 돌려 SCC를 찾아요. 첫 번째는 종료 순서를 얻고, 두 번째는 역방향 그래프에서 그 역순으로 DFS합니다.

3단계 절차
// 1) 원본 그래프 DFS
//    → 종료 순서(finish) 기록

// 2) 모든 간선 방향을 뒤집음
//    (역방향 그래프)

// 3) 종료 역순으로 역그래프 DFS
//    한 번의 DFS로 도달하는
//    정점들 = 하나의 SCC
왜 이게 통하나
// 종료가 가장 늦은 정점은
// SCC 응축 그래프의 "원천"

// 역그래프에서 거기서 출발하면
// 그 SCC 안에만 갇힘
// (밖으로 나가는 간선이
//  역방향에선 들어오는 간선)

// → 딱 그 SCC만 방문!
💡 핵심 통찰 — 종료 순서가 가장 늦은 정점부터 역그래프에서 DFS하면, 각 DFS가 정확히 하나의 SCC만 훑습니다. 방향을 뒤집었기 때문에 SCC 밖으로 "새어 나가지" 않아요.

타잔 — 단 한 번의 DFS로

타잔 알고리즘은 DFS 한 번으로 모든 SCC를 찾습니다. 코사라주보다 우아하죠. 비결은 각 정점에 두 개의 번호를 매기는 것입니다 — id(방문 순서)와 low(도달 가능한 가장 작은 id).

두 개의 번호
// id[u]: u를 방문한 순서 (고유)

// low[u]: u에서 DFS로 도달 가능한
//   정점 중 가장 작은 id
//   (단, 스택에 있는 정점만)

// 핵심 규칙:
//   low[u] == id[u] 이면
//   u는 SCC의 "루트"!
low 갱신 규칙
// 이웃 v를 볼 때:

// v가 미방문이면:
//   dfs(v) 후
//   low[u] = min(low[u], low[v])

// v가 스택에 있으면 (역방향/교차):
//   low[u] = min(low[u], id[v])

// 스택에 없으면 무시
//   (다른 SCC)
low-link의 의미 — low[u]는 "u에서 출발해 (현재 스택 위의 정점들을 통해) 되돌아갈 수 있는 가장 오래된 정점"입니다. 만약 u가 자기 자신보다 오래된 곳으로 못 돌아간다면(low[u]==id[u]), u가 이 SCC의 시작점이에요.

한 줄씩 실행 — id · low · 스택

[다음 ▶]마다 정점에 id/low가 매겨지고, 스택에 쌓입니다. low==id인 정점을 만나면 스택에서 SCC를 팝해요.

타잔 완성 코드

int id[MAXN], low[MAXN], onStk[MAXN], vis[MAXN];
int stk[MAXN], top, cnt;  /* cnt: 방문 순서 카운터 */

void tarjan(int u){
    id[u] = low[u] = cnt++;
    stk[top++] = u; onStk[u] = 1; vis[u] = 1;
    for (int k=0;k<deg[u];k++){
        int v = adj[u][k];
        if (!vis[v]) {                        /* 미방문 */
            tarjan(v);
            low[u] = min(low[u], low[v]);
        } else if (onStk[v]) {                /* 스택에 있음 */
            low[u] = min(low[u], id[v]);
        }
    }
    if (low[u] == id[u]) {                    /* SCC 루트! */
        while (1) {
            int w = stk[--top]; onStk[w] = 0;
            /* w를 현재 SCC에 추가 */
            if (w == u) break;
        }
    }
}
💡 코사라주 vs 타잔 — 코사라주는 DFS 2번 + 역그래프가 필요하지만 이해하기 쉽습니다. 타잔은 DFS 1번으로 끝나 더 효율적이지만 low-link 개념이 필요해요. 둘 다 O(V+E)입니다.

SCC의 활용 — 응축 그래프

SCC를 찾는 진짜 이유는 복잡한 순환 그래프를 단순한 DAG로 압축하기 위해서입니다. 각 SCC를 하나의 점으로 묶으면(응축), 반드시 사이클 없는 DAG가 돼요.

활용내용
2-SAT논리식 만족 가능성 판정 (SCC로 모순 탐지)
응축 그래프SCC를 점으로 → DAG → 위상 정렬 가능
웹 구조 분석서로 링크된 페이지 무리 찾기
데드락 탐지자원 대기 그래프의 순환 = SCC
강한 연결성전체가 하나의 SCC = 어디서든 왕복 가능
2-SAT의 아름다움 — "x이거나 y" 형태의 논리 제약들을 함의 그래프로 만든 뒤 SCC를 찾으면, x와 ¬x가 같은 SCC에 있으면 모순(만족 불가)입니다. 겉보기엔 논리 문제인데 SCC로 우아하게 풀려요. 이것이 SCC를 배우는 큰 이유 중 하나입니다.
