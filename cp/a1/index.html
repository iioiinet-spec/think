이분 매칭 호프크로프트–카프

층별 BFS로 최단 증가경로를 찾고, DFS로 여러 개를 한꺼번에 — O(E√V) 최대 매칭

C 시각화: 매칭 · 교대 경로 · 층 그래프 · 다중 증가

이분 매칭 — 짝을 최대한 많이 맺기

이분 그래프(bipartite graph)는 정점이 두 그룹(왼쪽 L, 오른쪽 R)으로 나뉘고, 간선은 서로 다른 그룹 사이에만 있는 그래프입니다. 매칭(matching)은 한 정점이 최대 하나의 짝만 갖도록 고른 간선 집합이에요.

최대 이분 매칭은 이런 짝을 최대한 많이 맺는 문제입니다. 지원자–일자리, 학생–연구실, 작업–기계 배정처럼 현실에서 아주 흔해요.

용어의미
이분 그래프정점이 L·R 두 그룹, 간선은 그룹 간에만
매칭서로 정점을 공유하지 않는 간선 집합
자유 정점아직 짝이 없는 정점
최대 매칭가능한 가장 많은 짝
💡 호프크로프트–카프는 최대 이분 매칭을 O(E√V)에 푸는 빠른 알고리즘입니다. 한 번에 하나씩 늘리는 대신, 여러 증가경로를 한꺼번에 처리해 페이즈 수를 √V로 줄이는 게 핵심이에요.

이 강의의 예제 — 이분 그래프

왼쪽 L={1,2,3,4}, 오른쪽 R={1,2,3,4}. 정점 L1이 허브처럼 R1~R4 모두에 연결돼 있습니다.

INPUT (u v = L측 u — R측 v 연결)
4 4 7 1 1 1 2 1 3 1 4 2 1 3 2 4 3
최대 매칭 (완전 매칭, 크기 4)
L1-R4 L2-R1 L3-R2 L4-R3 → 4쌍 모두 매칭 (Phase 2번에 완성)

왜 "층별"인가 — 호프크로프트–카프의 아이디어

단순한 방법(각 L 정점마다 증가경로 하나씩)은 최악에 O(V·E)입니다. 호프크로프트–카프는 두 단계를 반복해요.

① BFS — 층 나누기

자유 L 정점에서 시작해 최단 증가경로의 길이(층)를 BFS로 계산. 같은 길이의 증가경로들을 한 페이즈에 모아요.

② DFS — 다중 증가

층 그래프에서 정점을 공유하지 않는 증가경로를 DFS로 한꺼번에 여러 개 찾아 매칭을 늘립니다.

√V 페이즈의 마법 — 매 페이즈마다 최단 증가경로 길이가 반드시 늘어납니다. 증가경로 길이는 최대 O(√V)까지만 가능하므로, 페이즈 수가 O(√V)로 제한됩니다. 각 페이즈가 O(E)이니 전체 O(E√V)예요.

교대 경로와 증가 경로

이분 매칭의 심장은 증가 경로(augmenting path)입니다. 이걸 이해하면 모든 매칭 알고리즘이 쉬워져요.

개념정의
교대 경로매칭 간선과 비매칭 간선이 번갈아 나오는 경로
증가 경로자유 정점에서 시작해 자유 정점에서 끝나는 교대 경로
왜 "증가"인가 — 증가 경로에서 매칭/비매칭을 뒤집으면(비매칭→매칭, 매칭→비매칭) 매칭 크기가 정확히 1 늘어납니다. 경로 양 끝이 자유 정점이라, 비매칭 간선이 매칭 간선보다 하나 더 많거든요.

// 뒤집은 후: L2-R1, L1-R4 (2개)

//        ⟺

//   증가경로를 계속 찾아 뒤집기
//   → 없어지면 최대 매칭 완성
💡 네트워크 플로우와의 연결 — 이분 매칭은 사실 최대 유량 문제입니다(모든 간선 용량 1). 증가 경로는 플로우의 증가 경로와 똑같은 개념이에요. 호프크로프트–카프는 이 구조를 이분 그래프에 특화해 더 빠르게 만든 겁니다.

재배치 — 기존 매칭을 밀어내며 늘리기

증가 경로의 진짜 힘은 재배치에 있습니다. 이미 짝이 있는 정점도, 그 짝을 다른 곳으로 밀어낼 수 있으면 새 매칭을 받아들여요.

시나리오 — L2를 매칭하려면?
[다음 ▶]으로 증가 경로가 기존 매칭을 재배치하며 짝을 늘리는 과정을 보세요.
예제의 Phase 2 — L2는 R1에만 연결됩니다. 그런데 R1은 이미 L1과 매칭돼 있어요. 포기할까요? 아뇨! L1은 R4로 옮길 수 있습니다. 그래서 증가 경로 L2→R1→L1→R4를 따라 재배치하면, L1-R1이 L1-R4로 바뀌고 L2가 R1을 차지해 매칭이 하나 늘어요.

층 그래프 — 다중 증가의 비결

호프크로프트–카프가 단순 매칭보다 빠른 이유는 층 그래프(level graph) 덕분입니다. BFS로 각 정점의 "층"(자유 L에서의 최단 거리)을 매긴 뒤, 그 층을 따라서만 DFS 증가경로를 찾아요.

BFS — 층 계산
/* 자유 L 정점 = 층 0 */
for (int u=1;u<=nl;u++){
    if (pairU[u]==NIL){dist[u]=0;push(u);}
    else dist[u]=INF;
}
while (큐){
    u = pop();
    for (v : adj[u]){
        w = pairV[v];   /* v의 짝 */
        if (w==NIL) found=1;
        else if (dist[w]==INF){
            dist[w]=dist[u]+1; push(w);
        }
    }
}
DFS — 층 따라 증가
int dfs(int u){
    for (v : adj[u]){
        w = pairV[v];
        if (w==NIL ||
           (dist[w]==dist[u]+1 && dfs(w))){
            pairU[u]=v; /* 매칭! */
            pairV[v]=u;
            return 1;
        }
    }
    dist[u]=INF; /* 막힘 표시 */
    return 0;
}
💡 DFS가 dist[w]==dist[u]+1 조건을 지키는 게 핵심입니다. 이렇게 하면 정확히 층을 한 단계씩 내려가는 증가경로만 찾아, 여러 경로가 정점을 공유하지 않게 돼요. 덕분에 한 페이즈에서 여러 증가경로를 동시에 처리합니다.

한 페이즈 = BFS 1번 + DFS 여러 번

한 페이즈의 구조를 명확히 봅시다. BFS로 층을 나눈 뒤, 모든 자유 L 정점에 대해 DFS를 시도해 증가경로를 최대한 많이 찾습니다.

[다음 단계 ▶]로 한 페이즈 안에서 BFS와 여러 DFS가 어떻게 협력하는지 보세요.
단순 매칭 vs 호프크로프트–카프 — 단순 매칭은 증가경로를 한 번에 하나씩 찾아 최대 V번 반복(O(VE)). 호프크로프트–카프는 같은 길이의 증가경로를 한 페이즈에 모아서 처리해 페이즈가 O(√V)번뿐입니다. BFS로 층을 나눈 게 이 "묶음 처리"를 가능하게 해요.

왜 √V 페이즈인가

호프크로프트–카프의 복잡도 O(E√V)는 페이즈 수가 O(√V)라는 정리에서 나옵니다.

사실설명
길이 단조 증가매 페이즈마다 최단 증가경로 길이가 반드시 늘어남
√V번 후√V 페이즈 후, 남은 증가경로 길이 > √V
남은 매칭길이 >√V인 서로소 경로는 <√V개뿐
합계총 페이즈 O(√V), 각 O(E) → O(E√V)
💡 정점 V개인 이분 그래프에서 √V는 큰 개선입니다. 예를 들어 V=10,000이면 단순 매칭의 페이즈 10,000번이 호프크로프트–카프에선 약 100번으로 줄어요. 대회에서 큰 이분 그래프를 다룰 때 필수 알고리즘입니다.

이분 매칭 = 최대 유량

이분 매칭은 네트워크 플로우의 특수한 경우입니다. 이분 그래프를 플로우 네트워크로 바꾸면 최대 유량이 곧 최대 매칭이에요.

플로우로 변환
// 1) 발원지 S 추가
//    S → 모든 L 정점 (용량 1)

// 2) 원래 간선 L → R (용량 1)

// 3) 도착지 T 추가
//    모든 R 정점 → T (용량 1)

// 최대 유량 = 최대 매칭!
// (용량 1이라 각 정점 1번만)
왜 호프크로프트–카프?
// 에드몬드-카프로도 풀 수 있음
//   → O(VE²) (느림)

// 이분 구조에 특화하면
//   호프크로프트-카프 O(E√V)

// 층 그래프 + 다중 증가로
// 플로우보다 훨씬 빠르게!
특수화의 힘 — 일반 최대 유량 알고리즘(에드몬드–카프)으로 이분 매칭을 풀면 O(VE²)입니다. 하지만 이분 그래프는 구조가 단순해서(모든 용량 1, 두 그룹), 이를 활용한 호프크로프트–카프가 O(E√V)로 훨씬 빨라요. "특수 구조를 알면 더 빠른 알고리즘이 있다"는 좋은 예입니다.

쾨니그 정리 — 매칭과 커버

이분 그래프에는 아름다운 정리가 있습니다. 최대 매칭 = 최소 정점 커버(쾨니그 정리).

개념정의
정점 커버모든 간선의 한쪽 끝을 포함하는 정점 집합
최소 정점 커버가장 작은 그런 집합
쾨니그 정리이분 그래프에서 최대 매칭 = 최소 정점 커버
최대 독립집합= V − 최대 매칭 (여집합)
💡 정리의 연쇄 — 최대 매칭을 구하면 최소 정점 커버, 최대 독립집합까지 자동으로 얻습니다. 이분 그래프에서 이 세 문제가 모두 연결돼 있어요. 그래서 이분 매칭은 "겉보기엔 다른 여러 문제"를 푸는 만능 열쇠입니다.

이분 매칭의 활용

문제매칭으로 모델링
작업 배정일꾼 L — 작업 R, 가능한 조합에 간선
시간표 짜기수업 L — 시간대 R
결혼 문제선호하는 짝끼리 간선, 최대 매칭
체스판 도미노흑/백 칸으로 이분, 도미노 = 매칭
네트워크 신뢰성서로소 경로 찾기
모델링이 핵심 — 실전에서 어려운 건 알고리즘이 아니라 "이 문제가 이분 매칭이구나"를 알아채는 겁니다. 두 그룹으로 나뉘고, 짝을 최대한 많이 맺는 구조가 보이면 이분 매칭을 의심하세요. 체스판 도미노 채우기처럼 겉보기엔 전혀 매칭 같지 않은 문제도 이분 매칭으로 풀립니다.
