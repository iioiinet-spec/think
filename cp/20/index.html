객체는 마법이 아니다 — 구조체 + 함수

파이썬에서 obj.method()를 쓸 때, 내부에서 무슨 일이 벌어질까요? 많은 학생이 객체를 "마법 상자"처럼 여기지만, 사실 그 핵심은 C의 구조체와 함수 포인터만으로 거의 그대로 만들 수 있습니다.

이 강의의 목표는 하나입니다: 파이썬 객체를 C 구조체로 분해해서, "속성이 어디 저장되고", "메서드가 어떻게 호출되며", "self가 무엇인지"를 눈으로 확인하는 것. 이걸 이해하면 파이썬 OOP가 훨씬 투명해집니다.

💡 실제로 파이썬 인터프리터(CPython) 자체가 C로 작성되어 있고, 모든 파이썬 객체는 내부적으로 PyObject라는 구조체입니다. 우리가 이 강의에서 손으로 만들 구조가 CPython 내부와 놀랄 만큼 닮았습니다.

실습 예제 — 2D 벡터 Vec2

강의 내내 함께할 예제는 2차원 벡터입니다. 필드는 x, y 두 개, 메서드는 크기를 구하는 norm(), 더하는 add(), 출력하는 show(). 단순하지만 "필드 + 메서드 + self"의 삼박자를 모두 담고 있습니다.

C — 우리가 만들 목표
Vec2 a = Vec2_new(3, 4);
a.show(&a);            // Vec2(3.0, 4.0)
a.norm(&a);           // 5.0
Vec2 c = a.add(&a, &b); // 두 벡터 합
Python — 대응되는 모습
a = Vec2(3, 4)
a.show()              # Vec2(3.0, 4.0)
a.norm()              # 5.0
c = a.add(b)          # 두 벡터 합

차이가 보이나요? C는 &a를 손으로 넘기고, 파이썬은 self가 자동으로 넘어갑니다. 바로 이 지점이 3번 탭의 주제입니다.

먼저 데이터부터 — 구조체 필드 = 인스턴스 속성

메서드는 잠시 미뤄 두고, 데이터만 담아 봅시다. C의 구조체는 관련된 값들을 하나로 묶고, 파이썬은 __init__에서 self.속성에 값을 넣습니다.

C — 구조체 정의와 사용
#include <stdio.h>

/* 필드(멤버)를 묶은 구조체 */
typedef struct {
    double x;
    double y;
} Vec2;

int main(void) {
    Vec2 a;              /* 구조체 변수 생성 */
    a.x = 3;            /* 필드에 값 대입 */
    a.y = 4;
    printf("(%.1f, %.1f)\n", a.x, a.y);
    return 0;
}
Python — 클래스와 인스턴스
class Vec2:
    def __init__(self, x, y):
        self.x = x        # 속성에 값 대입
        self.y = y

a = Vec2(3, 4)      # 인스턴스 생성
print(f"({a.x:.1f}, {a.y:.1f})")
C 관점 — 구조체는 "메모리에서 x 다음에 y가 연속으로 놓인 덩어리"입니다. a.x는 그 덩어리의 시작에서 몇 바이트 떨어진 곳을 읽는 것뿐이에요.
Python 관점 — 인스턴스는 속성 이름→값을 담은 딕셔너리(a.__dict__)를 품고 있습니다. C보다 유연하지만(속성을 나중에 추가 가능) 그만큼 메모리와 속도를 더 씁니다.

메모리 그림으로 보기

필드 두 개짜리 Vec2가 메모리에 어떻게 놓이는지 봅시다. C는 정확히 이 모양으로 연속 배치됩니다.

3.0x (8바이트)
4.0y (8바이트)

→ sizeof(Vec2) = 16바이트. 딱 필드 크기의 합입니다. 파이썬 객체는 여기에 참조 카운트·클래스 포인터 등 관리용 데이터가 더 붙어 훨씬 큽니다(보통 수십 바이트).

필드 값을 바꿔 메모리 그림 확인

두 언어의 접근 방식

핵심 — 구조체 안에 "함수"를 넣는다

이제 진짜 재미있는 부분입니다. C에서 함수는 값이 아니지만, 함수의 주소(함수 포인터)는 변수에 담을 수 있습니다. 그래서 구조체 멤버로 함수 포인터를 넣으면 — 구조체가 "자기 함수"를 갖게 됩니다. 이것이 메서드의 정체입니다.

C — 함수 포인터를 멤버로
typedef struct Vec2 Vec2;
struct Vec2 {
    double x, y;                    /* 필드 */
    double (*norm)(const Vec2 *self); /* 메서드! */
};

/* 실제 함수 구현 */
double vec_norm(const Vec2 *self) {
    return sqrt(self->x*self->x
              + self->y*self->y);
}

/* 생성자: 필드 + 함수포인터 연결 */
Vec2 Vec2_new(double x, double y) {
    Vec2 v = { x, y, vec_norm };
    return v;              /* norm에 함수 주소 저장 */
}

/* 호출 */
Vec2 a = Vec2_new(3, 4);
printf("%.1f\n", a.norm(&a)); // 5.0
Python — 클래스 메서드
import math

class Vec2:
    def __init__(self, x, y):
        self.x = x                # 필드
        self.y = y

    def norm(self):              # 메서드!
        return math.sqrt(self.x*self.x
                       + self.y*self.y)



# 생성자는 __init__이 자동 담당



# 호출
a = Vec2(3, 4)
print(a.norm())             # 5.0
⚠️ 결정적 차이 — 파이썬은 a.norm()이지만 C는 a.norm(&a)입니다. 파이썬이 자동으로 넣어 주는 self를, C에서는 우리가 손으로 &a(자기 주소)를 넘겨야 합니다. 다음 탭에서 이 self를 파헤칩니다.

self는 마법이 아니라 "첫 번째 인자"다

파이썬 초보가 가장 헷갈리는 것 중 하나가 self입니다. "왜 def에는 self가 있는데 부를 때는 안 쓰지?" C로 보면 답이 명확합니다.

파이썬의 a.norm()은 사실 내부적으로 Vec2.norm(a)로 번역됩니다. 즉 점 앞의 객체가 첫 인자 self로 자동으로 들어가는 것입니다. C에는 그런 자동화가 없어서 우리가 직접 넘깁니다.

💡 self라는 이름은 관습일 뿐 — 파이썬에서 첫 인자 이름을 this나 me로 써도 동작합니다(권장하진 않음). 중요한 건 "위치가 첫 번째"라는 것. C의 함수 포인터에서 첫 인자를 self로 두는 것과 정확히 같은 약속입니다.

self가 왜 꼭 필요할까 — 인스턴스가 여럿일 때

메서드 코드는 하나인데 인스턴스는 여러 개입니다. norm()이 어느 벡터의 크기를 재야 할까요? 바로 self가 그 답 — "지금 이 호출의 대상"을 가리킵니다.

C — 같은 함수, 다른 self
Vec2 a = Vec2_new(3, 4);
Vec2 b = Vec2_new(6, 8);

vec_norm(&a);   // 5.0  (self = a)
vec_norm(&b);   // 10.0 (self = b)
// 함수는 하나, self만 바뀐다
Python — 같은 메서드, 다른 self
a = Vec2(3, 4)
b = Vec2(6, 8)

a.norm()   # 5.0  (self = a)
b.norm()   # 10.0 (self = b)
# 메서드는 하나, self만 바뀐다

한 걸음 더 — 메서드를 인스턴스마다 두면 낭비다

2번 탭 방식은 인스턴스마다 함수 포인터를 복사해 갖습니다. 인스턴스가 100만 개면 같은 함수 주소가 100만 번 저장되는 셈 — 낭비죠. 파이썬은 이렇게 하지 않습니다.

파이썬은 메서드를 클래스에 딱 한 벌만 두고, 모든 인스턴스는 __class__ 포인터로 그 클래스를 공유합니다. 이 "메서드 묶음"을 C에서는 vtable(가상 메서드 테이블)이라 부릅니다.

C — vtable 공유 방식
typedef struct Circle Circle;

/* 메서드 테이블 = 클래스 객체 */
typedef struct {
    double (*area)(const Circle*);
    void   (*describe)(const Circle*);
} CircleClass;

struct Circle {
    const CircleClass *klass; /* __class__ */
    double r;                 /* 인스턴스 필드 */
};

double circle_area(const Circle *self) {
    return 3.14159 * self->r * self->r;
}

/* 클래스 객체는 프로그램에 딱 하나! */
static const CircleClass VT =
    { circle_area, /* ... */ };

Circle Circle_new(double r) {
    Circle c = { &VT, r }; /* 공유 연결 */
    return c;
}

/* 호출: 인스턴스 -> 클래스 -> 메서드 */
Circle c = Circle_new(2);
c.klass->area(&c);        // 12.566
Python — 원래 이렇게 동작
import math

class Circle:
    def __init__(self, r):
        self.r = r          # 인스턴스 필드만

    def area(self):        # 메서드는 클래스에
        return math.pi * self.r ** 2

    def describe(self):
        print(self.area())

c = Circle(2)
d = Circle(5)

# 두 인스턴스가 클래스(메서드 묶음)를 공유
print(type(c) is type(d))  # True

# area 함수는 메모리에 딱 하나
print(Circle.area)        # 하나의 함수 객체

c.area()                  # 12.566
Python 내부 — c.area()를 만나면 파이썬은 ① 인스턴스 c에서 area를 찾고 → 없으면 ② type(c)(클래스)에서 찾습니다. 이 "인스턴스에 없으면 클래스로" 탐색이 바로 C의 c.klass->area와 같은 동작입니다.

메모리 절약 비교

인스턴스마다 메서드를 갖는 방식(2번 탭)과 vtable 공유 방식의 차이를 인스턴스 수를 늘려 가며 봅시다. 메서드 3개, 각 함수 포인터 8바이트로 가정합니다.

인스턴스 수에 따른 메모리

💡 인스턴스가 많아질수록 vtable 공유의 이득이 커집니다. 파이썬이 애초에 이 방식을 택한 이유죠. 그 대신 호출 때마다 "인스턴스 → 클래스 → 메서드"를 한 번 더 거치는 간접 참조 비용이 살짝 듭니다 — 메모리와 속도의 trade-off입니다.

전체 Vec2 — C와 파이썬 완성본 나란히

지금까지 배운 모든 것을 합친 완성본입니다. 필드 x·y, 메서드 norm·add·show, self 전달까지 — 두 언어가 같은 결과를 냅니다. 오른쪽 시뮬레이터로 직접 실행해 보세요.

C 완성본 (함수포인터 방식)
#include <stdio.h>
#include <math.h>

typedef struct Vec2 Vec2;
struct Vec2 {
    double x, y;
    double (*norm)(const Vec2*);
    Vec2   (*add)(const Vec2*, const Vec2*);
    void   (*show)(const Vec2*);
};

double v_norm(const Vec2 *s) {
    return sqrt(s->x*s->x + s->y*s->y);
}
Vec2 Vec2_new(double x, double y);
Vec2 v_add(const Vec2 *s, const Vec2 *o) {
    return Vec2_new(s->x+o->x, s->y+o->y);
}
void v_show(const Vec2 *s) {
    printf("Vec2(%.1f, %.1f)\n", s->x, s->y);
}
Vec2 Vec2_new(double x, double y) {
    Vec2 v = { x, y, v_norm, v_add, v_show };
    return v;
}

int main(void) {
    Vec2 a = Vec2_new(3, 4);
    Vec2 b = Vec2_new(1, 2);
    a.show(&a);
    printf("%.1f\n", a.norm(&a));
    Vec2 c = a.add(&a, &b);
    c.show(&c);
    return 0;
}
Python 완성본 (클래스 방식)
import math


class Vec2:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def norm(self):
        return math.sqrt(self.x**2 + self.y**2)

    def add(self, o):
        return Vec2(self.x + o.x, self.y + o.y)

    def show(self):
        print(f"Vec2({self.x:.1f}, {self.y:.1f})")






a = Vec2(3, 4)
b = Vec2(1, 2)
a.show()
print(f"{a.norm():.1f}")
c = a.add(b)
c.show()

정리 — 파이썬 객체를 C로 다시 읽기

구조체는 객체의 몸통(필드), 함수 포인터는 메서드, 첫 인자 self는 "이 호출의 대상", vtable은 클래스. 이 네 조각을 맞추고 나면 obj.method() 한 줄에 담긴 모든 일이 보입니다. 파이썬의 편리함이 C 위에 어떻게 쌓여 있는지 이해하면, 두 언어 모두 더 깊이 다룰 수 있습니다.
