트라이 + KMP의 실패 링크 — 수천 개 패턴을 텍스트 한 번의 스캔으로

C 시각화: 트라이 · 실패 링크(BFS) · 출력 링크 · 한 번의 스캔

패턴이 하나가 아니라면 — KMP를 1000번?

KMP는 패턴 하나를 O(N+M)에 찾습니다. 그런데 현실에서는 패턴이 여러 개예요 — 금지어 필터, 바이러스 시그니처, 검색어 하이라이트…

패턴이 k개면 KMP를 k번 돌려야 합니다. 텍스트를 k번 읽는 거죠. 패턴이 1000개면 텍스트를 1000번 훑어야 해요.

방법텍스트 스캔복잡도
패턴마다 단순 검색k번O(k·N·M)
패턴마다 KMPk번O(k·N + ΣM)
아호–코라식1번O(N + ΣM + z)
아호–코라식의 아이디어 — 패턴들을 트라이에 넣어 접두사를 공유시키고, 각 노드에 KMP의 실패 함수를 일반화한 실패 링크를 답니다. 그러면 텍스트를 딱 한 번 훑으며 모든 패턴을 동시에 찾아요. (z = 찾은 매칭 개수)
💡 두 편의 합체 — 트라이 편에서 여러 문자열의 접두사를 트리로 겹쳐 담는 법을, KMP 편에서 실패해도 되돌아가지 않는 법을 배웠습니다. 아호–코라식은 정확히 그 둘을 합친 것이에요. 트라이 위에 실패 링크를 달면 유한 상태 오토마타가 됩니다.

이 강의의 예제 — 원논문의 고전

1975년 아호–코라식 원논문에 실린 바로 그 예제입니다. 패턴 4개 he, she, his, hers, 텍스트 ushers.

패턴 (사전)
텍스트 T
완성된 오토마타  (실선=트라이, 보라 점선=실패 링크, 주황 점선=출력 링크)
정답 — 텍스트 한 번 스캔으로 3개 발견
"she" 시작 1 (ushers 의 s-h-e) "he" 시작 2 ← "she"를 찾는 순간 출력 링크로 함께 발견! "hers" 시작 2 패턴마다 KMP: 텍스트 4번 스캔 (24글자 읽음) 아호-코라식 : 텍스트 1번 스캔 ( 6글자 읽음)
💡 두 가지 마법을 눈여겨보세요. ① she를 찾는 순간 출력 링크로 he도 공짜로 잡힙니다. ② she 다음에 r이 왔을 때, 트라이엔 sher이 없지만 실패 링크를 타고 he로 옮겨가 hers를 이어서 찾아요.

실패 링크 — KMP의 pi를 트라이로

KMP에서 pi[i]는 "실패했을 때 되돌아갈 패턴 안의 위치"였습니다. 트라이에는 패턴이 여러 개 얽혀 있으니, 되돌아갈 곳도 노드가 됩니다.

fail[u] = u가 나타내는 문자열의 진접미사 중,
어떤 패턴의 접두사이기도 한 가장 긴 것의 노드
KMP와 나란히 — KMP의 pi는 "그 패턴 자신의 접두사=접미사"였죠. 아호–코라식의 fail은 "사전 전체의 어떤 접두사와 같은 접미사"입니다. 트라이의 노드가 곧 모든 패턴의 모든 접두사니까, 정의가 자연스럽게 확장돼요.

예제로 확인해 봅시다. 각 노드가 나타내는 문자열의 진접미사를 긴 것부터 보며, 트라이에 있는 첫 번째를 찾습니다.

노드문자열진접미사 (긴 것부터)fail
1h(없음)→ root
4shh ✓ 트라이에 있음!→ node1 (h)
5shehe ✓ 있음!→ node2 (he)
7hisis ✗ → s ✓ 있음!→ node3 (s)
9hersers ✗ → rs ✗ → s ✓→ node3 (s)
💡 node5(she) → node2(he)가 이 강의의 핵심입니다. she를 읽던 중 막히면, 이미 읽은 he는 버릴 필요가 없어요 — 그것도 사전의 접두사거든요! 게다가 he는 그 자체로 단어라서, 여기서 출력 링크가 태어납니다(4번 탭).

왜 BFS로 만드나

실패 링크는 항상 자기보다 얕은 노드를 가리킵니다(진접미사는 더 짧으니까). 그래서 얕은 곳부터 채우면 필요한 값이 이미 준비돼 있어요 — 그게 BFS입니다.

C — 실패 링크 만들기
void buildFail(void){
    /* 깊이 1은 전부 root로 */
    for (int c=0; c<ALPHA; c++)
        if (ch[0][c]){ fail[ch[0][c]]=0; push(ch[0][c]); }
    while (큐가 비지 않음){
        int u = pop();
        for (int c=0; c<ALPHA; c++){
            int v = ch[u][c];
            if (!v) continue;
            int f = fail[u];
            while (f && !ch[f][c]) f = fail[f];  /* 타고 올라감 */
            fail[v] = (ch[f][c] && ch[f][c]!=v) ? ch[f][c] : 0;
            push(v);
        }
    }
}
KMP와 비교
// KMP:
//   while (j>0 && P[i]!=P[j])
//       j = pi[j-1];

// 아호-코라식:
//   while (f && !ch[f][c])
//       f = fail[f];

// 똑같은 구조!
//   "안 되면 더 짧은 후보로"
// 다른 점: 배열 j → 트라이 노드 f
부모의 fail에서 출발 — 노드 v(= 부모 u에서 글자 c로 온 노드)의 fail을 구할 땐, 부모의 fail부터 봅니다. fail[u]에서 c로 갈 수 있으면 거기가 답. 없으면 fail[fail[u]]로 또 올라가요. KMP의 j = pi[j-1] 연쇄와 정확히 같습니다.
⚠️ 이 예제에선 while이 한 번도 안 돕니다 — 정직하게 말하면, {he, she, his, hers}에서는 while (f && !ch[f][c]) 루프가 한 번도 실행되지 않아요. 비-root fail을 가진 노드 중 자식이 있는 건 node4(sh, fail=node1)뿐인데, node1이 마침 e 자식을 갖고 있거든요. 그래도 코드에 while이 꼭 있어야 합니다 — 사전이 커지면 반드시 타고 올라갈 일이 생기니까요. KMP의 pi 만들기에서 ABABAC의 i=5가 3→1→0으로 미끄러졌던 그 상황이 트라이에서도 똑같이 일어납니다.

스캔 — 오토마타를 한 번 통과시킨다

실패 링크까지 달았으면 이제 상태 기계입니다. 텍스트 글자를 하나씩 먹이며 상태(노드)를 옮기고, 단어 노드에 닿을 때마다 보고해요. 텍스트는 딱 한 번 읽습니다.

C — 스캔
void search(const char* T){
    int cur = 0;
    for (int i=0; T[i]; i++){   /* i는 전진만! */
        int c = T[i]-'a';
        while (cur && !ch[cur][c])
            cur = fail[cur];   /* 실패 링크로 후퇴 */
        cur = ch[cur][c];        /* 한 글자 전진 */
        /* 출력 링크 체인으로 전부 보고 */
        for (int v=cur; v; v=out[v])
            if (wordAt[v] >= 0) report(v, i);
    }
}

// KMP:
//   while (j>0 && T[i]!=P[j])
//       j = pi[j-1];
//   if (T[i]==P[j]) j++;

// 아호-코라식:
//   while (cur && !ch[cur][c])
//       cur = fail[cur];
//   cur = ch[cur][c];

// 완전히 같은 뼈대!
// j(위치) 대신 cur(노드)
💡 텍스트 인덱스 i는 절대 뒤로 안 갑니다. KMP에서 배운 그 원칙 그대로예요. 후퇴하는 건 상태(cur)뿐이고, 그마저도 실패 링크를 타고 이미 읽은 접미사를 살린 채 물러섭니다.

출력 링크 — 한 자리에서 여러 단어

she를 찾은 그 순간 he도 끝납니다. 두 단어가 같은 위치에서 동시에 끝나요. 이걸 놓치지 않으려면?

out[u] = fail[u]가 단어면 fail[u], 아니면 out[fail[u]]
왜 fail 체인을 따라가나 — u에서 끝나는 다른 단어가 있다면, 그건 반드시 u의 접미사입니다. 그리고 접미사들은 fail 링크 체인에 늘어서 있어요! 그래서 fail 체인을 따라가며 단어인 노드만 모으면 됩니다. 그 체인을 미리 계산해 둔 게 출력 링크예요.
우리 예제
// node5 = "she", 단어!
// fail[5] = node2 = "he", 단어!
//   → out[5] = 2

// node9 = "hers", 단어!
// fail[9] = node3 = "s", 단어 아님
//   → out[9] = out[3] = 없음

// 그래서 "she"에서만 덤이 나옴

⚠️ fail 체인 대신 out 체인을 쓰는 이유 — fail 체인엔 단어가 아닌 노드도 잔뜩 섞여 있습니다. 매 글자마다 그걸 다 훑으면 느려져요. 출력 링크는 단어 노드만 이어놓은 지름길이라, 보고 비용이 실제 매칭 개수(z)에만 비례합니다.

스캔 횟수 비교 — 숫자로

버튼을 눌러 패턴 수가 늘 때 어떻게 벌어지는지 보세요.
💡 아호–코라식은 패턴 수에 흔들리지 않습니다. 텍스트 스캔은 항상 1번이에요. 패턴이 늘면 트라이가 조금 커질 뿐(전처리 O(ΣM)), 스캔 시간은 그대로 O(N)입니다. 이게 금지어 필터·바이러스 검사에서 아호–코라식을 쓰는 이유예요.
