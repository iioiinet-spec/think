그래프 — 점과 선으로 세상을 그리다

지하철 노선도, 친구 관계, 웹 페이지 링크, 도시 간 도로망… 모두 그래프입니다. 그래프는 단 두 가지로 이루어집니다.

구성 요소뜻기호
정점 (vertex, node)대상 하나하나 — 역, 사람, 도시V (개수 n)
간선 (edge)둘 사이의 관계 — 노선, 친구, 도로E (개수 m)
💡 트리도 그래프의 한 종류입니다! "연결 + 사이클 없음"이라는 조건이 붙었을 뿐이죠. 그래프는 사이클이 있어도 되고, 끊어져 있어도 되는 더 자유로운 구조입니다.

방향이 있느냐 없느냐

그래프를 나누는 가장 중요한 기준입니다. 간선에 방향이 있는지 보세요.

구분무방향 그래프유방향 그래프
간선 의미u—v : 서로 오갈 수 있음u→v : u에서 v로만
예시친구 관계, 양방향 도로팔로우, 일방통행, 선수과목
저장양쪽 모두 저장 (2번)한쪽만 저장 (1번)
차수차수 (degree)진입차수 / 진출차수
⚠️ KOI 최다 실수 — 무방향 그래프인데 간선을 한쪽만 저장하는 것! "1 2"가 입력되면 adj[1]에 2와 adj[2]에 1을 둘 다 넣어야 합니다. 지문에서 "양방향", "서로", "연결되어 있다"가 보이면 무방향입니다.

기본 용어 — 실험 그래프로 익히기

이번 강의의 예제입니다. 정점 9개, 간선 8개인 무방향 그래프. 자세히 보면 두 덩어리로 나뉘어 있습니다!

용어뜻위 그림에서
인접 (adjacent)간선으로 직접 이어진 관계1과 2는 인접, 1과 4는 아님
차수 (degree)그 정점에 붙은 간선의 수deg(2) = 3, deg(4) = 1
경로 (path)간선을 따라 이동한 정점의 나열1 → 2 → 5 → 7
사이클 (cycle)출발점으로 되돌아오는 경로1 → 2 → 5 → 3 → 1
연결 요소서로 오갈 수 있는 정점들의 덩어리2개: {1~7}, {8,9}
💡 차수의 합 = 간선 수 × 2 — 간선 하나가 양 끝 정점의 차수를 1씩 올리기 때문입니다. 위 그래프: 차수 합 2+3+3+1+3+1+1+1+1 = 16, 간선 8개 × 2 = 16 ✓ (악수 정리, handshaking lemma)
⚠️ 연결 요소가 2개라는 점이 이 강의의 숨은 주제입니다. 1번에서 DFS를 돌려도 8, 9번에는 절대 도달하지 못합니다. 모든 정점을 훑으려면 시작점을 바꿔 가며 여러 번 탐색해야 해요 — 4번 탭에서 다룹니다.

그래프는 이렇게 입력된다

거의 모든 문제가 같은 형식입니다. 첫 줄에 정점 수 n과 간선 수 m, 이어서 m줄에 걸쳐 간선 하나씩.

INPUT — 이 강의의 실험 그래프
9 8 ← 정점 9개, 간선 8개 1 2 ← 1번과 2번이 이어짐 1 3 2 4 2 5 3 5 3 6 5 7 8 9 ← 나머지와 떨어진 덩어리!
⚠️ 트리와 다른 점 — 트리는 간선이 항상 n−1개였지만, 일반 그래프는 m이 따로 주어집니다. m이 n−1보다 작으면 끊어져 있고, 크면 사이클이 있습니다. 그래서 반드시 m을 읽어야 해요.

저장법 ① 인접 행렬 (Adjacency Matrix)

adj[i][j] = 1이면 i와 j가 이어져 있다는 뜻. 2차원 배열 하나로 끝납니다.

C — 인접 행렬
int adj[1005][1005];  /* 전역: 자동 0 */

scanf("%d %d", &n, &m);
for (int e = 0; e < m; e++) {
    int u, v;
    scanf("%d %d", &u, &v);
    adj[u][v] = 1;
    adj[v][u] = 1;   /* 무방향! */
}

/* u와 v가 인접한가? → O(1) */
if (adj[u][v]) { ... }

/* u의 이웃 모두 훑기 → O(n) */
for (int v = 1; v <= n; v++)
    if (adj[u][v]) { ... }
Python — 인접 행렬
n, m = map(int, input().split())
adj = [[0] * (n + 1) for _ in range(n + 1)]

for _ in range(m):
    u, v = map(int, input().split())
    adj[u][v] = 1
    adj[v][u] = 1      # 무방향!

# u와 v가 인접한가? → O(1)
if adj[u][v]:
    ...

# u의 이웃 모두 훑기 → O(n)
for v in range(1, n + 1):
    if adj[u][v]:
        ...

인접 행렬 보기 — 정점을 클릭하면 그 행이 강조됩니다

대칭이죠? 무방향 그래프의 인접 행렬은 항상 adj[i][j] == adj[j][i]인 대칭 행렬입니다. 유방향이면 대칭이 깨집니다. 대각선(자기 자신)은 보통 0이에요.

저장법 ② 인접 리스트 (Adjacency List)

각 정점마다 이웃 목록을 따로 들고 있는 방식. 실전에서 압도적으로 많이 씁니다.

C — 2차원 배열 방식
int adjv[100005][20];  /* 이웃 목록 */
int deg[100005];        /* 이웃 수 = 차수 */

scanf("%d %d", &n, &m);
for (int e = 0; e < m; e++) {
    int u, v;
    scanf("%d %d", &u, &v);
    adjv[u][deg[u]++] = v;
    adjv[v][deg[v]++] = u;  /* 무방향! */
}

/* u의 이웃 모두 훑기 → O(deg(u)) */
for (int k = 0; k < deg[u]; k++) {
    int v = adjv[u][k];
    ...
}
Python — 리스트의 리스트
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]

for _ in range(m):
    u, v = map(int, input().split())
    adj[u].append(v)
    adj[v].append(u)   # 무방향!

# 번호 순서로 방문하고 싶으면 정렬
for i in range(1, n + 1):
    adj[i].sort()

# u의 이웃 모두 훑기 → O(deg(u))
for v in adj[u]:
    ...

⚠️ 정렬을 잊지 마세요 — 문제가 "번호가 작은 정점부터 방문"을 요구하면 인접 리스트를 오름차순 정렬해야 합니다. 입력 순서대로 들어가 있을 뿐 정렬되어 있지 않으니까요! 이 강의의 모든 탐색은 작은 번호부터 방문합니다.

DFS — 한 길로 끝까지 파고들기

깊이 우선 탐색(Depth-First Search)은 갈 수 있는 데까지 깊이 들어갔다가, 막히면 되돌아와 다른 길을 봅니다. 미로에서 한 손을 벽에 대고 걷는 것과 비슷해요.

단계하는 일
①현재 정점을 방문 표시하고 처리한다
②이웃 중 아직 방문 안 한 정점 하나로 들어간다 (재귀)
③더 갈 곳이 없으면 되돌아온다 (함수 종료)
C — 재귀 DFS
int visited[MAXN];

void dfs(int u) {
    visited[u] = 1;          /* ① 방문 표시 */
    printf("%d ", u);
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (!visited[v]) dfs(v); /* ② 깊이 */
    }
    /* ③ 함수 종료 = 되돌아감 */
}

dfs(1);   /* 1번에서 출발 */
Python — 재귀 DFS
import sys
sys.setrecursionlimit(300000)  # 필수!

visited = [False] * (n + 1)

def dfs(u):
    visited[u] = True       # ① 방문 표시
    print(u, end=' ')
    for v in adj[u]:
        if not visited[v]:
            dfs(v)             # ② 깊이
    # ③ 함수 종료 = 되돌아감

dfs(1)                     # 1번에서 출발
⚠️ 트리와 결정적 차이 — visited 배열이 반드시 필요! 트리는 사이클이 없어 부모만 피하면 됐지만, 그래프는 사이클이 있어 같은 곳을 무한히 맴돌 수 있습니다. visited가 없으면 무한 루프!
💡 방문 표시는 언제? 함수에 들어가자마자 표시해야 합니다. 나중에 하면 같은 정점이 큐/스택에 여러 번 들어가 중복 방문합니다.

BFS — 가까운 곳부터 물결처럼

너비 우선 탐색(Breadth-First Search)은 출발점에서 가까운 순서대로 훑습니다. 연못에 돌을 던졌을 때 물결이 동심원으로 퍼지는 모습이에요. 도구는 큐(queue)입니다.

단계하는 일
①시작 정점을 방문 표시하고 큐에 넣는다
②큐에서 하나 꺼내 처리한다
③그 이웃 중 방문 안 한 것을 방문 표시하고 큐에 넣는다 — 큐가 빌 때까지 반복
C — 배열 큐 BFS
int visited[MAXN], q[MAXN];

void bfs(int s) {
    int head = 0, tail = 0;
    visited[s] = 1;          /* ① */
    q[tail++] = s;
    while (head < tail) {
        int u = q[head++];    /* ② 꺼내기 */
        printf("%d ", u);
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k];
            if (!visited[v]) {
                visited[v] = 1;  /* ③ 넣을 때! */
                q[tail++] = v;
            }
        }
    }
}
Python — deque BFS
from collections import deque

def bfs(s):
    visited = [False] * (n + 1)
    visited[s] = True          # ①
    q = deque([s])
    while q:
        u = q.popleft()        # ② 꺼내기
        print(u, end=' ')
        for v in adj[u]:
            if not visited[v]:
                visited[v] = True  # ③ 넣을 때!
                q.append(v)
⚠️ 방문 표시를 큐에 "넣을 때" 하라 — "꺼낼 때" 하면 같은 정점이 큐에 여러 번 들어갑니다. 예를 들어 4와 5가 모두 2의 이웃인데 둘 다 3의 이웃이기도 하면, 표시가 늦어 중복 삽입되죠. 넣는 순간 표시가 철칙입니다.
파이썬 주의 — 큐로 list.pop(0)을 쓰면 O(n)이라 느립니다. 반드시 collections.deque의 popleft()(O(1))를 쓰세요. 이 하나로 시간 초과가 갈립니다.

BFS의 선물 — 최단 거리가 공짜로

모든 간선의 길이가 같다면(가중치 1), BFS가 방문하는 순서가 곧 거리 순서입니다. 그래서 dist[v] = dist[u] + 1 한 줄만 추가하면 최단 거리를 얻습니다.

Python — 최단 거리 BFS
from collections import deque

dist = [-1] * (n + 1)   # -1 = 도달 불가
dist[s] = 0
q = deque([s])
while q:
    u = q.popleft()
    for v in adj[u]:
        if dist[v] == -1:      # 미방문
            dist[v] = dist[u] + 1
            q.append(v)
C — 최단 거리 BFS
int dist[MAXN];

for (int i = 1; i <= n; i++) dist[i] = -1;
dist[s] = 0;
int h = 0, t = 0;
q[t++] = s;
while (h < t) {
    int u = q[h++];
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (dist[v] == -1) {
            dist[v] = dist[u] + 1;
            q[t++] = v;
        }
    }
}
💡 dist 배열이 visited 역할까지 겸합니다. −1이면 아직 안 갔다는 뜻이니 별도 배열이 필요 없죠. 깔끔한 관용구입니다.
⚠️ DFS로는 최단 거리를 못 구합니다! DFS는 한 길로 깊이 파고들기 때문에 처음 도달한 경로가 최단이라는 보장이 없어요. "최단 거리/최소 이동 횟수" → 무조건 BFS입니다.

모든 정점을 훑으려면 — 시작점을 바꿔 가며

1번에서 DFS를 돌리면 8·9번은 영영 방문되지 않습니다. 그래프가 끊어져 있기 때문이죠. 해결법은 간단합니다: 모든 정점을 훑으며, 아직 방문 안 한 정점이 보이면 거기서 새로 탐색을 시작합니다.

C — 연결 요소 세기
int cnt = 0;
for (int s = 1; s <= n; s++) {
    if (!visited[s]) {
        cnt++;                /* 새 덩어리 발견! */
        printf("요소 %d: ", cnt);
        dfs(s);               /* 이 덩어리 전부 방문 */
        printf("\n");
    }
}
printf("연결 요소 개수: %d\n", cnt);
Python — 연결 요소 세기
cnt = 0
for s in range(1, n + 1):
    if not visited[s]:
        cnt += 1              # 새 덩어리 발견!
        print(f'요소 {cnt}: ', end='')
        dfs(s)                # 이 덩어리 전부 방문
        print()

print('연결 요소 개수:', cnt)
💡 DFS든 BFS든 상관없습니다. "한 덩어리를 전부 방문한다"는 목적만 달성하면 되니까요. 연결 요소의 개수를 세는 문제, 각 덩어리의 크기를 구하는 문제 모두 이 틀에서 나옵니다.

완성 코드 — 입력부터 탐색까지

C — 전체 프로그램
#include <stdio.h>
#include <stdlib.h>
#define MAXN 105

int n, m;
int adjv[MAXN][MAXN], deg[MAXN];
int visited[MAXN];
int q[MAXN];

void dfs(int u) {
    visited[u] = 1;
    printf("%d ", u);
    for (int k = 0; k < deg[u]; k++) {
        int v = adjv[u][k];
        if (!visited[v]) dfs(v);
    }
}

void bfs(int s) {
    int h = 0, t = 0;
    visited[s] = 1; q[t++] = s;
    while (h < t) {
        int u = q[h++];
        printf("%d ", u);
        for (int k = 0; k < deg[u]; k++) {
            int v = adjv[u][k];
            if (!visited[v]) {
                visited[v] = 1;
                q[t++] = v;
            }
        }
    }
}

int cmp(const void *a, const void *b) {
    return *(int*)a - *(int*)b;
}

input = sys.stdin.readline

n, m = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    u, v = map(int, input().split())
    adj[u].append(v)
    adj[v].append(u)          # 무방향

# 작은 번호부터 방문하려면 정렬
for i in range(1, n + 1):
    adj[i].sort()

print('차수:', *[len(adj[i]) for i in range(1, n+1)])

def dfs(u, visited, acc):
    visited[u] = True
    acc.append(u)
    for v in adj[u]:
        if not visited[v]:
            dfs(v, visited, acc)

def bfs(s):
    visited = [False] * (n + 1)
    visited[s] = True
    acc, q = [], deque([s])
    while q:
        u = q.popleft()
        acc.append(u)
        for v in adj[u]:
            if not visited[v]:
                visited[v] = True
                q.append(v)
    return acc

vis = [False] * (n + 1)

def dfs_iter(s):
    visited = [False] * (n + 1)
    acc, st = [], [s]
    while st:
        u = st.pop()
        if visited[u]: continue  # 중복 pop 방지
        visited[u] = True         # pop 때 표시!
        acc.append(u)
        # 작은 번호를 먼저 꺼내려면
        # 큰 번호를 먼저 push (역순)
        for v in reversed(adj[u]):
            if not visited[v]:
                st.append(v)
    return acc

# 결과: 1 2 4 5 3 6 7   ← 재귀와 일치!
⚠️ BFS와 정반대! BFS는 큐에 넣을 때 표시해야 하지만, 재귀와 순서를 맞추려는 스택 DFS는 꺼낼 때 표시해야 합니다. 왜냐하면 재귀는 "이웃을 하나씩 차례로 내려가며" 그때그때 방문하는데, push 때 표시하면 이웃 전부를 한꺼번에 확정해 버려 나중 순서가 뒤틀리기 때문입니다.
💡 대가 — pop 때 표시하면 같은 정점이 스택에 여러 번 들어갈 수 있어 if visited[u]: continue가 필수이고, 스택 크기가 최대 O(E)까지 커집니다. 순서가 상관없는 문제라면 왼쪽의 순진한 버전도 완전히 올바른 DFS입니다 — 더 간단하고 메모리도 O(V)죠.
