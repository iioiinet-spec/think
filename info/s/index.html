특수수로 만든 나만의 암호
파이썬 코드로 하나씩 이해하기

완전수·과잉수·부족수라는 정수론 개념으로 직접 정의한 특수수 3종과, 그것을 만들어내는 파이썬 코드를 완전히 분해합니다.

이 연구는 무엇을 만들었나?

은행 OTP(몇 분마다 바뀌는 6자리 비밀번호)에서 출발한 궁금증이, "나만의 수로 나만의 암호를 만들어 보자"는 연구가 되었습니다. 핵심 아이디어는 두 겹의 매핑(이중 매핑)입니다.

① 기초 암호 A한글·영어·기호 → 특수수(숫자)글자를 나만의 특수수로 바꿈
② 기초 암호 B그 숫자 → 달 그림으로 다시 변환시기(기간)에 따라 규칙이 자동으로 바뀜

글자를 그냥 숫자로 바꾸는 것을 넘어, "어떤 특수수인지 그 조건을 모르면 규칙을 못 찾고", 게다가 "달 그림 대응이 기간마다 바뀌어" 두 겹으로 숨겼습니다. OTP처럼 시간에 따라 변하는 암호를 수학으로 흉내 낸 셈입니다.

이 강의자료의 목표

이 연구의 수학적 뼈대와 파이썬 코드를 초등학생도 한 줄씩 이해할 수 있도록 풀어냅니다. 특히 특수수를 만드는 코드를 완전히 분해하고, 직접 숫자를 넣어 확인하는 인터랙티브 도구를 담았습니다.

2 3가지 특수수유준수·수현수·수준수의 조건1~200 특수수 지도 (색칠 그리드)

모든 것의 출발점 — 진약수의 합

이 연구의 특수수들은 전부 "진약수의 합" 하나에서 갈라집니다. 진약수란 자기 자신을 뺀 약수입니다.

진약수의 합 S(n) = (n의 약수 중 자기 자신 n을 뺀 것들의 합)

예를 들어 12의 약수는 1, 2, 3, 4, 6, 12인데, 12를 뺀 진약수는 1, 2, 3, 4, 6이고 그 합은 16입니다. 이 S(n)을 n과 비교하면 수의 성격이 정해집니다.

부족수 (deficient)
S(n) < n
예: 8진약수 1+2+4 = 7 < 8 → 부족수
과잉수 (abundant)
S(n) > n
예: 12진약수 1+2+3+4+6 = 16 > 12 → 과잉수
완전수 (perfect)
S(n) = n
예: 6진약수 1+2+3 = 6 = 6 → 완전수 (6, 28, 496 …)
제곱수 (square) — 별도 조건
n = k × k
예: 36= 6×6 → 제곱수 (특수수에서 제외됨)
🔑
연구에서 유준수·수현수는 부족수를, 수준수는 과잉수를 골랐습니다. 부족수는 개수가 많아 다양한 글자에 대응하기 좋고, 과잉수는 상대적으로 적어 숫자·기호용으로 알맞습니다.

파이썬 코드, 한 줄씩 완전분해

이제 연구의 심장인 유준수 찾기 코드를 한 줄씩 뜯어봅니다. 겁먹지 마세요 — 앞에서 배운 정의를 그대로 코드로 옮긴 것뿐입니다.

divisors_sum = 0
n의 진약수를 더할 그릇을 0으로 준비합니다.
for i in range(1, n):

if n % i == 0:
n을 i로 나눈 나머지가 0이면 → i는 약수입니다.
divisors_sum += i
약수 i를 그릇에 더합니다. 이 반복이 끝나면 진약수의 합 완성.
if divisors_sum >= n:
진약수 합이 n 이상이면 부족수가 아니므로…

자리 숫자들을 곱할 그릇 (곱셈이므로 1로 시작!).
for digit in s_n:

Python yujun.py — 유준수(한글용) 찾기
# 101 미만 자연수 중 '유준수' 찾기
yujun_numbers = []

for n in range(1, 101):
    # 1. 부족수인지 확인 (진약수의 합 < 자신)
    divisors_sum = 0
    for i in range(1, n):
        if n % i == 0:
            divisors_sum += i
    if divisors_sum >= n:
        continue              # 부족수 아니면 탈락
    # 2. 제곱수가 아니어야 함
    if (n**0.5) == int(n**0.5):
        continue              # 제곱수면 탈락
    # 3. 각 자릿수의 곱이 두 자릿수인가?
    s_n = str(n)
    digit_product = 1
    for digit in s_n:
        digit_product *= int(digit)
    if 10 <= digit_product <= 99:
        yujun_numbers.append(n)   # 조건 통과!

print(f"유준수 리스트: {yujun_numbers}")
print(f"총 개수: {len(yujun_numbers)}개")
실행 결과유준수 리스트: [26, 27, 29, 34, 35, 37, 38, 39, 43, 44, 45, 46, 47, 52, 53, 55, 57, 58, 59, 62, 63, 65, 67, 68, 69, 73, 74, 75, 76, 77, 79, 82, 83, 85, 86, 87, 89, 92, 93, 94, 95, 97, 98, 99] 총 개수: 44개
💡
수현수·수준수 코드도 구조가 똑같습니다! 딱 세 군데만 바꾸면 됩니다: ① 범위(range) ② 부족수/과잉수 부등호 방향(>= ↔ <=) ③ 자리곱 조건(두 자릿수 10~99 ↔ 한 자릿수 <10). 수현수는 여기에 "합성수(소수 아님)" 확인만 추가됩니다.

직접 검증 — 보고서의 "수준수 12개"가 정말 맞을까?

진짜 과학자처럼, 우리도 코드로 확인해 봅시다. 보고서는 수준수(과잉수)가 12개라고 했지만, 조건을 바꾼 코드를 돌리면…

Python sujun.py — 수준수(숫자용) 찾기
# 101 미만 자연수 중 '수준수' 찾기 (과잉수 버전)
sujun_numbers = []
for n in range(1, 101):
    check_sum = 0
    for i in range(1, n):
        if n % i == 0:
            check_sum += i
    if check_sum <= n:
        continue              # 과잉수 아니면 탈락 (합 > 자신이어야)
    if (n**0.5) == int(n**0.5):
        continue
    prod = 1
    for ch in str(n):
        prod *= int(ch)
    if prod < 10:            # 자리곱이 한 자릿수
        sujun_numbers.append(n)

print(f"수준수: {sujun_numbers}")
print(f"총 개수: {len(sujun_numbers)}개")
실행 결과수준수: [12, 18, 20, 24, 30, 40, 42, 60, 70, 80, 90] 총 개수: 11개
🕵️
발견! 코드는 11개를 냅니다 (50이 빠짐). 보고서 표에 있던 50은 진약수 합이 1+2+5+10+25 = 43 < 50 이라 부족수이므로 과잉수 조건에 맞지 않습니다. 코드가 사람의 손계산 실수를 잡아낸 것! 1번 탭 판별기에서 "문제의 50"으로 직접 확인할 수 있습니다. 이것이 바로 코드로 검증하는 이유입니다 — 연구자가 AI로 수현수 3개(178·182·187)를 더 찾은 것과 똑같은 정신이죠.

이중 매핑 암호화 — 글자가 암호가 되기까지

이제 완성된 특수수로 실제 암호화를 해 봅니다. 두 단계입니다: ① 글자 → 특수수(기초 암호 A), 그다음 ② 각 숫자 → 달 그림 대응으로 변환(기초 암호 B).

기초 암호 B — 달 그림 이중 대응이란?

연구자는 0~9 숫자를 달의 위상 그림에 두 번 대응시켰습니다. 아래 표에서 위쪽 숫자를 아래쪽 숫자로 바꾸는 규칙입니다. (기간이 바뀌면 이 대응도 바뀌어 OTP처럼 작동!)

A 숫자7290538146
달 그림🌑🌒🌓🌔🌕🌖🌗🌘🔴🔵
B 숫자3816094275

즉 7→3, 2→8, 9→1, 0→6, 5→0, 3→9, 8→4, 1→2, 4→7, 6→5 로 각 자리 숫자를 바꿉니다.

보고서의 예시: 기초 암호 A 73 95 43 → 각 자리 변환 → 기초 암호 B 39 10 79

7→3, 3→9 / 9→1, 5→0 / 4→7, 3→9

🔐
왜 이게 안전할까? 최종 암호 숫자만 봐서는 ① 그게 어떤 특수수인지(조건을 모르면 못 찾음) ② 달 그림 대응이 무엇인지(기간마다 바뀜) 둘 다 알 수 없습니다. 두 겹의 비밀이 겹쳐 해독을 매우 어렵게 만드는 것이 이 연구의 핵심 아이디어입니다.
💭
더 생각해볼 점 (중·고등학교로 이어지는 질문): 이 암호는 같은 글자가 항상 같은 숫자가 되는 단일 치환 방식이라, 긴 글에서는 자주 나오는 글자의 빈도로 추측될 수 있습니다. 그래서 실제 현대 암호(AES, RSA 등)는 이런 빈도 분석까지 막도록 설계됩니다. 이 연구는 그 첫걸음으로 아주 훌륭합니다!

확인 퀴즈 — 8문제

보기를 클릭하면 바로 채점됩니다. 초록색이 정답, 빨간색이 오답입니다.

맞힌 개수: 0 / 8
112의 진약수(자기 자신 제외)를 모두 더하면?
12의 진약수는 1, 2, 3, 4, 6이고 합은 16입니다. 16 > 12 이므로 12는 과잉수입니다.
2진약수의 합이 자기 자신과 같은 수를 무엇이라 하나?
S(n) = n이면 완전수입니다. 6(1+2+3), 28(1+2+4+7+14)이 대표적입니다.
3유준수의 조건이 아닌 것은?
유준수는 부족수입니다. 과잉수를 쓰는 것은 수준수(숫자용)입니다.
4코드에서 for i in range(1, n): 이 1부터 n−1까지만 도는 이유는?
진약수 = 자기 자신을 제외한 약수. 그래서 n은 포함하지 않고 1~(n−1)만 검사합니다.
526의 각 자리 숫자의 곱(digit_product)은?
2 × 6 = 12. 두 자릿수(10~99)이므로 26은 유준수의 자리곱 조건을 만족합니다.
6보고서의 "수준수 50"을 코드로 확인하면 왜 빠질까?
50의 진약수 1+2+5+10+25 = 43 < 50. 부족수이므로 과잉수 조건(수준수)에 맞지 않아 코드가 정확히 제외합니다.
7이 암호가 "이중 매핑"이라 불리는 이유는?
기초 암호 A(글자→특수수)와 기초 암호 B(숫자→달 그림 변환)를 겹쳐 두 겹으로 숨기기 때문입니다.
8기초 암호 B 규칙(7→3, 5→0)으로 75를 변환하면?
각 자리를 따로 변환: 7→3, 5→0 이므로 75 → 30. 보고서의 표 7 대응 규칙 그대로입니다.
