CONCEPT집합의 분할 — 세 가지 조건
집합 S를 여러 부분집합으로 나누되, 아래 세 조건을 모두 만족해야 "분할"입니다.

01분할의 정의

① 각 부분집합은 비어 있으면 안 된다.

② 부분집합끼리 서로 겹치면 안 된다 (서로소).

③ 모두 합치면 원래 집합 S가 되어야 한다.

S = {1,2,3,4,5,6}
{1,2}, {3,4,5}, {6}   → 세 조건을 모두 만족 → 분할이다! ✔

핵심 한 줄: 집합의 분할 = 서로소 집합들의 모음 — 그래서 알고리즘 이름이 "서로소 집합(Disjoint Set)"입니다.

UNION-FINDUnion-Find 실험실
각 집합마다 대표자를 정하고, find(a) == find(b)로 같은 집합인지 확인합니다. 학생 6명으로 직접 실험해 보세요.

01시뮬레이터 — union · find · 사이클 판별까지

INTERACTIVE · 학생 1~6 (처음엔 모두 혼자 팀)
parent 배열
현재 분할 상태
union으로 팀을 합치고, find로 같은 팀인지 확인해 보세요.

💡 이미 같은 집합인 두 원소를 union하면 "사이클 발생!"이 뜹니다 — 크루스칼 알고리즘이 간선을 버리는 바로 그 순간입니다(유형 4·5).

02핵심 아이디어 정리

find(x)      : x가 속한 집합의 대표자를 찾는다
union(a, b)  : a의 집합과 b의 집합을 합친다 (대표자 밑에 붙이기)
find(a) == find(b)  →  같은 집합!

union(2, 3)을 하면 "2와 3"이 아니라 2의 대표자와 3의 대표자가 연결됩니다. 위 시뮬레이터의 parent 배열에서 직접 확인해 보세요.

CODEC 코드 & DFS 비교
기본 코드 → 경로 압축 → 그룹 개수 → 그룹 크기 순서로 확장합니다.

01기본 Union-Find (C)

int parent[1001];

int find(int x) {
    if (parent[x] == x) return x;
    return find(parent[x]);
}

void union_set(int a, int b) {
    int rootA = find(a);
    int rootB = find(b);
    if (rootA != rootB) {
        parent[rootB] = rootA;   // B의 대표를 A의 대표 밑에
    }
}

02경로 압축 최적화

int find(int x) {
    if (parent[x] == x) return x;
    parent[x] = find(parent[x]);   // 부모를 대표자로 바로 바꿈!
    return parent[x];
}

find를 하면서 지나간 모든 노드가 대표자에 직접 매달리게 되어, 트리가 납작해지고 다음 find가 매우 빨라집니다.

03그룹 개수 · 그룹 크기

// 그룹 개수: 자기 자신이 대표인 원소를 센다
int count = 0;
for (int i = 1; i <= n; i++) {
    if (find(i) == i) count++;
}

// 그룹 크기: size 배열을 함께 관리
if (rootA != rootB) {
    parent[rootB] = rootA;
    size[rootA] += size[rootB];   // 크기를 합친다
}
// 입력: 6 3 / 1 2 / 2 3 / 4 5  →  출력:
대표 1 : 크기 3
대표 4 : 크기 2
대표 6 : 크기 1

04DFS로 연결 요소 세기 & 비교 [보완: 복잡도 명시]

void dfs(int x) {
    visited[x] = 1;
    for (int i = 1; i <= n; i++) {
        if (graph[x][i] == 1 && visited[i] == 0) dfs(i);
    }
}
// 방문 안 한 정점에서 dfs를 시작할 때마다 groupCount++
구분Union-FindDFS/BFS
목적같은 집합인지 빠르게 확인연결된 모든 원소 탐색
동적 관계 추가좋음매번 다시 탐색 필요
그룹 전체 출력추가 작업 필요쉽다

위 DFS 코드는 인접 행렬(graph[101][101])이라 O(N²)입니다. N이 크면 인접 리스트로 바꿔 O(N+M)로 만들어야 합니다.

ENUMERATE가능한 모든 분할 만들기 (백트래킹)
원소를 순서대로, "기존 그룹에 넣기" 또는 "새 그룹 만들기" — 새 그룹은 항상 맨 뒤에만 만들어 중복을 막습니다.

01백트래킹 코드 (교재 §21과 동일한 출력 순서)

void backtrack(int x, int groupCount) {
    if (x > n) { printGroups(groupCount); return; }

    for (int i = 0; i < groupCount; i++) {   // ① 기존 그룹에 넣기
        group[i][groupSize[i]++] = x;
        backtrack(x + 1, groupCount);
        groupSize[i]--;
    }
    group[groupCount][0] = x;                // ② 새 그룹 만들기
    groupSize[groupCount] = 1;
    backtrack(x + 1, groupCount + 1);
    groupSize[groupCount] = 0;
}

[보완] 배열이 group[10][10]이므로 n ≤ 9까지만 안전합니다. 새 그룹을 항상 맨 뒤에만 만들기 때문에 {1},{2}와 {2},{1}이 중복 출력되지 않습니다.

02분할 생성기 — 코드와 똑같은 순서로!

INTERACTIVE · n과 그룹 개수 필터를 바꿔 보세요

K를 고르면 "정확히 K개 그룹" 분할만 남고, 그 개수가 바로 스털링 수 S(n, K)입니다. 전체 개수는 벨 수 B(n)!

COUNT스털링 수 S(n,k) · 벨 수 B(n)
S(n,k) = n개를 정확히 k개의 비어있지 않은 그룹으로 나누는 경우의 수. 벨 수는 그 합입니다.

01점화식 — n번째 원소의 선택

S(n, k) = S(n-1, k-1)  +  k × S(n-1, k)
          └ 혼자 새 그룹      └ 기존 k개 그룹 중 하나에 합류
long long dp[31][31];
dp[0][0] = 1;
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= k; j++)
        dp[i][j] = dp[i-1][j-1] + (long long)j * dp[i-1][j];

02S(n,k) 표 탐험기 — 셀을 클릭해 보세요!

INTERACTIVE · 셀을 클릭하면 점화식 분해를 보여줍니다 (마지막 열 = 벨 수)
표의 셀을 클릭하면 S(n,k) = S(n-1,k-1) + k×S(n-1,k) 분해가 표시됩니다.

확인해 보세요: S(3,2)=3 (분할 생성기의 K=2 결과와 일치!), S(5,2)=15, B(4)=15, B(5)=52.
