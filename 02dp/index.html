DP(다이나믹 프로그래밍)란?

DP는 거창한 것이 아닙니다. "큰 문제를 작은 문제로 쪼개고, 작은 문제의 답을 표에 저장해 재사용한다" — 이 한 문장이 전부입니다. 우리가 1·2편에서 다룬 점화식이 바로 그 "쪼개는 규칙"입니다.

왜 저장(메모)이 필요한가 — 재귀 호출 횟수 폭발

피보나치를 저장 없이 재귀로 짜면 같은 계산이 수없이 반복됩니다. F(5)를 부르면 F(3)이 2번, F(2)가 3번… n을 키워 보세요.

"조합적 점화식"이란?

1·2편의 피보나치는 값 하나가 커지는 점화식이었습니다. 이번 편은 경우의 수·최댓값을 세는 점화식입니다. "마지막 한 수(마지막 계단, 마지막 괄호, 마지막 물건)를 어떻게 두었는지로 경우를 나누면, 더 작은 같은 문제가 나타난다" — 이것이 조합적 점화식을 세우는 공통 요령입니다.

워밍업 ① 계단 오르기 — 1차원 DP

한 번에 1칸 또는 2칸씩 오를 때, n번째 계단까지 가는 방법의 수. 마지막 걸음으로 경우 나누기: 마지막이 1칸이면 (i-1)에서 왔고, 2칸이면 (i-2)에서 왔다.

dp[i] = dp[i-1] + dp[i-2] ,   dp[1] = 1, dp[2] = 2

워밍업 ② 격자 경로 — 2차원 DP

왼쪽 위에서 출발해 오른쪽 또는 아래로만 이동할 때 오른쪽 아래까지 가는 경로 수. 각 칸에 오는 마지막 한 걸음은 위에서 or 왼쪽에서 — 두 경우뿐입니다.

dp[r][c] = dp[r-1][c] + dp[r][c-1] ,   dp[0][0] = 1 (출발점)

도착 칸의 값: -

채우는 순서가 중요한 이유

dp[r][c]를 계산하려면 위 칸과 왼쪽 칸이 먼저 완성되어 있어야 합니다. 그래서 왼쪽 위 → 오른쪽 아래로(행 우선) 채웁니다. "내가 참조하는 칸은 나보다 먼저 채워져야 한다" — 모든 DP의 순서 규칙입니다.

카탈란 수 — 올바른 괄호가 낳은 마법의 수열

카탈란 수 Cn은 "여는 괄호 n개, 닫는 괄호 n개로 만드는 올바른 괄호열의 개수"입니다. 1, 1, 2, 5, 14, 42, 132, … — 전혀 달라 보이는 문제들이 전부 이 수열로 수렴합니다.

n = 3 의 다섯 가지 (C₃ = 5)

((())) (()()) (())() ()(()) ()()()

빠짐없이, 겹침 없이 정확히 5개 — 직접 나열하며 확인해 보세요.

점화식 유도 — "첫 괄호 짝"으로 경우 나누기

모든 올바른 괄호열은 맨 앞의 여는 괄호와 그 짝을 기준으로 딱 한 가지로 분해됩니다:

( A ) B   — A: 짝 안쪽 (쌍 i개),  B: 짝 바깥 나머지 (쌍 n-1-i개)

안쪽 A가 i쌍이면 바깥 B는 n-1-i쌍. i가 0부터 n-1까지 모든 경우를 더하면:

Cn = Σi=0n-1 Ci · Cn-1-i ,    C0 = 1

1편의 시그마, 그리고 곱의 법칙(안쪽 경우 × 바깥 경우)이 결합된 조합적 점화식입니다.

카탈란 계산기 — 점화식 전개 단계 실행

카탈란 수로 치환되는 대표 문제들 (패턴 인식이 실력!)

문제대응 관계
올바른 괄호열 n쌍정의 그 자체
노드 n개 이진 탐색 트리(BST)의 개수루트 왼쪽 서브트리 i개 × 오른쪽 n-1-i개 → 같은 점화식
볼록 (n+2)각형의 삼각형 분할한 변이 속한 삼각형으로 양쪽 분할 → 같은 점화식
대각선을 넘지 않는 격자 경로y = x 위로 안 올라가는 경로 = Cn
스택으로 만들 수 있는 출력 순열push=( , pop=) 로 번역하면 괄호열

대회 요령: 작은 경우를 손으로 세어 1, 2, 5, 14, 42가 나오면 "아, 카탈란!" — 점화식을 새로 증명할 필요 없이 치환합니다.

닫힌 공식도 있다

Cn = 2nCn / (n+1)     예) C₄ = ₈C₄/5 = 70/5 = 14

이항계수로 O(n)에 계산 가능하지만, mod가 소수가 아닐 때 나눗셈이 곤란하므로 대회에서는 점화식 DP가 더 안전한 경우가 많습니다.

배낭 문제(0/1 Knapsack) — 최적화 점화식

무게 한도 K인 배낭에 물건들을 넣어 가치의 최댓값을 구합니다. 각 물건은 넣거나(1) 안 넣거나(0) 둘 중 하나. "경우의 수 덧셈"이 "최댓값 max"로 바뀐 것 말고는 지금까지와 똑같은 점화식 설계입니다.

3단계 설계

① 상태dp[i][w] = 앞에서 i번째 물건까지 고려했고, 무게 한도가 w일 때의 최대 가치
② 점화식dp[i][w] = max( dp[i-1][w] (안 넣음), dp[i-1][w-wi] + vi (넣음, w ≥ wi일 때만) )
③ 초기값dp[0][w] = 0 (물건이 없으면 가치 0), i 오름차순으로 채움

공간 절약 — 1차원 배열 최적화 (고득점 포인트)

for (int i = 1; i <= N; i++)
    for (int w = K; w >= wt[i]; w--)   /* 반드시 큰 쪽 → 작은 쪽! */
        dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);

dp[i-1][⋅]와 dp[i][⋅]를 한 배열에 겹쳐 쓰는 기법. w를 역방향으로 돌아야 dp[w-wt[i]]가 아직 "이전 물건까지"의 값이라는 보장이 유지됩니다. 정방향으로 돌면 같은 물건을 여러 번 넣는 배낭(완전 배낭)이 되어 버립니다 — 두 문제의 차이가 순회 방향 하나!

부분수열 점화식 ① LIS — 최장 증가 부분수열

수열에서 순서를 유지하며 골라낸, 값이 증가하는 가장 긴 부분수열의 길이. 핵심 질문: "i번째 원소로 끝나는 LIS는 얼마나 긴가?"

dp[i] = max( dp[j] + 1 )   (단, j < i 이고 a[j] < a[i]) ,   기본값 dp[i] = 1

부분수열 점화식 ② LCS — 최장 공통 부분수열

두 문자열이 공통으로 가지는 가장 긴 부분수열. 2차원 상태가 필요합니다: dp[i][j] = A의 앞 i글자와 B의 앞 j글자의 LCS 길이. 마지막 글자로 경우를 나눕니다.

A[i] = B[j] 이면   dp[i][j] = dp[i-1][j-1] + 1  (대각선 + 1)
A[i] ≠ B[j] 이면   dp[i][j] = max( dp[i-1][j], dp[i][j-1] )  (위, 왼쪽 중 최대)

2단계 — 오늘 배운 점화식들의 C 코드 사전

    

      

        
카탈란 수 (DP)

        
long long C[35];
C[0] = 1;
for (int n = 1; n <= 30; n++)
    for (int i = 0; i < n; i++)
        C[n] += C[i] * C[n-1-i];

      

      

        
격자 경로

        
dp[0][0] = 1;
for (int r = 0; r < H; r++)
  for (int c = 0; c < W; c++) {
    if (r) dp[r][c] += dp[r-1][c];
    if (c) dp[r][c] += dp[r][c-1];
  }

      

      

        
0/1 배낭 (2차원)

        
for (int i = 1; i <= N; i++)
  for (int w = 0; w <= K; w++) {
    dp[i][w] = dp[i-1][w];
    if (w >= wt[i] &&
        dp[i-1][w-wt[i]] + val[i] > dp[i][w])
      dp[i][w] = dp[i-1][w-wt[i]] + val[i];
  }

      

      

        
LIS O(n²)

        
int best = 0;
for (int i = 0; i < n; i++) {
    dp[i] = 1;
    for (int j = 0; j < i; j++)
        if (a[j] < a[i] && dp[j]+1 > dp[i])
            dp[i] = dp[j] + 1;
    if (dp[i] > best) best = dp[i];
}

      

    

    
LCS

    
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++) {
        if (A[i-1] == B[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
        else dp[i][j] = dp[i-1][j] > dp[i][j-1] ? dp[i-1][j] : dp[i][j-1];
    }

    

      다섯 코드 모두 구조가 같습니다: 초기값 → 의존성을 지키는 순서의 반복문 → 점화식 한 줄.
      DP 코딩 실력은 곧 점화식을 정확히 옮겨 적는 실력입니다.
    

  







  
자주 하는 실수 포인트


  

    
❌ 실수 1 — 초기값(base case) 누락·오류

    
계단에서 dp[2] = 2인데 1로 두거나, 카탈란에서 C₀ = 1을 0으로 두면 전체가 무너집니다. 점화식은 맞는데 답이 틀리면 가장 먼저 초기값을 의심하세요.

  


  

    
❌ 실수 2 — 채우는 순서(의존성) 위반

    
dp[i]를 계산할 때 아직 안 채워진 칸을 참조하면 쓰레기값을 읽습니다. 2차원이면 특히 주의: 격자·LCS는 행 우선(왼쪽 위 → 오른쪽 아래)이 안전합니다.

  


  

    
❌ 실수 3 — 1차원 배낭에서 정방향 순회

    
for (int w = wt[i]; w <= K; w++)   /* 0/1 배낭인데 정방향! */
    dp[w] = max(dp[w], dp[w-wt[i]] + val[i]);

    
같은 물건이 여러 번 들어가는 완전 배낭이 되어 버립니다. 0/1 배낭은 반드시 w를 K부터 역방향으로.

  


  

    
❌ 실수 4 — LIS에서 dp[i] = 1 초기화 생략

    
앞에 자기보다 작은 원소가 하나도 없으면 자기 혼자로 길이 1이 기본값입니다. 0으로 두면 그 원소로 시작하는 수열이 전부 1씩 짧게 계산됩니다.

  


  

    
❌ 실수 5 — 자료형: 경우의 수는 순식간에 폭발

    
카탈란 C₂₀ = 6,564,120,420 — 이미 int 초과. 격자 경로도 20×20이면 천억 단위입니다. long long + (문제가 요구하면) mod는 기본 장착.

  


  

    
❌ 실수 6 — 저장 없는 재귀로 제출

    
1탭에서 본 것처럼 호출 횟수가 지수적으로 폭발합니다. 재귀로 짜고 싶다면 메모이제이션 배열(계산했으면 즉시 반환)을 반드시 붙이세요. 반복문 DP와 성능이 같아집니다.
