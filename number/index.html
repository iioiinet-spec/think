수의 전체 체계 — 러시아 인형처럼 포개져 있다

수의 집합은 자연수 ⊂ 정수 ⊂ 유리수 ⊂ 실수로 차곡차곡 포개져 있고, 실수 안에서 유리수가 아닌 나머지가 무리수입니다. 아래 그림에서 각 상자가 바깥 상자에 완전히 포함되는 것에 주목하세요.

각 집합의 정의

집합정의예
자연수 ℕ개수를 셀 때 쓰는 수 1, 2, 3, …1, 7, 2026
정수 ℤ자연수 + 0 + 음의 정수-3, 0, 5
유리수 ℚ두 정수의 비 p/q (q ≠ 0)로 나타낼 수 있는 수2/3, -7/2, 0.75
무리수실수 중 분수로 나타낼 수 없는 수√2, π, e
실수 ℝ유리수 ∪ 무리수 = 수직선 위의 모든 점위 전부
🔍
소수 표현으로 한 방에 구분! 소수로 썼을 때 유한소수이거나 순환소수(0.333…, 0.121212…)면 유리수, 순환하지 않는 무한소수면 무리수입니다. 0.101001000100001… 은 규칙은 있어도 같은 마디가 반복되지 않으므로 무리수입니다.
⚠️
겉모습에 속지 말 것! √9 는 근호가 있지만 √9 = 3 이므로 자연수입니다. 근호를 벗겨서 가장 간단한 형태로 만든 뒤 판별해야 합니다. 참고로 실수 바깥에는 x² = −1의 해 같은 허수를 포함하는 복소수 ℂ가 있습니다(실수 ⊂ 복소수).

유리수의 사칙연산 — 분수 계산의 4가지 공식

유리수는 항상 p/q 꼴로 쓸 수 있으므로, 유리수의 사칙연산은 곧 분수의 사칙연산입니다. (아래에서 b ≠ 0, d ≠ 0)

덧셈 — 통분해서 분자끼리
a/b + c/d = (ad + bc) / bd
예제2/3 + 1/6 = (2·6 + 1·3)/18 = 15/18 = 5/6
뺄셈 — 통분해서 분자끼리
a/b − c/d = (ad − bc) / bd
예제3/4 − 1/6 = (18 − 4)/24 = 14/24 = 7/12
곱셈 — 분자끼리, 분모끼리
a/b × c/d = ac / bd
예제3/4 × 8/9 = 24/36 = 2/3
나눗셈 — 역수를 곱한다
a/b ÷ c/d = a/b × d/c = ad / bc
예제5/6 ÷ 10/9 = 5/6 × 9/10 = 45/60 = 3/4
🔗
정수론과의 연결고리 — 기약분수. 계산 결과는 분자·분모를 최대공약수(GCD)로 나눠 기약분수로 만듭니다. 15/18은 GCD(15, 18) = 3으로 약분해 5/6. GCD를 빠르게 구하는 유클리드 호제법이 바로 정수론의 출발점입니다.
🔒
유리수는 사칙연산에 닫혀 있다(closure). 유리수끼리 더하고, 빼고, 곱하고, (0이 아닌 수로) 나눠도 결과는 항상 유리수입니다. 위 공식의 결과가 전부 (정수)/(정수) 꼴인 것이 그 증거입니다.

분수 계산기 — 과정까지 보여줍니다

분자와 분모(정수)를 넣고 연산을 고르면, 공식 적용 → GCD로 약분 과정을 그대로 보여줍니다.

무리수의 사칙연산 — 근호(√)를 다루는 법

무리수 연산의 대표는 근호가 있는 수의 계산입니다. 곱셈·나눗셈은 근호 안에서 자유롭게, 덧셈·뺄셈은 동류 근호끼리만 — 이 두 줄이 전부입니다. (아래에서 a > 0, b > 0)

곱셈 — 근호 안끼리 곱한다
√a × √b = √(ab)
예제√6 × √10 = √60 = √(4·15) = 2√15
나눗셈 — 근호 안끼리 나눈다
√a ÷ √b = √(a/b)
예제√18 ÷ √2 = √9 = 3
덧셈·뺄셈 — 동류 근호끼리만!
m√a ± n√a = (m ± n)√a
예제3√2 + 5√2 = 8√2  (√2 + √3 은 더 못 줄임)
근호 간단히 하기
√(k²·a) = k√a
예제√12 = √(4·3) = 2√3,  √48 = 4√3
분모의 유리화 ①
1/√a = √a / a
예제1/√2 = √2/2 = 0.7071…
분모의 유리화 ② — 켤레 곱하기
1/(√a−√b) = (√a+√b)/(a−b)
예제1/(√3−√2) = (√3+√2)/(3−2) = √3+√2
합차공식: (√a + √b)(√a − √b) = a − b

유리화 ②가 성립하는 이유가 바로 이 합차공식입니다. 켤레(부호만 바꾼 짝)를 곱하면 근호가 사라지고 유리수가 됩니다. 예: (2 + √3)(2 − √3) = 4 − 3 = 1.

⚠️
무리수는 사칙연산에 닫혀 있지 않다! 유리수와 결정적으로 다른 점입니다. √2 + (−√2) = 0 (유리수), √2 × √2 = 2 (유리수). 무리수끼리 연산했는데 결과가 유리수가 될 수 있습니다. 반면 (0이 아닌) 유리수 + 무리수, 유리수 × 무리수는 항상 무리수입니다.
🔗
정수론과의 연결고리 — √2가 무리수인 이유. √2 = p/q (기약분수)라 가정하면 p² = 2q²이 되어 p²이 짝수 → p도 짝수 → q도 짝수가 되어 "기약"이라는 가정과 모순! 이 귀류법 증명은 소인수분해의 유일성(산술의 기본정리)에 기대고 있습니다.

종합 예제 — 한 문제에 규칙 4개

√12 + √27 − 6/√3 = 2√3 + 3√3 − 2√3 = 3√3

① √12 = 2√3  ② √27 = 3√3  ③ 6/√3 = 6√3/3 = 2√3 (유리화)  ④ 동류 근호끼리 (2+3−2)√3

지수의 사칙연산 — 지수법칙 6가지

an은 a를 n번 곱한 것. 여기서 모든 법칙이 나옵니다. 핵심 감각은 "곱셈이 지수의 덧셈으로, 거듭제곱이 지수의 곱셈으로 내려온다"는 것입니다. (a ≠ 0, b ≠ 0)

① 곱셈 → 지수의 덧셈
a^m × a^n = a^(m+n)
예제2³ × 2⁴ = 2⁷ = 128
② 나눗셈 → 지수의 뺄셈
a^m ÷ a^n = a^(m−n)
예제3⁵ ÷ 3² = 3³ = 27
③ 거듭제곱의 거듭제곱 → 지수의 곱셈
(a^m)^n = a^(mn)
예제(3²)³ = 3⁶ = 729
④ 곱·몫의 거듭제곱 → 각각에 분배
(ab)^n = a^n b^n,  (a/b)^n = a^n/b^n
예제(2×5)³ = 2³×5³ = 1000
⑤ 지수 0과 음의 지수
a⁰ = 1,  a^(−n) = 1/a^n
예제7⁰ = 1,  2⁻³ = 1/2³ = 1/8
⑥ 유리수 지수 = 거듭제곱근
a^(1/n) = ⁿ√a,  a^(m/n) = ⁿ√(a^m)
예제8^(2/3) = (∛8)² = 2² = 4
💡
왜 a⁰ = 1이고 a⁻ⁿ = 1/aⁿ일까? 법칙 ②가 계속 성립하도록 정의를 확장한 것입니다. a³ ÷ a³ = a⁰ 인데 값은 1이므로 a⁰ = 1. a² ÷ a⁵ = a⁻³ 인데 값은 1/a³ 이므로 a⁻³ = 1/a³. 규칙이 정의를 만드는 대표적인 예입니다.
⚠️
자주 나오는 함정 3가지. ① a^m × b^n 은 밑이 다르면 지수를 못 더합니다. ② 2³ × 2⁴ ≠ 4⁷ (밑은 그대로!). ③ (a + b)² ≠ a² + b² — 지수법칙은 곱셈·나눗셈에만 적용되고 덧셈에는 적용되지 않습니다.

무리수 연산과의 연결

√a = a^(1/2) 이므로, 2번 탭의 근호 법칙은 사실 지수법칙의 특수한 경우입니다: √a × √b = a^(1/2) b^(1/2) = (ab)^(1/2) = √(ab). 같은 법칙이 다른 옷을 입고 있을 뿐입니다.

로그란? — 지수의 역연산

"a를 몇 번 곱하면 N이 될까?"의 답이 로그입니다. 즉 로그는 지수를 꺼내 읽는 기호입니다.

a^x = N  ⟺  x = log_a N   (a > 0, a ≠ 1, N > 0)

예: 2³ = 8 이므로 log₂8 = 3.  조건 진수 N > 0, 밑 a > 0, a ≠ 1은 시험 단골!

로그의 사칙연산 — 지수법칙이 번역된 것

지수에서 곱셈이 덧셈으로 내려왔던 것을 기억하나요? 로그는 지수를 읽는 기호이므로, 지수법칙 ①②③이 그대로 로그법칙이 됩니다.

① 곱의 로그 = 로그의 합
log_a(MN) = log_a M + log_a N
예제log₁₀2 + log₁₀5 = log₁₀10 = 1
② 몫의 로그 = 로그의 차
log_a(M/N) = log_a M − log_a N
예제log₂48 − log₂3 = log₂16 = 4
③ 거듭제곱의 로그 = 지수가 앞으로
log_a M^k = k · log_a M
예제log₃9⁵ = 5 log₃9 = 5 × 2 = 10
④ 기본값 두 가지
log_a a = 1,  log_a 1 = 0
예제log₇7 = 1,  log₅1 = 0 (5⁰ = 1이므로)
⑤ 밑변환 공식
log_a b = log_c b / log_c a
예제log₄8 = log₂8 / log₂4 = 3/2
⑥ 밑변환의 따름 공식
log_a b = 1 / log_b a
예제log₂8 × log₈2 = 3 × 1/3 = 1
⚠️
가장 흔한 오개념: log(M + N) ≠ log M + log N 입니다! 로그의 덧셈이 되는 것은 진수의 곱셈이지, 진수의 덧셈이 아닙니다. 또 (log M)ᵏ ≠ k log M 도 조심하세요 — 지수가 진수에 붙어 있을 때만 앞으로 나옵니다.
🔗
왜 로그가 정수론·알고리즘에서 중요할까? 큰 수의 자릿수가 log₁₀으로 바로 나오고(2¹⁰⁰의 자릿수 = ⌊100 log₁₀2⌋ + 1 = 31자리), 이분 탐색·빠른 거듭제곱 같은 알고리즘의 횟수가 O(log n)으로 표현되기 때문입니다.

확인 퀴즈 — 8문제

보기를 클릭하면 바로 채점됩니다. 초록색이 정답, 빨간색이 오답입니다.

맞힌 개수: 0 / 8
1√9 가 속하는 가장 작은(좁은) 수의 집합은?
√9 = 3 입니다. 근호가 있어도 벗겨 보면 자연수! 물론 자연수이므로 정수·유리수·실수에도 모두 속합니다.
20.121212… (12가 무한히 반복) 는 어떤 수?
순환소수는 항상 분수로 나타낼 수 있어 유리수입니다. 실제로 0.121212… = 12/99 = 4/33.
32/3 + 1/6 = ?
통분하면 4/6 + 1/6 = 5/6. 공식으로는 15/18이 나오지만 GCD 3으로 약분해야 기약분수 5/6이 됩니다.
4√8 × √2 = ?
√8 × √2 = √16 = 4. 무리수 × 무리수가 자연수가 된 예 — 무리수는 곱셈에 닫혀 있지 않습니다!
51/(√5 − 2) 를 유리화하면?
켤레 (√5 + 2)를 분모·분자에 곱하면 분모는 합차공식으로 5 − 4 = 1. 따라서 답은 √5 + 2.
62⁵ ÷ 2⁸ = ?
지수의 뺄셈: 2^(5−8) = 2⁻³ = 1/2³ = 1/8. 음의 지수는 역수를 뜻합니다.
7log₂32 = ?
"2를 몇 번 곱하면 32?" 2⁵ = 32 이므로 log₂32 = 5.
8log₁₀4 + log₁₀25 = ?
곱셈법칙으로 log₁₀(4 × 25) = log₁₀100 = 2. log의 합 = 진수의 곱! (진수의 합 29가 아닙니다)
